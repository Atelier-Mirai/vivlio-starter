プログラミング言語

プログラミング言語でプログラムを開発することを「プログラミング」（programming）、プログラミング言語で記述したプログラムを「ソースコード」（source code）という。語彙、文法、記法などが自然言語よりも厳密に定義されており、記述したソースコードはソフトウェアによって自動的に解析、処理、変換などすることができる。

プログラミング言語は人間にとって理解、記述しやすい語彙や文法で構成された言語であり、そのままではコンピュータ（のCPU）が解釈、実行することができないため、ソフトウェアによってCPUが実行可能な言語（機械語、マシン語）によるプログラムに変換して実行される。開発時に一度にまとめて変換処理を行うことを「コンパイル」（compile）、そのような変換ソフトを「コンパイラ」（compiler）と呼び、実行時に変換と実行を同時並行で行うソフトウェアを「インタプリタ」（interpreter）という。

プログラミング言語は人間にとっての理解のしやすさや機械語に対する抽象度の高さによって分類されることがあり、機械寄りの言語を「低水準言語」（low-level language）あるいは「低級言語」と呼び、人間寄りの言語を「高水準言語」（high-level language）あるいは「高級言語」という。機械語の命令コードと一対一に対応する命令語を用いてプログラミング言語を行う低水準言語のことを特に「アセンブリ言語」（assembly language）と呼び、機械語への変換ソフトを「アセンブラ」（assembler）という。

プログラムをどのようなものとして捉え、構築していくかについて一定の設計思想やルールがある場合が多く、これに基いて分類されることもある。手続きを順番に記述していく手続き型言語（procedural language）あるいは命令型言語（imperative language）や、関連するデータ群と手続き群を一つのまとまりとして捉えるオブジェクト指向言語（object-oriented language）、プログラムを関数の組み合わせとして捉える関数型言語（functional language）、データ間の関係や論理を記述していく論理型言語（logic programming language）などの種類がある。

また、主な利用目的や主要な処理系の実装方式により分類することもあり、記述や実行の手間を軽減して迅速にプログラム開発ができる「スクリプト言語」（script language）あるいは「軽量言語」（LL：Lightweight Language）、特定の分野や処理に特化した「ドメイン固有言語」（DSL：Domain Specific Language）などの分類がある。

プログラミングとは

プログラミング 【programming】

プログラミングとは、コンピュータに意図した動作を行わせるために、まとまった処理手順を作成し、与えること。作成された手順のことをコンピュータプログラム（computer program）あるいは単にプログラムという。プログラミングを行う人や職種のことをプログラマ（programmer）という。

狭義には、プログラミング言語やそれに相当する仕組みや道具を用いて、人間が読み書きしやすい形式のプログラム（ソースコード）を記述していくコーディング（coding）作業を指す。広義には、その前後に行われる、設計や試験（テスト）、修正（デバッグ）、実行可能形式への変換（コンパイルやビルドなど）といった一連の作業を含む。

プログラムの作成
プログラミングを行うには、まず何をするプログラムを作るのかを明確に定義し、仕様や要件を自然言語で記述したり、大まかな処理の流れを箇条書きやフローチャートなどの図表を用いて設計する。集団でソフトウェア開発を行う場合はプログラムの記述者とは別の設計者が専門に作業を行い、仕様書や設計書などの形でまとめる場合もあるが、個人が小規模のプログラムを作成する場合はこの工程を頭の中で行い、作業や手順としては省略する場合もある。

どんなプログラムを作りたいか決まったら、これをコンピュータが解釈できるプログラミング言語を用いてソースコードとして記述していく。言語やプログラムの記述法には様々な種類があるが、手続き型の言語（手続き型プログラミング）の場合、実行すべき命令を先頭から順に書き下していく。必要に応じて、複数の命令をひとまとめにして名前をつけて呼び出せるようにしたり（関数やサブルーチンなど）、条件分岐や反復（繰り返し）などで命令の流れの制御を行う。

プログラムの実行
ソースコードそのものはコンピュータ（の処理装置）が解釈・実行できる形式ではないため、これを機械語（マシン語）のプログラムなど実行可能な形式に変換する必要がある。ソースコードを機械語などのコード（オブジェクトコード）に変換する工程をコンパイル（compile）と呼び、プログラムの起動処理やライブラリなど実行に必要なコードを連結する工程をリンク（link）という。これら一連の工程を行って実行可能ファイルやパッケージを作ることをビルド（build）という。

スクリプト言語（軽量言語）などの場合はこうした明示的な変換工程は不要で、ソースコードを機械語に変換しながら同時に実行するインタプリタなどの処理系で直に実行することができる。記述したコードをすぐ実行でき手軽だが、変換しながら実行するため実行可能ファイルを生成する場合より実行速度やメモリ効率では劣る。

プログラムの修正
作成したプログラムが一度で完全に思い描いたとおりに動作する場合もあるが、大抵は何らかの誤りや不具合を抱えているものである。このため、ビルドしたプログラムを実行してみてテスト（動作試験）を行い、仕様通りに動くか調べる。

誤り（バグ）が発見されると原因や解決策を考え、正しく動作するようにプログラムを書き換える（デバッグ）。バグには単純な記述ミスのようなものから、そもそも解くべき問題に対して選択した計算手順（アルゴリズム）が合っていないといった根本的なレベルのものまで様々な種類がある。

デバッグ作業が完了したら再びビルドとテストを行い、誤りが正されていることを確認する。このビルド→テスト→デバッグの繰り返しによって次第にプログラムの完成度や品質が上がっていき、実際に実用可能なプログラムに仕上げることができる。実際のプログラミングにおいては作業時間の多くがこの繰り返しの工程に費やされる。


## 三つの言語の特徴

* `HTML(HyperText Markup Language（ハイパーテキストマークアップランゲージ）)` <br>
  　HyperText Markup Language を日本語で表すなら、「ハイパーテキストに目印をつける言語」くらいの意味になります。<br>
  　目印をつける(Markup)というのは、文書の各部分（見出し・段落・表・リストなど)が果たしている役割が分かるようにすることです。そのために専用の「タグ」を使います。 <br>
  　コンピュータに理解できるよう文書構造を定義することが、HTMLの最も重要な役割です。

* `CSS(Cascading Style Sheet（カスケーディング・スタイルシート）)` <br>
  　ウェブサイトの見た目を飾り付け、彩る為の言語です。文字の大きさや色の指定から画像の配置など、様々な装飾を行うことができます。<br>
  　HTMLのタグが親子関係(包含関係)にある場合、多くの設定値は親要素に指定されたものが子要素、孫要素に引き継がれていきます。このように設定値が上から下へ伝播（でんぱ）していく様子を、階段状の瀧を意味する cascade（カスケード）になぞらえて、命名されました。

* `JavaScript（ジャバスクリプト）` <br>
  　ウェブサイトに、「双方向性・相互作用性（インタラクティブ）」を齎（もたら）すために用いられるプログラミング言語です。 <br>
  　例えば、閲覧者の操作に反応して表示が書き換わったり、ページが表示される際に写真などの要素に動きや効果を加えたり、サーバと通信してデータを取得したりするなど、現在のウェブサイトには欠かせないプログラミング言語となっています。


:::{.tip}
* HTML は、ウェブサイトの文書構造を記述します。
* CSS は、ウェブサイトの飾り付けを担当します。
* JavaScript は、ウェブサイトを制御します。
:::

HTMLとCSSに加えて、JavaScript を用いることで、より豊かなウェブサイトの作成が可能となります。JavaScript の基礎を学んで行きましょう。


## JavaScript の 使用方法

JavaScriptは、HTMLに組み込まれて使われることが多いプログラミング言語です。 [^128]

[^128]: Node.js（ノード・ジェイエス）など、ウェブサイトに読み込まれるのではなく、サーバサイドでも実行される環境もあります。

次に紹介するのは、画面に「こんにちは。JavaScript」と表示するためのプログラムです。

まず、土台となる `index.html` を以下のように記してください。

**▼index.html**

```
<!DOCTYPE html>
<html>
  <head>
    <meta charset="utf-8">
    <title>JavaScript練習</title>

    <!-- StyleSheetの読み込み -->
    <link rel="stylesheet" href="style.css">

    <!-- JavaScriptの読み込み -->
    <script src="hello.js"></script>
  </head>

  <body>
    <h1>JavaScript練習</h1>
  </body>
</html>
```

`<head>`内で、スタイルシートとJavaScriptを読み込むよう、記述しています。

```
    <!-- StyleSheetの読み込み -->
    <link rel="stylesheet" href="style.css">
```

```
    <!-- JavaScriptの読み込み -->
    <script src="hello.js"></script>
```

これにより `hello.js` という JavaScriptのプログラムを読み込んで実行できるようになります。

(特に必要ではありませんが) スタイルシートも記述しておきましょう。

**▼style.css**

```
h1 {
  color: orange;
  font-size: 36px;
  text-align: center;
}
```

そして `hello.js` という JavaScriptのプログラム は次のように書きましょう。

**▼hello.js**

```
// メッセージを表示する
alert("こんにちは。JavaScript")
```

書き終わりましたら、ブラウザから `index.html` を開いてみましょう。画面には「こんにちは。JavaScript」と言うメッセージが表示されます。

![](hello_js.png)

## JavaScript 解説

「こんにちは。JavaScript」と表示されました。
このプログラムに関して、それぞれの行を解説していきます。


### コメント

```
// メッセージを表示する
```

一行目の `//` と書かれた部分は、コメントです。 コードの説明書きに使います。

処理の大まかな塊ごとに、コメントがついていると、プログラムの全体像が掴みやすく、作成者の意図も分かりやすくなります。一行ずつ全てにコメントを入れる必要はありませんが、要所要所で分かりやすくコメントをつけるように心がけましょう。 [^129]

[^129]: `/* コメント */` というコメントの書き方もあります。


### alert 関数

```
alert("こんにちは。JavaScript")
```

二行目のコードを書くことで、画面にメッセージを表示することが出来ました。

JavaScriptには、よく使う処理に名前を付けて呼び出す働き・機能があります。
この働き・機能のことを「関数」と呼びます(英語では、機能「function」です)。
メッセージ表示機能は便利ですので、JavaScriptには標準で`alert関数`が用意されています。 [^130]
`alert('表示させたいメッセージ')` と書くことで、`alert関数`が使えます。

[^130]: 標準で備わっている関数を利用する他、プログラマが自分で好きな関数を作成できる機能もありますので、後述します。


### `;(セミコロン)`の有無

**▼セミコロン有りのコード**

```
// メッセージを表示する
alert("こんにちは。JavaScript");
```

**▼セミコロン無しのコード**

```
// メッセージを表示する
alert("こんにちは。JavaScript")
```

二種類のコードを掲載いたしました。とてもよく似ているコードで、違いは行末に`;(セミコロン)`が有るのか無いのかだけで、どちらも正しく動きます。

JavaScript には、自動セミコロン挿入機能があり、多くの場合、セミコロンを付けずとも正しく実行されます。ここでは見た目がすっきりするので、セミコロンを付けないで書くことにします。

[JavaScriptにセミコロンは入れるのか？入れないのか？](https://zenn.dev/tetsuyaohira/articles/3f84f3971f67d2) という記事もございます。よろしかったらお読みください。


## 変数

それでは、必要となるプログラミングに関する基礎知識を学んで行きましょう。

プログラミング言語には、文字列や数値などのデータに名前をつけることで、繰り返し利用できるようにする変数という機能があります。「変数」とは「値」が格納できる「箱」のようなものであると想像すると理解しやすいでしょうか。

より正確な定義が、[IT用語辞典](https://e-words.jp/w/変数.html) にございますので、確認してみましょう。

> 変数とは、コンピュータプログラムのソースコードなどで、データを一時的に記憶しておくための領域に固有の名前を付けたもの。変数につけた名前を変数名と呼び、記憶されているデータをその変数の値という。データの入れ物のような存在で、プログラム中で複数のデータを扱いたいときや、同じデータを何度も参照したり計算によって変化させたい場合に利用する。
>
>
> 変数をプログラム中で利用するには、これからどんな変数を利用するかを宣言し、値を代入する必要がある。コード中で明示的に宣言しなくても変数を利用できる言語もある。変数に格納された値を利用したいときは、変数名を記述することにより値を参照することができる。

定義ですので、少し難しいかもしれませんが、何度も出てきますので、すぐに使いこなせるようになる事と思います。


[note] <b>変「数」の名前の由来</a>

英語では、`variable` と言います。「変わりうるもの」という意味です。

例えばプログラムの中で、年齢を扱っているような場合を考えて見ましょう。10歳だった人は五年経てば15歳に、10年経てば20歳にと、年月の経過とともに変化して行きます。

あるいは、身長や体重の例はどうでしょうか。50.0kgの人は翌日になれば50.1kgになっているかもしれませんし、さらに次の日には49.8kgになっているかもしれません。

10   ⇒ 15   ⇒ 20

50.0 ⇒ 50.1 ⇒ 49.8

以上は「数値」が「変わりうるもの」という例でした。


他の例として「名前」はどうでしょうか。生まれたばかりの赤ちゃんにはまだ名前がありませんが「お七夜」の祝宴で「太郎」と命名され、家族に迎え入れられます。あるいは婚姻により「田中」さんが「山田」さんに名前が変わることもあります。

名無し ⇒ 太郎

田中   ⇒ 山田

以上は「文字列」が「変わりうるもの」という例でした。

`variable` を、変「数」という日本語に訳されましたが、「数」だけではなく「文字列」やそのほかいろいろ「変わりうるもの」全般を表すプログラミング用語であることを、心に留めておいてください。

[/note]


では、これから学ぶ JavaScript ではどのように「変数」を使えば良いのでしょうか。JavaScript では、どのような名前の変数を自分が書くプログラムで使うのか、最初に「宣言」する必要があります。変数宣言をする為のキーワードとして `const, let, var` の3つが用意されていますので、順に見て行きましょう。


:::{.tip}
* `let`   は、「変数」を宣言する時に使います。何度でも値の変更が可能です。
:::

それでは、使って見ましょう。

**▼変数宣言の例**

```
// 年齢を表す為に、age という変数を宣言する
let age
```

変数の宣言をすると、その変数に値を入れる（代入する）ことができます。

**▼変数に値を入れる例**

```
// 年齢を表す為に、age という変数を宣言する
let age

// age という変数に値を入れる
age = 0
```

変数の宣言と同時にその変数に値を入れる（代入する）こともできます。

**▼変数宣言と、変数への値の代入を同時に行う例**

```
// 年齢を表す為に、
// age という変数を宣言するとともに、
// age という変数に値を入れる
let age = 0
```

変数には値を何度でも代入することができます。
次の例は、0, 1, 2 と変化させた例です。

**▼変数への値の代入を何度も行った例**

```
age = 0
age = 1
age = 2
```

また、変数名の宣言は一度だけで十分です。同じ変数名で何度も宣言する必要はありませんし、また出来ませんので、ご注意下さい。

次の例はエラー（誤り）となります。

**▼変数宣言を何度も行った例**

```
let age
let age = 0
age = 1
let age = 2
```


[note] <b>=(イコール)記号について</a>

`=(イコール)`は、数学ですと、**ageと1が「等しい」**ということを意味する記号ですが、プログラミングの世界では **左の変数に右の値を入れる（代入する）** という記法になります。

初期には「等しい」ことを示す `=`と区別するために別の記号が用いられた時代もありましたが、C言語（シーげんご）やRuby（ルビー）, JavaScript（ジャバスクリプト）など現在主流の多くの言語では「代入する」記号として`=`を用いています。（「等しい」意味では == や === を用います。）

始めのうちは **`age ⇦ 1**` のことだと、頭の中で置き換えると良いでしょう。

[/note]

## 定数

それでは、次に「定数」を見て行きましょう。
[IT用語辞典](https://e-words.jp/w/定数.html) には、次のように書かれています。

> 定数とは、コンピュータプログラムのソースコードなどで、ある特定の数値やデータに名前を与えたもの。変数とは異なり、宣言時に決めた値をコードの中で後から変更することができない。
>
> プログラムのコード中で何度も参照される値で、実行時に変更する必要のないものに用いられる。値（数値や文字列）が開発の途中で変更される可能性がある場合や、値が長くて何度も入力すると打ち間違える危険がある場合、値に分かりやすい名前をつけたい場合などに用いられる。

「定数」[^132] を宣言する為には、`const` というキーワードを使います。英語の `constant(一定の、不変の)`に由来しています。定数は一度決めた値を変更することは出来ません。（しようとするとエラーになります。）

[^132]: 厳密には「再代入できない変数」の宣言です。


:::{.tip}
* `const`   は、「定数」を宣言する時に使います。初回のみ値を設定できます。
:::

それでは、使って見ましょう。

先程の変数の例では、`age = 0`や`height = 50`と書いていました。`age`という変数に`0`を代入しており、「年齢が零歳である」、「身長が50cmである」とすぐ理解することができます。

他の例を挙げて見ましょう。例えばじゃんけんゲームを作ることを考えて見ます。プレイヤーの手とコンピュータの手を比べて、強かったほうが勝ち というゲームです。

グーを `0`、チョキを `1`、パーを `2` で表すことにすると、

```
`player = 2`
`computer = 0`
alert("プレイヤーの勝ち")
```

のような、プログラムを書くことになりそうです。


ここで、この `0` という数値は一体どういう意味なのでしょうか。グーを意味しているのか、あるいはチョキなのか、パーなのか、覚えておくのも大変です。

そこで分かりやすい名前を付ける為に用いられるのが定数です。

**▼定数宣言の例**

```
// 定数宣言
const GUU   = 0 // グー
const CHOKI = 1 // チョキ
const PAA   = 2 // パー
```

`const GUU` と 定数を宣言すると同時に `= 0` で値を設定しています。

こうして定数を宣言しましたので、プログラム中では次のように使うことが出来ます。

**▼定数を利用した例**

```
// 定数宣言
const GUU   = 0 // グー
const CHOKI = 1 // チョキ
const PAA   = 2 // パー

// コンピュータの手
let computer = GUU
```

`let computer = GUU`と、プログラムを書くことが出来るようになりました。`let computer = 0` と書くよりも文字数は増えますが、`0`, `1`, `2` の単なる数字とは異なり、その意味するところが明確で分かりやすくなりますので、活用してきましょう。

`var` は、以前は使われていましたが、数々の不具合があるので、それに代わる変数宣言の為のキーワードとして `let, const` が導入されました。

今でも、古いプログラムコードには使われていますが、これから書くときには、`let` または `const` を用いるようにしましょう。


:::{.tip}
* `var` は、以前は使われていましたが、数々の不具合があるので、今日では使用しません。

## 命名規則

`age` という名前の変数や、`GUU` という名前の定数を宣言しましたが、変数や定数にはどのような名前を付けられるのでしょうか。

変数名には アルファベット(と一部の記号) `a-z, A-Z, $, _(アンダースコア)` が使えます。これらのアルファベット(と一部の記号)の後ろに `0-9` までの数字を続けることも出来ます。
また、JavaScript 自身が使う為の単語（予約語と言います）そのものは、変数名や定数名として宣言することは出来ませんが、予約語の一部を含むものは使うことができます。


:::{.tip}
**使える例**

* `let a;                 // 一文字の変数名も使えます`
* `let computer;          // 英単語も使えます`
* `let conputer;          // つづりが間違っていても使えます`
* `let player1;           // 英単語の後ろに数字を繋げても良いです`
* `let player9876;        // 沢山の数字を繋げることも出来ます`
* `let player9876;        // 沢山の数字を繋げることも出来ます`
* `let playerName;        // 英単語を繋げるときには(慣習的に)先頭を大文字にします`
* `let player_name;       // _ で繋げても良いです (JavaScriptでは一般的ではありません)`
* `const GUU;             // 定数を宣言する時には(慣習的に)全て大文字にします`
* `const GUU_CHOKI_PAA;   // いくつかの単語を繋げるときは _ で繋ぎます`<br>

* `let knife;             // if は予約語なので使えませんが、ifを含む単語は使えます`
:::

:::{.tip}
**使えない例**

* `let 123;               // 数字そのものを変数名にすることは出来ません`
* `let 20th_century_pear; // 数字から始まる変数名は使用出来ません`
* `let twentieth-century; // -(ハイフン)は使えない記号です`<br>

* `let if;                // if は予約語なので使えません。`
:::

## 演算子


### 四則演算

![](operator_five.png)

JavaScriptはプログラミング言語ですので、様々な計算を行うことができます。「**演算子**」とは聞きなれない用語かもしれませんが、平たく言えば、$7 + 5$ の $+$ や、$18 - 3$ の $-$ のように、どのような計算(演算)を行うかを示すための記号が「**演算子**」です。

足し算には$+$を、引き算には$-$を使うことは、日常慣れ親しんだ算数や数学と同じです。掛け算や割り算は、キーボードに $\times$ や $\div$ の記号がないので、代わりに `*（アスタリスク）`や`/（スラッシュ）`を用います。


### 剰余演算

「剰余」演算は耳慣れないかもしれませんが、平たく言えば、「余りを求める」演算の事です。

$$
22 \div 7 = 3 余り 1
$$


ですが、この余りの `1` を求める演算の事です。

$$
22 \% 7 = 1
$$


剰余演算は周期性の有るものの種類を分類する際などに有用です。例えば、ある年の元日(=1月1日)が月曜日だったとします。元日から七日経過した日(=1月8日)の曜日を知る為には、

$$
1 \% 7 = 1 (1日は月曜日)
$$


$$
8 \% 7 = 1 (8日も月曜日)
$$


と、7 で割った余りがどちらも1になるので、月曜日と知ることが出来ます。

元日から十日経過した日(=1月11日)の曜日を知る為には、

$$
11 \% 7 = 4 (1は月, 2は火, 3は水, 4は木曜日) <br>
$$


として、木曜日であることを求めることが出来ます。[^133]

[^133]: 年月日から曜日を求める公式として [ツェラーの公式](https://ja.wikipedia.org/wiki/ツェラーの公式) がよく知られています。


それでは、1月20日は何曜日になるでしょうか。1月28日はどうでしょうか。剰余演算に親しむところから、是非、求めて見てください。 [^134]

[^134]: `20 % 7 = 6`なので土曜日、`28 % 7 = 0`なので日曜日になります。

#### 単項演算子

単項演算子`-(マイナス)`は、ある数の正負を反転させるための演算子です。
と言うと難しく聞こえますが、負の数を表すには頭に負であることを示す`-(マイナス)`符号を付ければよいと、普段から使い慣れています。

例えば、`7` という数に、単項演算子`-(マイナス)`を作用させると、`-7` になります。あるいは数ではなく変数であっても同様です。`let score = 10` の場合、`-score` と書きます。

`2 + 3` のようによく使う四則演算の例ですと、加算演算子`+` は、二つの項(この場合は`2`と`3`)を必要とします。二つの項を必要とするので、二項演算子と呼ばれます。

正負を反転させるための演算子である、`-` は、一つの項のみを必要とするので、単項演算子と呼ばれます。

単項演算子`+(プラス)`は、ある数の正負を反転させることなくそのままに保つ演算子です。
例えば、`7` という数に、単項演算子`+(プラス)`を作用させると、`+7` となります。実用上の意味はありませんが、プログラミング言語の対称性のために備えられている演算子です。


#### 冪乗（べきじょう）

$$

$$


$$

JavaScriptでは、冪乗を行う演算子として、`**` を使います。`*(アスタリスク)` を空白を開けずに続けて書いてください。2つでひとまとまりの冪乗の為の演算子になります。

#### 代入演算子

代入演算子は、変数に値を設定する際に用いられる演算子です。これまでも `let age = 0` や `let height = 50`、`let score = 10`などとして、登場してきたものです。

`let age = 0` と、年齢を表す為の変数 `age` を宣言し、宣言と同時に、 その値として `0` を設定しています。その後、年齢が変更したならば、`age = 1` や `age = 5` のように書くことで、値を更新できます。

このように新しい変数に値を設定する際や、その後の更新を行う際に用いるための演算子が `=` です。


他に `+=, -=, **=, *=, /=, %=` と多くの演算子が用意されています。
これらは「複合代入演算子」または「自己代入演算子」と呼ばれる演算子です。

とあるゲーム(例えば野球など)で、現在の点数`score` が、`10`点だったとします。その後、満塁ホームランが決まり、点数 `score` を `4`点増やすことを考えます。このときのコードは次のように書けます。

```
score = score + 4
```

一見、奇妙な数式に見えますが、`=(イコール)`は、プログラミングの世界では **左の変数に右の値を入れる（代入する）** という記法として使われています。

ですので、`score` の値を `4` 点 増やしたものを、`score` に代入する という意味になります。


その後、二塁打が決まり、`2` 点獲得しました。これをコードで書くと、

```
score = score + 4
score = score + 2
```

となります。このような処理は頻出する為、次のように簡潔に書けるようになっています。

```
score += 4
score += 2
```

これが「複合代入演算子」別名「自己代入演算子」です。

「`score` を `4` 点 増やす」と、そのまま読む事もできます。


「`1` 点 増やす」際には次のように書けます。

```
score += 1
```

そして、この場合、インクリメント演算子（増分演算子）を用いると、さらに短く次のように書けます。

```
score++
```


## 条件分岐

プログラムを作っていく中で、条件分岐を行う事は欠かせません。
特定の条件を満たすか否かにより、行う処理を分岐することができます。

ここでは `if（イフ）文` を使った条件分岐について学んでいくとともに、
論理NOT, 論理AND, 論理OR, 比較演算子, 等価演算子, 条件演算子 についてもご紹介いたします。


### `if文`


`if文` は次のような構文が基本形となります。
条件式の評価結果が `true（トゥルー）` (真（しん）・成り立つ)であるならば、実行する文が実行されます。

（反対語は `false（フォールス）` (偽（ぎ）・成り立たない)で、後ほど登場します。）

**▼if文**

```
if (条件式) {
  実行する文
}
```

**もし、条件式が真(成り立つ)ならば、波括弧{ \**内に書かれた文を実行する } という意味です。

例として、映画館の料金を表示するプログラムを考えてみましょう。

次のコードでは条件式が `true` (真・成り立つ)であるため、 `if文` の中身が実行され、コンソール画面に「大人料金です」と表示されます。

**▼if文の条件式がtrue**

```
// 年齢は20歳
let age = 20

if (age >= 18) {
  console.log("大人料金です")
}
```

コンソール画面で確かめて見ましょう。
少し長い行数ですので、直接入力しても良いですが、エディタに入力し、コンソール画面に貼り付けると、入力ミスもなく安心です。

### 比較演算子, 等価演算子

さて、上の例では、年齢が18歳以上であるか、あるいは12歳以上であるかなどを調べていました。この判定に用いた演算子が、比較演算子です。


**比較・等価演算子**

| 　　 | 　　 |
| --- | --- |
| `age < 18` | 18歳未満 |
| `age <= 18` | 18歳以下 |
| `age > 18` | 18歳超 |
| `age >= 18` | 18歳以上 |
| `age === 18` | 18歳である |
| `age !== 18` | 18歳ではない |


### 条件演算子(三項演算子)

とある映画館では、大人料金が1000円、子ども料金が500円だとします。


**料金表**

| 区分 | 料金 |
| --- | --- |
| 大人料金 | 1000円 |
| 子ども料金 | 500円 |
`if文` を使うと、次のように書けます。

```
// 料金の為の変数宣言
let price

age = 20
if (age >= 18) {
  price = 1000
} else {
  price = 500
}
```

これと同等のことを、条件演算子(三項演算子)を使って書くと次のように書けます。

```
price = (age >= 18) ? 1000 : 500
```

`age >= 18` が「真ならば、成り立つならば、`true`ならば」`1000`、そうでなければ、`500` が、`price` に代入されます。

二項演算子 `+` は、二つの項を取りますので、

「1000円 + 500円」

のように、演算子 `+` は一つで、「1000円」と「500円」が二つの項になります。

三項演算子では、演算子は`?` と `:` の二つで、「大人」「1000円」「500円」と三つの項を取ります。

「大人 ? 1000円 : 500円」

と、質問しているような構文です。

```
// 二項演算子
price = 1000 + 500

// 三項演算子
price = 大人 ? 1000 : 500
```

対比が見て取れるかと思います。簡単な条件式であれば、if文よりも短く簡潔に記述することが出来ますので、使って見てください。


### 論理演算子 - 論理NOT, 論理AND, 論理OR

とある映画館では、大人料金が1000円、子ども料金が500円でしたが、女性向けに料金を割り引くこととしました。大人の女性であれば800円にします。


**料金表**

| 区分 | 料金 |
| --- | --- |
| 大人料金 | 1000円 |
| 　女性割引料金 | 800円 |
| 子ども料金 | 500円 |
こういった場合に活躍するのが、「論理演算子」です。`NOT(否定), AND(かつ), OR(または)` の三種類があり、演算子の記号は、`!, &&, ||` です。


**論理演算子**

| 名称 | 意味 | 演算記号 |
| --- | --- | --- |
| `論理NOT(論理否定)` | 否定 | `!(エクスクラメーションマーク一つ)` |
| `論理AND(論理積)` | かつ | `&&(アンパサンド二つ)` |
| `論理OR(論理和)` | または | `||(バーティカルバー二つ)` |


#### 論理AND(論理積)演算子

論理AND(論理積)演算子を使い、女性向けの割引料金を設定する `if文` は次のようになります。

```
// 料金の為の変数宣言
let price
// 性別は女性である
let gender = "female"
// 年齢は20歳である
let age = 20

if (age >= 18 && gender === "female") {
  // 18歳以上 かつ 性別が女性
  price = 800
} else if (age >= 18) {
  // 18歳以上
  price = 1000
} else {
  // それ以外
  price = 500
}
```

8行目の `age >= 18 && gender === "female"` に着目してください。
「18歳以上」 かつ 「性別が女性に等しい」ことを表現する為に、論理AND(論理積) `&&(アンパサンド二つ)` を使っています。

また、論理AND(論理積) `&&`  を用いて、範囲を示すことも出来ます。

```
// 18歳以上 かつ 65歳以下
if (18 <= age && age <= 65) {
```

機会があれば使って見てください。


#### 論理OR(論理和)演算子

次に、論理和の例もやって見ましょう。

女性向けの割引料金が好評であったので、高齢者向けにも対象を拡大して、
65歳以上であるか、または、女性であれば 特別料金800円とすることにしました。


**料金表**

| --- | --- |
| 大人料金 | 1000円 |

  // 18歳以上
  price = 1000
} else {

  price = 500

```

1行目の `age >= 65 || gender === "female"` に着目してください。
「高齢者」 または 「性別が女性に等しい」ことを表現する為に、論理OR(論理和) `||(バーティカルバー二つ)` を使っています。


#### 論理NOT(論理否定)演算子

最後に、論理NOT(否定)をご紹介します。

先ほどの映画館の例で見て行きましょう。
特別日には、特別なイベントを行うことにし、その旨をお知らせすることにします。

「特別日」は、毎月一日の映画の日の他、親子連れで来やすいよう、春休み、夏休み、冬休み期間中の日程となります。

「年齢が18歳以上である」のように簡潔な条件であれば、すぐに `age >= 18` とその真偽を確かめる条件式を書くことが出来ますが、「特別日」のような複雑な条件の場合には、(後ほど学びますが)真偽値を返す「関数」を作成するのが一般的です。

関数名は `isSpecialDay` としましょう。

`isSpecialDay` は、本日が特別日の場合には `true`、普通日の場合には `false` を返すことにします。

`true` は、条件式が「真・成立している」ことを示す特別の値です。
`false` は、条件式が「偽・成立していない」ことを示す特別の値です。 [^135]

[^135]: 真偽値(真理値)についてもう少し説明すべきですが、紙幅の都合上、割愛いたします。概要は https://e-words.jp/w/真理値.html にございますので、三項になさってください。

```
if (isSpecialDay()) {
  alert("本日は特別日です。イベントあります。")
}
```

これで、「本日は特別日です。イベントあります。」とのお知らせが表示されるようになります。

`if (age >= 18) {` で18歳以上であるかどうかを判定しましたが、
`if (isSpecialDay()) {` で特別日であるかどうかを判定、 `alert()`関数でお知らせを表示しています。

それでは、普通日の時に、イベントがないことをお知らせするにはどのように書いたら良いでしょうか。

ここで登場するのが、論理NOT(否定)です。論理NOT、論理否定は、「真偽値を反転させる」際に用います。条件式の「真・成立している・true」を「偽・不成立・false」に「偽・不成立・false」を「真・成立している・true」にします。

論理NOTを使うと、先ほどの `isSpecialDay` 関数を用いて、次のように書けます。

```
if (!isSpecialDay()) {
  alert("本日は普通日です。イベントありません。")
}
```

`isSpecialDay` の 先頭に、`!` が付いていることに注意してください。論理否定演算子`!` を `isSpecialDay` に作用させることにより、真偽値を反転させています。

### 入れ子(ネスト)になった `if文`

先ほどの映画館の例で見て行きましょう。再び料金改定を行い、特別日と普通日に分けて料金設定することにしました。


**料金表**

| 日 | 区分 | 料金 |
| --- | --- | --- |
| 特別日 | 大人料金 | 700円 |
|  | 子ども料金 | 300円 |
| 普通日 | 大人料金 | 1000円 |
|  | 子ども料金 | 500円 |
わたくしたちは、すでに論理積演算子を学んでいますので、これを用いて次のように書くことも出来ます。

```
if (特別日 && 大人) {
  price = 700
} else if (特別日 && 子ども) {
  price = 300
} else if (普通日 && 大人) {
  price = 1000
} else if (普通日 && 子ども) {
  price = 500
}
```

そして、ここでは、入れ子(ネスト)になった `if文` をご紹介いたします。

次のように `if文` の中に `if文` を入れることが出来ます。

```
// 入れ子になった if 文
if (特別日) {
  // 特別日の料金処理
  if (大人) {
    price = 700
  } else {
    price = 300
  }
} else {
  // 普通日の料金処理
  if (大人) {
    price = 1000
  } else {
    price = 500
  }
}
```

論理積でまとめる代わりに、特別日、普通日、それぞれの場合で、料金設定の為の処理を記述しています。

具体的なコードにすると以下のようになります。

```
// 入れ子になった if 文
if (isSpecialDay()) {
  // 特別日の料金処理
  if (age >= 18) {
    price = 700
  } else {
    price = 300
  }
} else {
  // 普通日の料金処理
  if (age >= 18) {
    price = 1000
  } else {
    price = 500
  }
}

// 特別日判定関数
// 特別日なら true を
// 普通日なら false を返す
function isSpecialDay() {
  let today = new Date()
  let d = today.getDate()
  if (d === 1) {
    return true
  }
  return false
}
```

分かりやすさに応じて、適宜使い分けしてください。


## 繰り返し

プログラミングに不可欠な三つの処理は、「逐次」「条件」「反復」です。
「逐次」は上から下に順番にコードを実行することです。そして「条件」判断を行い、分岐処理が為されます。最後に「反復」です。同じ処理を繰り返し何度でも実行することです。


先に登場したコードにもう一度登場してもらいましょう。特別日であることを宣伝しています。

```
alert("本日は特別日です。イベントあります。")
```

それでは、三回宣伝したいときはどのようにすれば良いでしょうか？

```
alert("本日は特別日です。イベントあります。")
alert("本日は特別日です。イベントあります。")
alert("本日は特別日です。イベントあります。")
```

このように三回書いたらできます。それでは百回表示させたいときはどうでしょうか。エディタのコピー機能を使うこともできますが、それでも百回書くのは大変です。

同じ処理を繰り返えすための構文として、JavaScript には `for 文` が備わっています。さまざまな書き方が可能ですが、ここでは基本の `for文` を使って見ましょう。`for 文` は、反復したい回数がわかっているときに用いると便利です。 [^136]

[^136]: 反復回数が不明な場合には、while文を用いることが出来ます。MDN [while](https://developer.mozilla.org/ja/docs/Web/JavaScript/Reference/Statements/while)に説明がございます。

```
// 三回宣伝するためのコード
for(let i = 0; i < 3; i++) {
  alert("本日は特別日です。イベントあります。")
}
```

![](for0.png)
![](for1.png)
三回、宣伝できました。青い「OK」ボタンを押すと、残り二回表示されますのでご確認下さい。上のコード中、 `3` と書かれているところを、`100` と書き換えると、百回表示させることもできます。 [^137]

[^137]: [アラートループ事件](https://ja.wikipedia.org/wiki/アラートループ事件) 兵庫県警によりウイルス罪として女子中学生ら五人が捕えられた誤認・冤罪事件。JavaScriptの生みの親であるブレンダン・アイクは「Chromeの初版よりも10年も前にリリースされたNetscape 4でもユーザーがループするJavaScriptをキルすることができた。」「この事件の公判で専門家証人になるつもりでいる。これはウイルスなどではなく、犯罪扱いされるべきではない。」とツイートした。

for文は繰り返す範囲を指定した反復処理を書くことができます。
名著「 [JavaScript Primer 迷わないための入門書](https://jsprimer.net/)」から、 `for文` の説明を見てみましょう。

**▼for文**

```
for (初期化式; 条件式; 増分式) {
    実行する文
}
```

for文の実行フローは次のようになります。

1. 初期化式 で変数の宣言
2. 条件式 の評価結果がtrueなら次のステップへ、falseなら終了
3. 実行する文 を実行
4. 増分式 で変数を更新
5. ステップ2へ戻る

---
もう少し、実用的な例をやって見ましょう。
次のコードは、`for文` で 1から10までの値を合計して、その結果を出力するプログラムです。

**▼1から10までの合計を求めるプログラム**

```
// totalの初期値は0
let total = 0

// for文の実行の流れ
// まず、iを1で初期化
// iが10以下（条件式を満たす）ならfor文の処理を実行
// iに1を足し(i++)、再び条件式の判定へ
for (let i = 1; i <= 10; i++) {
  // 1から10の値をtotalに加算している
  total = total + i
}

// コンソールに55が出力される
console.log(total)
```

#### 整数型

整数型は、-9007199254740991($- 2^{53} - 1$) 〜 9007199254740991($2^{53} - 1$) までを扱えます。

大きな数値の時には、-9007_1992_5474_0991 〜 9007_1992_5474_0991 のように、`_アンダースコア` で読みやすく区切ることも出来ます。約±9000兆の範囲の整数を扱えることが分かります。


#### 浮動小数点型

浮動小数点型は、非常に大きな数値や、非常に小さな数値を表す際に便利です。

$\pm 2^{-1074}$ から $\pm 2^{1024}$ までの範囲の数を表すことが出来ます。

JavaScript で扱える最小値 $2^{-1074}$ は、$\frac{1}{2^{1074}}$ のことです。

$$
\frac{1}{2^{1074}} = \frac{1}{20...(320桁省略)...84} ≒ 0.00...(320桁0が続く)...05
$$


と、とても小さな数です。このような小さな数を表す為には、指数表示が有用です。

次は指数表示の例です。0.0005 と小さな数を、5 × 10の何乗かで表しています。

$$
\frac{1}{2000} = 0.0005 = 5 \times 10^{-4} = 5e\text{-}4
$$


JavaScript で扱える最大値 を $2^{1024}$ を指数表示すると次のようになります。

$$
  2^{1024} = 1797...(300桁省略)...37216 = 1.797 \times 10^{308} = 1.797e308
$$

### 整数型 と 文字列型 の型変換

一般にプログラミング言語では、整数型の「123」と文字列型の「"123"」は異なるものとして扱われます。

例えばウェブサイトのフォーム上から、年齢「10」歳と入力された値も、全て「文字列」として扱われます。

「文字列型」から「数値型」へ変換すると、正しく計算を行うことができます。 [^139]

[^139]: JavaScriptには暗黙の型変換機能が備わっており、一応それらしい結果を返すときもありますが、いらぬ不具合を避ける為にも、明示的に型変換することをお薦めします。


#### 文字列型のデータ から 数値型のデータに 型変換する

```
// 変数str に 文字列型のデータ「"123"」を設定します。
str = "123"

// 変数num に
// 文字列型のデータ「"123"」を型変換して、
// 数値型のデータ「123」を設定します。
num = Number(str)
```


#### 数値型のデータ から 文字列型のデータに 型変換する

```
// 変数num に 数値型のデータ「123」を設定します。
num = 123

// 変数str に
// 数値型のデータ「123」を型変換して、
// 文字列型のデータ「"123"」を設定します。
str = String(num)
```


## 配列


#### 配列

配列はとてもよく使われるデータ構造です。いくつかの変数を一緒のものとして扱いたい時に、重宝します。

ここでは、山田さんのテストの点数を扱うことにしましょう。
国語が85点、算数が95点、理科が85点、社会が75点でした。
合計点は何点でしょう。平均点は何点でしょう。

素朴にそれぞれ変数を使って求めると次のようなコードになります。

```
kokugo = 85
sansuu = 95
rika   = 85
shakai = 75

goukei = kokugo + sansuu + rika + shakai
heikin = goukei / 4
```

科目数が増えてくるとどうでしょうか。
今回は山田さんの点数のみでしたが、鈴木さんや田中さんの点数も扱いたい場合はどうしましょうか。

こういった場合、「配列」を使うと、点数のデータをまとめて扱えるのでとても便利です。

テストのそれぞれの科目の点数の集まり「配列」として、 `scores` という変数を宣言します。配列は、データの入れ物であり、宣言したばかりでは、入れ物としての器はありますが、その中身がない状態です。

初期値として `85, 95, 85, 75` の四つの「要素」があるようにします。配列に格納されたそれぞれのデータのことを、「要素」と呼びます。

**▼配列の宣言**

```
scores = [85, 95, 85, 75]
```

`scores = []` と書き、中身が空っぽの配列を作成しています。次に、`scores = [85, 95, 85, 75]` と書き、 空っぽであった `配列 scores` の中に、 四つのの要素 `85, 95, 85, 75` を格納しています。[^140]

[^140]: 容器だけで中身は空っぽの幕の内弁当と、おかずとして 85, 95, 85, 75 が入っている幕の内弁当を想像すると分かりやすいでしょうか。


#### 配列内の要素の指定法

配列内の各要素を指定するには、配列名の後に `[何番目かを指示する数字]` と書きます。この「何番目かを指示する数字」のことを、**添字(そえじ)**と呼びます。

配列の要素は、 `0, 1, 2` と  `0` から数え始めます。`scores[0]` と書くと `85` を取得できます。`scores[1]` と書くと `95` を、`scores[2]` と書くと `85`を、`scores[3]` と書くと `75` を取得できます。

配列はとてもよく使う基礎的な**データ構造**で、プログラミングでは不可欠なものです。


#### 配列を使ったプログラミング

合計点と平均点を求めて見ましょう。

```
// 点数配列を宣言 ＆ 初期データを投入
let scores = [85, 95, 85, 75]

// 合計点を、for 文を使って求める
let total = 0
for (let i = 0; i < 4; i++) {
  total = total + scores[i]
}

// 平均点を求める
let average = total / 4

// 結果を表示する
console.log(total)
console.log(average)
```

以前に学んだ繰り返しの為の `for` 文 を使って、合計点を求めています。
`total` を 0に初期化しておき、その後 `scores[0], scores[1], scores[2], scores[3]` と順次足し込んで行くことで、合計点が求められます。

平均点は、合計点を `4` で割ることで求められます。

科目数が十二科目になった場合にも変更は容易ですし、最高点や最低点などの算出も楽です。
配列の利点が見えるのではないでしょうか。


#### `for...of`文

繰り返しを実行するために、`for...of`文 を用いることも出来ます。
先の例では、個々の要素を指定する為に、変数 `i` を用意し、`scores[0], scores[1], scores[2], scores[3]` と変化させることで、全ての要素を取得しました。
取得したいのは「配列の全ての要素」であって、その為の手段として、変数 `i` を変化させることで、`scores[0], scores[1], scores[2], scores[3]` を取得しています。であれば、変数 `i` を用意することなく、「配列の全ての要素」を取得できたほうが、やりたい本質的な処理に注力できますし、コードも読み書きしやすくなります。

その為の構文が、`for...of`文 です。 MDN [for...of](https://developer.mozilla.org/ja/docs/Web/JavaScript/Reference/Statements/for...of) にも詳しく説明がございますが、まずは使って見ましょう。

```
// 点数配列を宣言 ＆ 初期データを投入
let scores = [85, 95, 85, 75]

// 合計点を、`for...of` 文 を使って求める
let total = 0
for (const score of scores) {
  total += score
}

// 平均点を求める
let average = total / scores.length

// 結果を表示する
console.log(total)
console.log(average)
```

6行目の `for (const score of scores) {` に注目してください。
配列 `scores` の要素が順番に `score` に取り出されます。
7行目の `total += score` で、`total += score` で、合計の為の変数 `total` に 足し込んでいます。
「配列の全ての要素を取得し、合計を取る」という処理の本質のみを簡潔に実装できています。
配列に慣れてきましたら、使って見てください。

配列の補足となりますが、配列には「いくつ要素があるか」を取得する為のメソッド（関数）も備わっています。

11行目 では 平均点を求める為に、`let average = total / scores.length` と書かれています。先のコードでは、 `total / 4` と、 `4` で割ることで平均点を求めていました。 科目数が増えるたびに、`total / 5` や `total / 6` のように修正することが必要で、手間ですし、修正漏れから誤ったプログラムになってしまう虞（おそ）れもございます。配列の要素数を返すメソッド（関数）`length` を用いることで、 `scores.length` と簡潔・明快に書けます。活用して行きましょう。

socres.splice(3, 1)

## 連想配列(ハッシュ)

配列と並ぶ重要な**データ構造**に、 **連想配列** (ハッシュや辞書とも呼ばれます)があります。 [^141]

[^141]: JavaScriptでは連想配列(ハッシュ・辞書)と呼ばれる特別な種類があるのではなく、後ほど紹介する「オブジェクト」として扱われています。

```
// 配列の宣言と初期値設定
let scores = [85, 95, 85, 75]
```

先に紹介した配列では、`0, 1, 2, 3` と言う添字で要素を指定する必要がありました。そのため、0番目の要素が国語であり、1番目の要素が算数であり、2番目の要素が理科であり、3番目の要素が社会であると言うことを把握しておく必要がありました。

把握しておくのも大変ですし、また、指定する添字を誤ってしまうと、当然違う科目の点数を得ることとなります。

「連想配列」では、`0, 1, 2, 3` と言う添字に代えて、`kokugo, sansuu, rika, shakai` という科目名で指定できるようになります。

```
// 連想配列の宣言と初期値設定
let hash_scores = { kokugo: 85, sansuu: 95, rika: 85, shakai: 75 }
```


### 要素の取得

配列では、各要素の値(バリュー)を取得する為に添字を用いましたが、連想配列では、「キー(鍵)」を用いて、各要素の値（バリュー）を取得します。

国語の点数を取得する為には、キー `kokugo` を用いて、次のように書きます（`[]ブラケット`を使うので、ブラケット記法と呼ばれます）。

```
hash_scores["kokugo"]
```

連想配列の各要素にアクセスしやすいよう、`.` を用いるドット記法も用意されています。

```
hash_scores.kokugo
```


### 要素の更新、追加、削除

要素の更新や追加は配列と同様に行えます。

国語の点数を95点に更新する例です。ブラケット記法、ドット記法、それぞれの例を示します。

```
hash_scores["kokugo"] = 95
hash_scores.kokugo = 95
```

新しい要素を追加することも出来ます。

```
hash_scores["programming"] = 100
hash_scores.programming = 100
```

削除するためには、`delete` 演算子を用います。

```
delete hash_scores["shakai"]
delete hash_scores.shakai
```

コンソール画面での実行例です。ぜひご自身で入力して確かめてください。

### `for...in` 文

[for...in](https://developer.mozilla.org/ja/docs/Web/JavaScript/Reference/Statements/for...in) 文は、配列の際の `for...of` 文 と同様、「連想配列の全ての要素」を処理する際に便利な繰り返しの為の構文です。

```
// 連想配列を宣言 ＆ 初期データを投入
let hash_scores = { kokugo: 85, sansuu: 95, rika: 85, shakai: 75 }

// 合計点を、`for...in` 文を使って求める
let total = 0
for (let key in hash_scores) {
  let value = hash_scores[key]
  total += value
}

// 平均点を求める
let count = Object.keys(hash_scores).length
let average = total / count

// 結果を表示する
console.log(total)
console.log(average)
```

配列の場合とほぼ同様のコードとなっておりますが、変更点をご案内します。

一般に 連想配列は、`hash = { key1: value1, key2: value2, key3: value3 ... }` と、 `key` と `value` の組み合わせになっています。

6行目の `for (let key in hash_scores) {` では、連想配列の `key` が入ります。つまり、繰り返しの都度、`key` には `kokugo, sansuu, ... ` など科目名が入ります。

ここで得られたキーを用いて、各科目の点数を取得しているのが、7行目のコード `let value = hash_scores[key]`です。

これで `value` には、各科目の点数が入りましたので、 `total` に足し込んで、合計点を求めることが出来ます。


以上で合計点を求めることができましたが、平均点を求めるための処理である12, 13行目も解説します。

平均点を求める為には、合計点数を科目数で割る必要があります。4科目あるので4で割れば良いのです。配列の場合は要素数を `lenght` メソッドで取得することができましたが、連想配列の場合には、要素数を求めるメソッドは用意されていないため、少し手間をかける必要があります。

まず、 `Object.keys`メソッドにより 連想配列からキーのみを取り出した配列を作ります(`["kokugo", "sansuu", "rika", "shakai"]`という新しい配列が得られます)。
(それが `Object.keys(hash_scores)` と書かれている部分です)

次に、この取得したキー配列の要素数を求める為に、`lenght` メソッド を使います。
(それが `Object.keys(hash_scores).length` と書かれている部分です)

この二段構えで、連想配列の要素数（＝科目数）を取得することができたので、 `count` という変数に代入しています。

最後に 13 行目で、合計点を科目数で割って、平均点を求めることが出来ました。


少し手間暇かかりますが、連想配列も、配列と並んでよく使われます。少しずつ習得なさってください。


## 関数

関数とは、ある一連の手続き（文の集まり）を一つの処理としてまとめる機能です。関数を利用することで、同じ処理を毎回書くのではなく、一度定義した関数を呼び出すことで同じ処理を実行できます。
[^142]

[^142]: 例えば、`if文`を学ぶ際に登場した`isSpecialDay()`関数もそうです。

JavaScriptでは、関数を定義するために `function` キーワードを使います。 `function` から始まる文は関数宣言と呼び、次のように関数を定義できます。

**▼関数宣言**

```
// 関数宣言
function 関数名(仮引数1, 仮引数2) {
    // 関数が呼び出されたときの処理
    // ...
    return 関数の返り値
}

// 関数呼び出し
const 関数の結果 = 関数名(引数1, 引数2)
alert(関数の結果) // => 関数の返り値を表示する
```

関数は次の4つの要素で構成されています。


:::{.tip}
**関数を構成する4つの要素**<br>

- 関数名  
  利用できる名前は変数名と同じ

- 仮引数  
  関数呼出時に渡された値が入る変数で、複数の場合は,(カンマ)で区切る

- 関数の中身  
  「{」と「}」で囲んだ関数の処理を書く場所

- 関数の返り値  
  関数を呼び出したときに、呼び出し元へ返される値
:::

宣言した関数は、`関数名()`と関数名に括弧を付けて呼び出します。
関数を引数（ひきすう）と共に呼ぶ際は `関数名(引数)` とし、引数が複数ある場合は  `関数名(引数1, 引数2)`のように、`,（カンマ）` で区切ります。
関数内では `return文` により、関数の実行結果として任意の値を返せます。 [^143]

[^143]: 出典：JavaScript Primer 迷わないための入門書


それでは、理解を深めるために、例を挙げてみましょう。
とても単純ですが、a と b という二つの数を足し算する関数です。

```
// 関数宣言
// 関数名は addで 仮引数は a と b
function add(a, b) {
  // 関数が呼び出されたときの処理
  // 二つの数を足して、answer という 変数に代入する。
  let answer = a + b
  // return文を使い、関数の返り値を、呼びだし元に返す。
  return answer
}

// 関数呼び出し
// 足し算を行う関数 add に 引数 5 と 27 を渡して、呼び出す。
// そして、呼び出した結果を、result という変数で受け取る。
let result = add(5, 27)
// 関数の結果を表示する
alert(result)
```

#### アロー関数

ここでは、新しい関数の書き方「**アロー関数**」について、ご紹介いたします。
（アロー関数は、関数を表す為の記号 ` => ` が、「矢印、arrow（アロー）」に見えるので、そう呼ばれます。）

二つの数 `a` と `b` を与えると、その和を返す関数`add`を定義してみましょう。

新しく導入されたアロー関数を使う場合は、次のように書きます。

**▼新しく導入されたアロー関数**

```
// アロー関数
const add = (a, b) => {
  return a + b;
}
```

従来の関数宣言では次のようになります。

**▼従来の関数宣言**

```
// 従来の関数宣言
function add(a, b) {
  return a + b;
}
```

使い方はどちらも同じで、次のようにして結果を表示できます。

**▼関数の呼び出し**

```
// 関数の呼び出しと結果の表示
const answer = add(9, 23);
alert(answer);
```


#### 従来型の関数と「アロー関数」への変形

「変数」とは「値」が格納できる「箱」のようなものです。ですので「9」や「23」といった数値、「September」などの文字列を代入（格納）できるのは、理解しやすいでしょう。

そして、JavaScriptの関数は「第一級オブジェクト」と呼ばれ、関数を変数に代入できる！性質を持ちます。9と23を足し算した結果である32を変数に代入するのではなく、「足し算する」という「機能・働き・関数」そのものを変数に代入できるという、特筆すべき性質を持っています。

ですので、次のように 変数 `add` に 関数 `tashizan`を代入することができます。

```
// 変数 add に 関数tashizanを代入する
const add = function tashizan(a, b) {
  return a + b;
}
```

そして使うときには、あたかも変数`add`が関数であるかのように（関数を代入しているのですから全く関数と同じように）利用できます。

```
// 関数の呼び出しと結果の表示
const answer = add(9, 23);
alert(answer);
```

変数 `add` に代入して、「足し算をする」という機能を呼び出せるのであれば、わざわざ 関数に `tashizan` と命名する必要も無いので、名前を省けます（「無名関数」と言います）。

```
// 変数 add に 無名関数を代入する
const add = function (a, b) {
  return a + b;
}
```

`function` と毎回書くのも文字が長いので省略して、代わりに アロー関数の記法 ` => ` を ` () ` の次に書きます。

**▼アロー関数の完成**

```
const add = (a, b) => {
  return a + b;
}
```

このテキストでは、`function` というキーワードの存在が、始めての方に分かりやすいかと考え、従来型の関数宣言でコーディングしています。

「アロー関数」は、従来型の関数宣言と全く同一のものではありませんが、簡潔に書けるよう、これからの主流として導入され、広く用いられていますので、ぜひ使ってみてください。


またより詳しい説明が、(準)公式サイトMDN [アロー関数式](https://developer.mozilla.org/ja/docs/Web/JavaScript/Reference/Functions/Arrow_functions) にございますので、ご興味を持った方はお読みになってください。


## オブジェクト指向

プログラミングを行う上で、大切な考え方に「オブジェクト指向」があります。
ウィキペディアより引用してご紹介いたします。

> **オブジェクト指向**は、ソフトウェア開発とコンピュータプログラミングのために用いられる考え方である。元々は特定のプログラミングパラダイムを説明するために考案された言葉であり、その当時の革新的技術であったGUI（グラフィカル・ユーザーインターフェース）とも密接に関連していた。明確な用語としては1970年代に誕生し、1990年頃にはソフトウェア開発の総合技術としての共通認識を確立している。
>
> **オブジェクト**とは、**データ構造とその専属手続きを一つにまとめたもの** / **情報資源とその処理手順を一つにまとめたもの** を指している。データとプロセスを個別に扱わずに、双方を一体化したオブジェクトを基礎要素にし、メッセージと形容されるオブジェクト間の相互作用を重視して、ソフトウェア全体を構築しようとする考え方がオブジェクト指向である。
>
> オブジェクト指向（object-oriented）という言葉自体は、1972年から80年にかけてプログラミング言語「Smalltalk」を開発したゼロックス社パロアルト研究所の計算機科学者アラン・ケイが、その言語設計を説明する過程で誕生している。大学院時代のケイがプログラミング言語「Simula」に感化されて日夜プログラミング・アーキテクチャの思索に耽っていた1967年頃、今何をしているのかと尋ねてきた知人に対して「object-oriented programmingだよ」とその時の造語で答えたのが原点であるという。
>
> **オブジェクト指向プログラミング(OOP)** とは、「**オブジェクト**」という概念に基づいたプログラミングパラダイムの一つである。 オブジェクトは、任意個数のフィールド （属性、プロパティまたは変数）で構成されるデータと、任意個数の（メソッドまたは関数）で構成されるコードのひとまとまりで構成される。
>
> オブジェクトの特徴として、オブジェクト自身の手続きが自身のデータフィールドを読み書きできることが挙げられる(オブジェクトにはthisやselfという概念がある)。また、OOPでは、相互に作用するオブジェクトを組み合わせてプログラムを設計する。OOP言語のありかたは多様であるが、最も一般的といえるものは、オブジェクトがクラス(型)のインスタンス(実例)であり、また、オブジェクトの型もクラスとして規定されるクラスベースといわれるものである。

Ruby や JavaScript など、プログラミング言語の多くは、オブジェクト指向 でコーディングできるよう、設計されています。


## クラスの作成とその活用


### 通常の手続き型プログラミングプログラミングを行った場合

それでは、四角形とその面積を求めることを題材に、オブジェクト指向に関する理解を深めて行きましょう。

四角形ですので、幅 `width` と 高さ `height` があります。そして面積 `area`を求める為の関数も用意します。

普通にコードを書くと、おそらく次のようになるでしょう。

```
// 面積を求める関数を定義
function area(width, height) {
  return width * height
}

// 四角形の幅と高さ
let width  = 10
let height = 20

// 幅と高さを表示させる
console.log(width)  // => 10 と幅が表示される
console.log(height) // => 20 と高さが表示される

// 面積を求めたいので、area 関数に幅と高さを渡す
console.log(area(width, height)) // => 200 と面積が表示される

// 高さを半分にして、正方形にしてみる
height /= 2
console.log(height) // => 10 と高さが半分になっている

// もう一度面積を表示させてみる
console.log(area(width, height)) // => 100 と面積が表示される
```

これで無事に面積を求めることが出来ました。


### オブジェクト指向でプログラミングする

しかしながら、面積に加えて周りの長さも算出したい、ブラウザ上で表示させる為に、位置情報や色情報も持たせたいといった欲求にはどのように対応すれば良いでしょうか。

`function length()` と関数を準備する、 `X座標, Y座標` を表す為の変数 `x` や `y` を用意する、色情報の為に `color` という変数を用意し `color = "#ff0000"` として色を指定できるようにするとして、一応の対策は可能です。

加えて、さらに 四角形１、四角形２、四角形３・・・とたくさんの図形がある場合にはどうでしょうか。

沢山の変数や関数の管理が煩雑で、プログラムも作りにくく、ミスも起きやすい状況が容易に想像できます。

もう少し楽にできたらと、先人達の試行錯誤から「オブジェクト指向」という考え方が生まれてきました。

わたくしたちは、すでに「オブジェクト指向」と言う考え方を学習しましたので、これに基づいて実装していきましょう。

一個の四角形が持つ「属性」とそれを「操作する方法（メソッド・機能・関数）」を一まとめにしたもの、それが「オブジェクト（インスタンス・実例とも）」です。

一個の四角形の雛形となるもの、それが「クラス（分類）」です。 [^144]

[^144]: CSSでも分類分けを行う為にクラスが登場しました。「分類分け」を行うという概念は共通しますが、JavaScript など プログラミング言語におけるクラスと、CSS の クラスとは別物です。

雛形、クラスは、よく鯛焼きの型に例えられます。金属で出来た型、それがクラス・雛形です。そしてこの型から生み出されるたくさんの鯛焼き達、それがオブジェクト（インスタンス・実例）です。個々のオブジェクト（鯛焼き）はあんこが入っている鯛焼きもあり、あるいはクリームが入っていたり、チョコレートが入っていたりする鯛焼きがある、そういった具合です。

先に学んだ四角形クラスの例で言えば、四角形には幅と高さがあり、面積を求める演算（メソッド・機能・関数）が定義されている。それが四角形クラスに共通する性質・雛形・クラスです。

四角形クラスから生み出されてくる四角形には、例えば幅が10ピクセルで高さが20ピクセルものもあるでしょうし、幅が30ピクセルで高さも30ピクセルのものもあるでしょう、そして、それぞれ生み出されたオブジェクト（インスタンス・実例）としての個々の四角形は、面積を求めるための `area`と言うメソッド（機能・関数）を持っていますので、ある四角形の面積は200平方ピクセルですし、別の四角形の面積は900平方ピクセルといった具合に、個々のオブジェクトに備わっているメソッドを呼び出すことで、求めることができます。


### 「四角形クラス」を設計する

それでは、四角形クラスを設計してみましょう。四角形クラスを鋳型（いがた）として生成される実例（インスタンス）は、自分自身の情報データ(属性)とその属性の操作方法(メソッド・関数) [^145]を持ちます。

[^145]: メソッドとは、方法、方式、手法、やり方、などの意味を持つ英単語で、一般の外来語としては、一定の形式として確立した奏法、教授法、指導法、その他様々な技法のことを「○○メソッド」のように言います。オブジェクト指向プログラミングでは、データと手続きを「オブジェクト」として一体化（カプセル化）して定義し利用しますが、この、オブジェクトに内包された手続き（データに対する処理内容を記述したプログラム）のことをメソッドと言います。出典: IT用語辞典

一個一個の四角形はどのような属性を持てば良いでしょうか。もちろん幅と高さです。

まとめると次のようになります。


:::{.tip}
**属性 Attribute**

* width 属性: 幅を表す為に
* height 属性: 高さ表す為に
:::

続いて、これらの属性を操作するためにどのような関数を用意すればよいでしょうか。

まず、四角形クラスのインスタンス（実例）を生成することになるわけですが、その際に、四角形のもつ各属性(width, height)の初期値を設定する必要があります。初期値を設定するために、JavaScriptではコンストラクタと呼ばれるメソッド(関数)が用意されています。コンストラクタは日本語では「構築子（こうちくし）」と翻訳されます。

そして、もちろん、四角形の面積を求める為の関数も備えている必要があるでしょう。
---
まとめると次のようになります。


:::{.tip}
**操作 Method**

* constructor() 関数: 各属性の初期値を設定する機能の為に
* area() 関数: 面積を求める機能の為に
:::

以上の考察を元に、四角形クラスを作成して行きましょう。

`JavaScript` も、オブジェクト指向言語ですので、クラス(型・雛型)からインスタンス(実例)を作成し、コーディングすることが出来ます。

四角形クラスを作成し、そのインスタンスを生成、面積を求める例は次のようになります。

```
// 長方形クラス(型)の宣言
class Rectangle {
  // 構築子
  constructor(height, width) {
    this.width  = width  // 幅
    this.height = height // 高さ
  }

  // 面積を求めるメソッド(関数)
  area() {
    return this.width * this.height
  }
}

```

```
// 実際に使ってみる
// 幅10cm, 高さ20cm の長方形のインスタンス(実例)を作成
rect1 = new Rectangle(10, 20)

// 幅と高さを表示させてみる
console.log(rect1.width)  // => 10 と幅が表示される
console.log(rect1.height) // => 20 と高さが表示される

// 面積を求めたいので、area メソッドを呼び出す
console.log(rect1.area()) // => 200 と面積が表示される

// 高さを半分にして、正方形にしてみる
rect1.height /= 2
console.log(rect1.height) // => 10 と高さが半分になっている

// もう一度面積を表示させてみる
console.log(rect1.area()) // => 100 と面積が表示される
```

もし、位置情報や色情報など新しい属性を持たせたい場合には、構築子に追加すれば良いですし、周りの長さを求めるなど、新しいメソッドが必要になった場合も、容易に変更できます。

## JavaScriptとHTMLとの関係


### ウェブブラウザ と `DOM(Document Object Model)`

文書構造の為の `HTML`、装飾・配置の為の `CSS` と並んで、 `JavaScript` は、ウェブサイト作成の為に広く用いられます。

その理由は、**`JavaScript**から`HTML`を制御・操作できる` からです。


`HTML`文書をブラウザで読み込むと、`DOM(Document Object Model)` と呼ばれるプログラミング用のデータ表現が生成されます。`JavaScript` には、この `DOM(Document Object Model)` を操作できる機能が備わっていますので、ブラウザに読み込まれた `HTML`文書の内容(コンテンツ)やその構造を、`JavaScript` によって操作することができます。

例えば、`DOM` には HTML文書そのものを表現する `document` グローバルオブジェクトがあります。 `document` グローバルオブジェクトには、指定した`HTML` 要素を取得したり、新しく `HTML` 要素を作成するためのメソッド(方法・方式・手法、関数)が実装されています。 この `document` グローバルオブジェクトを使うことで、`HTML`に書かれた各要素を `JavaScript` から操作できます。

### (1) DOMから操作したい要素を取得する

DOMから操作したい要素を取得するためには、次の3つのメソッドが用意されています。MDN に詳細が記されています。抜粋しつつ、それぞれのメソッドに関して見ていきましょう。

* `[Document.getElementById()](https://developer.mozilla.org/ja/docs/Web/API/Document/getElementById)`
* `[Document.querySelector()](https://developer.mozilla.org/ja/docs/Web/API/Document/querySelector)`
* `[Document.querySelectorAll()](https://developer.mozilla.org/ja/docs/Web/API/Document/querySelectorAll)`


#### Document.getElementById()

> Document の getElementById() メソッドは、 id プロパティが指定された文字列に一致する要素を表す Element オブジェクトを返します。要素の ID は指定されていれば固有であることが求められているため、特定の要素にすばやくアクセスするには便利な方法です。
>
> ID を持たない要素にアクセスする必要がある場合は、 querySelector() で何らかのセレクタを使用して要素を検索することができます。
>
>
> [tip] <b></a>
>
> **構文**<br>
> `element = document.getElementById(id)`<br>
>
> **引数**`id`<br>
> 探す要素の ID です。 ID は大文字と小文字の区別がある文字列で、文書内で固有です。指定された ID の要素は一つしかありません。<br>
>
> **返値**<br>
> 指定された ID に一致する DOM 要素オブジェクトを記述した Element オブジェクト、または文書内に一致する要素がなければ null です。
>
> [/tip]
> 

「**探す要素の ID**」とありますので、
`document.getElementById()` を使って、DOMツリーから ID が `"html"` の要素 を選択したい場合には、`document.getElementById("html")` と書けば良いことが分かります。

コンソール画面で、`element = document.getElementById("html")` と入力すると、`<li id="html">` と表示されます。表示された `<li id="html">` にマウスカーソルを重ねると、どの部分を選択しているのか、確認できます。

![](dom1a.png)


#### Document.querySelector()

> Document の querySelector() メソッドは、指定されたセレクタまたはセレクタ群に一致する、文書内の**最初の Element** を返します。一致するものが見つからない場合は null を返します。
>
>
> [tip] <b></a>
>
> **構文**<br>
> `element = document.querySelector(selectors)`<br>
>
> **引数** `selectors`<br>
> DOMString で、照合する 1 つ以上のセレクタを設定します。この文字列は妥当な CSS セレクタでなければなりません。<br>
>
> **返値**<br>
> Element オブジェクトで、文書内で指定された CSS セレクタに**最初に一致する要素**を示すオブジェクト、もしくは、一致する要素がない場合は null を返します。

> 指定されたセレクタに一致するすべての要素のリストが必要な場合は、代わりに querySelectorAll() を使用してください。
>
> [/tip]
> 

「`selectors` の文字列は妥当な CSS セレクタでなければなりません。」とありますので、CSSで `<h1>`要素を選択する際に`h1`と書くのと同じように、`document.querySelector()` を使って、DOMツリーから `<h1>`要素 を選択したい場合には、`document.querySelector("h1")` と書けば良いことが分かります。

- `document.querySelector("li")`

- `document.querySelector(".hardware")`

- `document.querySelector("#html")`

#### Document.querySelector**All**()

先に紹介した querySelector() メソッドとよく似ていますが、メソッド名の最後に**All** と付いています。

違いもまたその点に有り、「querySelector() メソッドは、指定されたセレクタまたはセレクタ群に一致する、文書内の**最初の Element** を返します。」とあるように、いくつかの要素が該当した場合には、最初の一つが返ってきます。

ですので、`document.querySelector("li")` と書くと、六つの `<li>` 要素の内、最初の一つを得ることが出来ます。

六つの `<li>` 要素 全てを取得するには、`document.querySelectorAll()` メソッドを使います。

`document.querySelector**All**("li")` と書くと、六つの `<li>` 要素、「**全て**」を取得できます。

![](dom3.png)
コンソール画面で、`elements = document.querySelectorAll("li")` と入力すると、`NodeList(6) [ li.hardware, li.software, li.software, li#html, li#css, li#javascript ]` と表示されます。表示された `li.hardware` や `li.software` などにマウスカーソルを重ねると、どの部分を選択しているのか、確認できます。


`NodeList(6) [ li.hardware, li.software, li.software, li#html, li#css, li#javascript ]` と表示されているように、通常の配列のように扱えます。

ですので、 `elements[0]` と書いて `li.hardware` を、`elements[1]` と書いて (最初の)`li.software` を、扱うことができます。

また、配列のように扱えますので、繰り返しの為の構文「`for...of`」文を用いると、取得した六つの要素を表示できます。

コンソール画面に入力して、確認してみましょう。

```

  console.log(element);

### innerHTML, innerText, textContent の違い

MDN [textContent](https://developer.mozilla.org/ja/docs/Web/API/Node/textContent) に、三つの取得/設定メソッドの違いについて、記載されておりますので、抜粋して掲載いたします。どのメソッドを用いるべきか、使い分けの参考にしてください。


:::{.tip}
* `textContent` は、 `<script>` と `<style>` 要素を含む、すべての要素の中身を取得します。
* `innerText` は「人間が読める」要素のみを示します。


* `textContent` はノード内のすべての要素を返します。
* `innerText` はスタイルを反映し、「非表示」の要素のテキストは返しません。


* `textContent` は テキストの内容を返します。
* `innerHTML` は、その名が示すとおり `HTML` を返します。
:::

## イベントリスナと関数


### イベントリスナ

先程のプログラムでは、利用者(ユーザ)が何もすることなく、勝利メッセージが表示されました。これではゲームらしくありません。「何か操作を行った時」に、「メッセージを表示する」ようにしましょう。

JavaScriptには、**イベントリスナ** と呼ばれる仕組みが有り、「何か操作を行った時」に指定した動作をする機能が備わっています。

それでは、**イベント**、**リスナー** とはなんでしょうか。それぞれの用語の意味を調べてみましょう。

> プログラミングにおけるイベント (英: event) は、プログラム内で発生した動作・出来事、またそれらを表現する信号である。 メッセージあるいはアクション（動作）とも呼ばれる。
> イベントの例としてウェブブラウザにおける「ページが読み込まれた時」、「クリック動作」、「スクロール操作」などさまざまなイベントがある。 [^147]

[^147]: 出典：Wikipedia

> リスナーとは、聞く人、聞き手、聴取者、聴講生、などの意味を持つ英単語。一般の外来語としてはラジオ（局/番組）の聴取者を意味する用法が有名である。
>
> プログラミングの分野では、何らかのきっかけに応じて起動されるよう設定されたサブルーチンや関数、メソッドなどをイベントリスナー（event listener）あるいは単にリスナーという。例えば、「マウスがクリックされると起動するよう指定された関数」のことを「マウスクリックを待ち受けるリスナー」といったように呼ぶ。 [^148]

[^148]: 出典：IT用語辞典

つまり、**イベントリスナー**とは次のようになります。

> 「ページ読み込みやクリック動作など、ウェブページで行われる様々な動作を常時起動し待ち受けて(聴取し続けて)、イベントが起きた時に指定された処理を行う関数のこと」[^149]

この、**イベントリスナー** の仕組みを利用して、「利用者」が「グー」を出したときに、勝ち負けを判定するなど、じゃんけんゲームを作り上げていきます。
