# 条件分岐

:::{.chapter-lead}
プログラムを作っていく中で、条件分岐を行う事は欠かせません。
特定の条件を満たすか否かにより、行う処理を分岐することができます。

ここでは `if（イフ）文` を使った条件分岐について学んでいくとともに、
論理NOT, 論理AND, 論理OR, 比較演算子, 等価演算子, 条件演算子 についても習得します。  
:::



## 単純な条件分岐

条件分岐のために用いられる基本的な構文が `if文`です。
条件式の評価結果が `true（トゥルー）` (真（しん）・成り立つ)であるならば、実行する文が実行されます。

（反対語は `false（フォールス）` (偽（ぎ）・成り立たない)で、後ほど登場します。）


### `if文` の基本形

次は `if文` の基本形です。

**▼if文**

```
if (条件式) {
  実行する文
}
```

**もし、条件式が真(成り立つ)ならば、波括弧{ \**内に書かれた文を実行する } という意味です。

例として、映画館の料金を表示するプログラムを考えてみましょう。

次のコードでは条件式が `true` (真・成り立つ)であるため、 `if文` の中身が実行され、コンソール画面に「大人料金です」と表示されます。

**▼if文の条件式がtrue**

```
// 年齢は20歳
let age = 20

if (age >= 18) {
  console.log("大人料金です")
}
```

コンソール画面で確かめて見ましょう。
少し長い行数ですので、直接入力しても良いですが、エディタに入力し、コンソール画面に貼り付けると、入力ミスもなく安心です。

## 複数の条件分岐


### 複数の条件分岐

複数の条件分岐を書く場合は、 `if文` に続けて `else（エルス） if文` を使うことで書けます。 たとえば、次のように3つに条件分岐するプログラムを書けます。

**▼else if文**

```
// 年齢は10歳
let age = 10

if (age >= 18) {
  console.log("大人料金です")
} else if (age >= 12) {
  console.log("中高生料金です")
} else if (age >= 6) {
  console.log("小学生料金です")
}
```

`if文` と `else if文` では、条件に一致した場合の処理をブロック内に書いていました。 一方で条件に一致しなかった場合の処理は、 `else文` を使うことで書けます。

**▼else if文**

```
// 年齢は5歳
let age = 5

if (age >= 18) {
  console.log("大人料金です")
} else if (age >= 12) {
  console.log("中高生料金です")
} else if (age >= 6) {
  console.log("小学生料金です")
} else {
  console.log("無料です")
}
```


## 比較演算子, 等価演算子

さて、上の例では、年齢が18歳以上であるか、あるいは12歳以上であるかなどを調べていました。この判定に用いた演算子が、比較演算子です。


**比較・等価演算子**

| 　　 | 　　 |
| --- | --- |
| `age < 18` | 18歳未満 |
| `age <= 18` | 18歳以下 |
| `age > 18` | 18歳超 |
| `age >= 18` | 18歳以上 |
| `age === 18` | 18歳である |
| `age !== 18` | 18歳ではない |


## 条件演算子(三項演算子)

とある映画館では、大人料金が1000円、子ども料金が500円だとします。


**料金表**

| 区分 | 料金 |
| --- | --- |
| 大人料金 | 1000円 |
| 子ども料金 | 500円 |
`if文` を使うと、次のように書けます。

```
// 料金の為の変数宣言
let price

age = 20
if (age >= 18) {
  price = 1000
} else {
  price = 500
}
```

これと同等のことを、条件演算子(三項演算子)を使って書くと次のように書けます。

```
price = (age >= 18) ? 1000 : 500
```

`age >= 18` が「真ならば、成り立つならば、`true`ならば」`1000`、そうでなければ、`500` が、`price` に代入されます。

二項演算子 `+` は、二つの項を取りますので、

「1000円 + 500円」

のように、演算子 `+` は一つで、「1000円」と「500円」が二つの項になります。

三項演算子では、演算子は`?` と `:` の二つで、「大人」「1000円」「500円」と三つの項を取ります。

「大人 ? 1000円 : 500円」

と、質問しているような構文です。

```
// 二項演算子
price = 1000 + 500

// 三項演算子
price = 大人 ? 1000 : 500
```

対比が見て取れるかと思います。簡単な条件式であれば、if文よりも短く簡潔に記述することが出来ますので、使って見てください。


## 論理演算子 - 論理NOT, 論理AND, 論理OR

とある映画館では、大人料金が1000円、子ども料金が500円でしたが、女性向けに料金を割り引くこととしました。大人の女性であれば800円にします。


**料金表**

| 区分 | 料金 |
| --- | --- |
| 大人料金 | 1000円 |
| 　女性割引料金 | 800円 |
| 子ども料金 | 500円 |
こういった場合に活躍するのが、「論理演算子」です。`NOT(否定), AND(かつ), OR(または)` の三種類があり、演算子の記号は、`!, &&, ||` です。


**論理演算子**

| 名称 | 意味 | 演算記号 |
| --- | --- | --- |
| `論理NOT(論理否定)` | 否定 | `!(エクスクラメーションマーク一つ)` |
| `論理AND(論理積)` | かつ | `&&(アンパサンド二つ)` |
| `論理OR(論理和)` | または | `||(バーティカルバー二つ)` |


### 論理AND(論理積)演算子

論理AND(論理積)演算子を使い、女性向けの割引料金を設定する `if文` は次のようになります。

```
// 料金の為の変数宣言
let price
// 性別は女性である
let gender = "female"
// 年齢は20歳である
let age = 20

if (age >= 18 && gender === "female") {
  // 18歳以上 かつ 性別が女性
  price = 800
} else if (age >= 18) {
  // 18歳以上
  price = 1000
} else {
  // それ以外
  price = 500
}
```

8行目の `age >= 18 && gender === "female"` に着目してください。
「18歳以上」 かつ 「性別が女性に等しい」ことを表現する為に、論理AND(論理積) `&&(アンパサンド二つ)` を使っています。

また、論理AND(論理積) `&&`  を用いて、範囲を示すことも出来ます。

```
// 18歳以上 かつ 65歳以下
if (18 <= age && age <= 65) {
```

機会があれば使って見てください。


### 論理OR(論理和)演算子

次に、論理和の例もやって見ましょう。

女性向けの割引料金が好評であったので、高齢者向けにも対象を拡大して、
65歳以上であるか、または、女性であれば 特別料金800円とすることにしました。


**料金表**

| 区分 | 料金 |
| --- | --- |
| 大人料金 | 1000円 |
| 　高齢者・女性料金 | 800円 |
| 子ども料金 | 500円 |

```
if (age >= 65 || gender === "female") {
  // 65歳以上 または 性別が女性
  price = 800
} else if (age >= 18) {
  // 18歳以上
  price = 1000
} else {
  // それ以外
  price = 500
}
```

1行目の `age >= 65 || gender === "female"` に着目してください。
「高齢者」 または 「性別が女性に等しい」ことを表現する為に、論理OR(論理和) `||(バーティカルバー二つ)` を使っています。


### 論理NOT(論理否定)演算子

最後に、論理NOT(否定)をご紹介します。

先ほどの映画館の例で見て行きましょう。
特別日には、特別なイベントを行うことにし、その旨をお知らせすることにします。

「特別日」は、毎月一日の映画の日の他、親子連れで来やすいよう、春休み、夏休み、冬休み期間中の日程となります。

「年齢が18歳以上である」のように簡潔な条件であれば、すぐに `age >= 18` とその真偽を確かめる条件式を書くことが出来ますが、「特別日」のような複雑な条件の場合には、(後ほど学びますが)真偽値を返す「関数」を作成するのが一般的です。

関数名は `isSpecialDay` としましょう。

`isSpecialDay` は、本日が特別日の場合には `true`、普通日の場合には `false` を返すことにします。

`true` は、条件式が「真・成立している」ことを示す特別の値です。
`false` は、条件式が「偽・成立していない」ことを示す特別の値です。 [^164]

[^164]: 真偽値(真理値)についてもう少し説明すべきですが、紙幅の都合上、割愛いたします。概要は https://e-words.jp/w/真理値.html にございますので、三項になさってください。

```
if (isSpecialDay()) {
  alert("本日は特別日です。イベントあります。")
}
```

これで、「本日は特別日です。イベントあります。」とのお知らせが表示されるようになります。

`if (age >= 18) {` で18歳以上であるかどうかを判定しましたが、
`if (isSpecialDay()) {` で特別日であるかどうかを判定、 `alert()`関数でお知らせを表示しています。

それでは、普通日の時に、イベントがないことをお知らせするにはどのように書いたら良いでしょうか。

ここで登場するのが、論理NOT(否定)です。論理NOT、論理否定は、「真偽値を反転させる」際に用います。条件式の「真・成立している・true」を「偽・不成立・false」に「偽・不成立・false」を「真・成立している・true」にします。

論理NOTを使うと、先ほどの `isSpecialDay` 関数を用いて、次のように書けます。

```
if (!isSpecialDay()) {
  alert("本日は普通日です。イベントありません。")
}
```

`isSpecialDay` の 先頭に、`!` が付いていることに注意してください。論理否定演算子`!` を `isSpecialDay` に作用させることにより、真偽値を反転させています。

## 入れ子(ネスト)になった `if文`

先ほどの映画館の例で見て行きましょう。再び料金改定を行い、特別日と普通日に分けて料金設定することにしました。


**料金表**

| 日 | 区分 | 料金 |
| --- | --- | --- |
| 特別日 | 大人料金 | 700円 |
|  | 子ども料金 | 300円 |
| 普通日 | 大人料金 | 1000円 |
|  | 子ども料金 | 500円 |
わたくしたちは、すでに論理積演算子を学んでいますので、これを用いて次のように書くことも出来ます。

```
if (特別日 && 大人) {
  price = 700
} else if (特別日 && 子ども) {
  price = 300
} else if (普通日 && 大人) {
  price = 1000
} else if (普通日 && 子ども) {
  price = 500
}
```

そして、ここでは、入れ子(ネスト)になった `if文` をご紹介いたします。

次のように `if文` の中に `if文` を入れることが出来ます。

```
// 入れ子になった if 文
if (特別日) {
  // 特別日の料金処理
  if (大人) {
    price = 700
  } else {
    price = 300
  }
} else {
  // 普通日の料金処理
  if (大人) {
    price = 1000
  } else {
    price = 500
  }
}
```

論理積でまとめる代わりに、特別日、普通日、それぞれの場合で、料金設定の為の処理を記述しています。

具体的なコードにすると以下のようになります。

```
// 入れ子になった if 文
if (isSpecialDay()) {
  // 特別日の料金処理
  if (age >= 18) {
    price = 700
  } else {
    price = 300
  }
} else {
  // 普通日の料金処理
  if (age >= 18) {
    price = 1000
  } else {
    price = 500
  }
}

// 特別日判定関数
// 特別日なら true を
// 普通日なら false を返す
function isSpecialDay() {
  let today = new Date()
  let d = today.getDate()
  if (d === 1) {
    return true
  }
  return false
}
```

分かりやすさに応じて、適宜使い分けしてください。
