## ウェブサイト作成の為の三つの言語

ウェブサイトを作成する為に用いられる言語には、大きく次の三つがあります。

* `HTML(HyperText Markup Language（ハイパーテキストマークアップランゲージ）)` <br>
  　HyperText Markup Language を日本語で表すなら、「ハイパーテキストに目印をつける言語」くらいの意味になります。<br>
  　目印をつける(Markup)というのは、文書の各部分（見出し・段落・表・リストなど)が果たしている役割が分かるようにすることです。そのために専用の「タグ」を使います。 <br>
  　コンピュータに理解できるよう文書構造を定義することが、HTMLの最も重要な役割です。

* `CSS(Cascading Style Sheet（カスケーディング・スタイルシート）)` <br>
  　ウェブサイトの見た目を飾り付け、彩る為の言語です。文字の大きさや色の指定から画像の配置など、様々な装飾を行うことができます。<br>
  　HTMLのタグが親子関係(包含関係)にある場合、多くの設定値は親要素に指定されたものが子要素、孫要素に引き継がれていきます。このように設定値が上から下へ伝播（でんぱ）していく様子を、階段状の瀧を意味する cascade（カスケード）になぞらえて、命名されました。

* `JavaScript（ジャバスクリプト）` <br>
  　ウェブサイトに、「双方向性・相互作用性（インタラクティブ）」を齎（もたら）すために用いられるプログラミング言語です。 <br>
  　例えば、閲覧者の操作に反応して表示が書き換わったり、ページが表示される際に写真などの要素に動きや効果を加えたり、サーバと通信してデータを取得したりするなど、現在のウェブサイトには欠かせないプログラミング言語となっています。

日々更新される各言語の詳細を知悉（ちしつ）することは大変ですが、良く使う基本的なコードのみで十分にウェブサイトを作成することが出来ます。こつこつ学んでいきましょう。


## HTMLの基本

テキストエディタ Windsurf（パルサー） の使い方をご説明する過程で、すでに以下のHTML を作成しています。とても短いコードですが、HTMLの基本が詰まっていますので、ご説明いたします。

**▼index.html**

```
<!DOCTYPE html>
<html>
  <head>
    <meta charset="utf-8">
    <title>HTML学習</title>
  </head>
  <body>
    <h1>初めてのHTML</h1>
    <p>
      HTMLを学習して、素敵なサイトを作れるように 成ります。
    </p>
  </body>
</html>
```


### ファイル名について

ファイル名を、`index.html` としました。
`index`は、「索引」「指針」「指標」などの意味を持つ英単語です。
ウェブサイトにアクセスした際、特にページの指定がされなければ、ブラウザは `index.html`を表示します。例えば、`https://example.com/` にアクセスすると、ブラウザは`https://example.com/index.html`のファイルを表示します。

`.html`は、**拡張子**と呼ばれます。ファイルの種類が `HTML` であることを示しています。
拡張子が `.html`であるファイルを開くと、**OS**により既定のブラウザが開きます。


### DOCTYPE

HTML文書の先頭に書かれている `<!DOCTYPE html>` は、**DOCTYPE宣言**と呼ばれます。その唯一の役割は **HTML5[^107]で書かれている旨をブラウザに伝えることです。今日では無意味な記述ですが、歴史的な経緯により、先頭に書く必要があります。

[^9999]: ウェブの発明者であるバーナーズリー氏が主催するW3Cの規格HTML5と、ブラウザ開発企業であるMozilla, Apple, Operaによって設立された団体WHATWGが推進する規格HTML Living Standard がありましたが、HTML Living Standard に合流する形で、HTML5は廃止されています。


### `<html>` と `<head>` と `<body>` について

`<html>` 全体は、大きく二つの部分 `<head>` と `<body>`  に分かれます。
`<head>`  は、ウェブページに関する設定を書く場所で、以下のことを記述しています。

* 文字コードとして、`utf-8` を使用すること
* 文書の表題が、「HTML学習」であること

---
`<body>` は、コンテンツ(内容・出し物)を書く場所で、以下を記述しています。

* 大見出し(h1)に、「初めてのHTML」
* 段落(p)に、「HTMLを学習して(略)」


[note] <b>文字コードと「文字化け」について</a>

かつてコンピュータが英数字しか扱えなかった時代、アルファベットの「`A`」は十六進数の`41`(十進数の`65`)、「`B`」は`42`と番号を割り振っていました。これを ASCII（アスキー）コードと言います。

その後、日本語を扱えるように拡張され、例えば「`日`」は 十六進数の `93 FA`と番号付けされました。これを Shift JIS（シフトジス）コードと言います。

Shift JIS コードは長らく使われてきましたが、現在ではほぼ世界中の文字を扱えるようになった UTF-8（ユーティーエフエイト） が標準となっており、例えば「`日`」には `E6 97 A5` が割り振られています。

同じ「`日`」という文字ですが、割り当てられている文字コードが違うため、古くからあるウェブサイトでは Shift JIS の `93 FA` が、新しく作られたウェブサイトでは UTF-8 の `E6 97 A5` という文字コードで書かれています。

そのため、古いサイトを閲覧した際などに、正しく文字が読めないことが起こります。これがいわゆる「文字化け」です。

`<meta charset="utf-8">` は、今から作成するサイトが UTF-8 で書かれていることを示すコードで、文字化けを防ぐ大事な役割を担っています。

[/note]

### 要素とは

![](html11.png)

`HTML`では、**コンテンツ(内容)** を **開始タグ** と **終了タグ** で **マークアップ** (印付け)することにより、その種類を明確にします。マークアップした部分は左のような構造になり、全体を要素と呼びます。

この例では、開始タグ`<h1>`と、終了タグ`</h1>`で囲まれた範囲「初めてのHTML」が「大見出し」であることを示しています。

人が見ると、「見出し」が「始めてのHTML」、「本文」が「HTMLを学習して・・・」であることは一目瞭然ですが、コンピュータにとってそれが何であるかは自明ではありません。そこで、`<h1>始めてのHTML</h1>`と書くことで、「始めてのHTML」が「大見出し」であることを「印付け（マークアップ）」します。

それが何であるかを示す適切なタグを使うことが、良いウェブサイト作りの第一歩です。  [^108]

[^108]: 全部で100以上のタグが用意されていますが、良く用いるタグは十数個です。少しずつ学んでいきましょう。


### タグは入れ子にする

![](right_html.png)

複数のHTMLタグでマークアップする場合、開始タグと終了タグは、他のタグの中に完全に入った状態(入れ子)にしましょう。

![](wrong_html.png)

上は正しいHTMLで、下は誤ったHTMLです。タグの対応状態を確認してください。


### 親要素と子要素

![](html14.png)

正しく入れ子の形で記述すると、階層構造が作られます。このとき、上位階層の要素を「親要素」、下位階層の要素を「子要素」と呼びます。

例えば、`<body>` は `<h1>` の親要素、 `<h1>` は `<body>` の子要素です。


### コメント(覚書・説明書き)

![](comment.png)

ソースコードを分かり易くするための覚書、説明書きが出来るよう、**コメント** と呼ばれる構文があります。

作成したウェブサイトを閲覧する際にはないものとして扱われますので、文法上、絶対に書かなければならないものではありませんが、適切なコメントを書くことにより、自らの助けともなります。

また、不要となったコードを一時的に非表示にしたり、参考となるコードを残しておく為にも使うことが出来ます。

`<!--  -->` の間にコメントを書いていきますが、Windsurf など ほとんどのテキストエディタでは、`command + /` (コマンドキーと `/`キーを同時に押す)でコメントにしたり、もう一度押すとコメントを解除できます。エディタの支援機能を活用して、効率的にコーディングしていきましょう。


### 字下げ(インデント)

HTMLでは、空白や改行は意味を成さない為、任意に字下げや改行が行えます。そして、タグの親子関係(入れ子関係)が分かりやすいよう、綺麗に字下げ(インデント)することを心がけましょう。
自分が書きたいことを明確にできるとともに、コードの不具合も発見しやすくなります。

![](good_html1.png)

**好ましい字下げ 1**<br>
まったく字下げをせず一行に書いていますが、短いコードなのでひと目で分かります。


![](good_html2.png)

**好ましい字下げ 2**<br>
タグごとに字下げをした例です。タグの中にタグが配置されているという、抱合関係・親子関係が明確で、これも好ましいコードです。


![](good_html3.png)

**好ましい字下げ 3**<br>
全てのタグを字下げすることにより、行数の増加や、深すぎる字下げが視認性の妨げとなることもあります。`<h1>`タグをひと目で理解できるため字下げせずに書いた例です。これも好ましいコードです。

![](bad_html.png)

**不適切な字下げ**<br>
字下げが滅茶苦茶になっています。これは少し極端な例ですが、慣れないうちはつい目茶苦茶になりがちです。上の三つの例のように見易く綺麗な字下げを心がけましょう。

== HTMLを記述する

作成したいウェブサイトのイメージができたので、実際に HTML を書いて、ウェブサイトを作り始めていきましょう。


### ビューポート

作成した`index.html`を編集し、次のように更新しましょう。

**▼index.html**

```
<!DOCTYPE html>
<html>
  <!-- サイトに関する設定事項を記述します -->
  <head>
    <!-- 文字コードの指定 -->
    <meta charset="utf-8">

    <!-- ページのタイトル -->
    <title>桜吹雪</title>

    <!-- ビューポート(視点)の指定をすると、
         iPhone でも きれいに見せることができる -->
    <meta name="viewport" content="width=device-width">
  </head>

  <!-- 表示したい内容・出し物・コンテンツを記述します -->
  <body>

  </body>
</html>
```

HTMLの文法で解説したように、分かりやすくするために `<!-- -->` で囲まれた「コメント」を入れています。  [^109]

ページのタイトルを、

```
<title>桜吹雪</title>
```

としましたので、ブラウザのタブに「桜吹雪」と表示されます。

[^109]: 作成例は説明の為に多くのコメントを記入していますが、冗長だと感じる方もいらっしゃることでしょう。ご自身の読みやすさに合わせて、適宜、調整されてください。


ここで、着目したいのは13行目の次の記述です。

```
<meta name="viewport" content="width=device-width">
```

先に述べたように、iPhone で見る人が多いので、この短いコードを入れておくことで、後々、それぞれの端末の表示幅に応じた配置ができるようになります。

=== サイト名

HTML は、ウェブサイトの「文書構造」を記述するための言語です。そして、サイトに表示する文字が、「見出し」なのか「本文」なのかなど、それぞれの要素に適した「タグ」を使って「印付け（マークアップ）」を行うことが、肝要でした。

ウェブサイトの完成図を見ると、いくつかの部品に分けられそうです。それぞれの部品に適した「印付け（マークアップ）」を行っていきましょう。

![](header.png)

まずは、サイト名から作っていきましょう。

サイト名を表すための専用のタグはありませんが、`<header>`タグを使って書くのが良さそうです。現在、空っぽになっている `<body>`タグの中に書き加えて、次のようにしましょう。

**▼index.html**

```
<!-- 表示したい内容・出し物・コンテンツを記述します -->
<body>
  <!-- ヘッダー(サイト名) -->
  <header>さくら</header>
</body>
```

`<header>さくら</header>`を全て手入力しても良いですが、Windsurf には コーディングが容易に行えるよう「入力支援機能」が備わっています。

![](shien1.png)

候補が表示されますので、キーボードの下矢印キーを<br>
押して二番目の選択、エンターキーを押します。

![](shien2.png)

すると、`<header>` と `</header>` が自動で入力 <br>
されます。間にカーソルが点滅していますので、<br>
`さくら` と 入力し <header>タグは完成です。


さまざまな入力支援機能を活用して、<br>
効率的にウェブサイト制作を進めていきましょう。

[/note]

### ヒーローイメージ

![](hero.png)

「ヒーローイメージ」とは「サイトを印象づける大きな画像」のことです。

春を彩る美しい桜を詠んだ俳句を紹介するサイトですから、大きな桜の絵を飾ることにしましょう。


桜の画像はどのようにしたら良いでしょうか。自分で写真を撮りに行くのはもちろん良いですし、巻末の参考リンクにも掲載したように、多くのサイトでも豊富な写真やイラストが提供されていますので、そちらを利用するのも便利です。

ここでは、 [ぱくたそ](https://www.pakutaso.com/) より桜の写真を使わせていただくこととしましょう。


それでは、先ほどのヘッダーのあとに続けてコードを書いていきましょう。

**▼index.html**

```
    <!-- ヘッダー(サイト名) -->
    <header>さくら</header>

    <!-- ヒーローイメージ（サイトを印象づける大きな画像） -->
    <figure class="hero">
      <!-- フリー写真 ぱくたそ https://www.pakutaso.com/ より -->
      ![](images/sakura.webp)
    </figure>
  </body>
</html>
```

ここでの要点は、何といっても次の一行です。

**▼index.html**

```
![](images/sakura.webp)
```

`<img>`は、画像を表示するために用意されているタグです。
先に登場した、`<header>`タグと比べると、大きく二つの違いがあります。


[notice] <b><img>と<header>との比較</a>

`<img src ="images/sakura.webp" alt="青空と咲き誇る小彼岸桜">`<br>
　`<header>さくら</header>`

[/notice]


---
* **コンテンツ(内容物)**や **終了タグ`</img>**`がない。 <br>
　`<header>`タグの場合には、印付け(マークアップ)した「さくら」の前後を`<header>` と `</header>` で囲んでいました。<br>
　`<img>`は、画像の表示を目的としているため、**開始タグ**のみで目的を達成できます。そのため`<header>`タグにはあった**コンテンツ(内容物、さくら)**や**終了タグ**`</header>`がありません。<br>
このように、コンテンツを持たない要素のことを**空要素**と言います。

* `src`と`alt`の二つの**属性**を持っている。 <br>
  　**属性**は、**要素への追加情報を指定する**ためのもので、必須の属性と任意の属性があります。<br>
  　`src` 属性は、source(源泉)の略で表示したい画像の所在を与えます。これは必須の属性です。ここでは、`src="images/sakura.webp"`と書くことで、`images`ディレクトリ(フォルダ)の中にある `sakura.webp` という画像を表示することを示しています。 <br>
  　もう一つの、`alt` 属性は、alternative(代替（だいたい）)の略で何らかの理由で画像が見つからないときに表示する代替（だいたい）テキストを指定します。これも(一部の例外はありますが)ほぼ必須の属性です。目の見えない方が利用する音声読み上げソフトも、代替テキストを利用して画像の内容を伝えます。`alt`属性には、その画像に相応しいテキストを記述しましょう。


もう一つのポイントは、この`<img>`タグを囲むように書かれた `<figure>`タグです。

**▼index.html**

```
<figure class="hero">
  (略)
</figre>
```

![](hero-non-css.png)

`<figure>`タグは、その内容物（コンテンツ）が、「図版」であることを示すために用いるタグです。写真の他にも、イラストや図、詩やプログラムコードなど、さまざまなものを含むことができます。

また`<figure>`タグを、単に `<figure>` と書くのではなく `<figure class="hero">` と、**属性(要素への追加情報)**を記述しています。`class`は**属性名**と呼ばれ、`hero`は**属性値**と呼ばれます。属性名`class`は、特に**クラス名**とも呼ばれます。

**クラス名**を付けると「特定の要素を他の要素から区別する、異なる要素を同じものとして取り扱う」ことが容易になります。

例えばこのサイトでは、多くの桜の写真を掲載します。ヒーローイメージの桜に`class="hero"` と `hero`クラスを付与しておくと、多数の写真の中から、ヒーローイメージの写真を区別できるようになります。

さて、結果を確認してみましょう。右のようになります。
完成形のヒーローイメージと比べると、桜の写真が見切れています。
写真が大きく画面幅に全てを収めきれていないためですが、後ほどCSSを用いて見栄え良く出来ますので、今は先へ進むことと致しましょう。


[note] <b>画像形式について</a>

コンピュータで画像を扱う為の様々な規格が考案されてきました。今日、次の三種類の規格は広く使われています。


* JPEG（ジェイペグ） - フルカラー・非可逆圧縮・写真用途
* GIF（ジフ） 　- 256色の図やイラスト・簡易アニメーション用途
* PNG（ピング） - フルカラー・可逆圧縮・半透明化用途


そして、これらの特徴を併せ持つ規格として、次世代画像形式のWebP（ウェッピー） が実用化されています。


* WebP（ウェッピー） - JPEG（ジェイペグ） / GIF（ジフ） / PNG（ピング） を兼ね備える


ファイルサイズも小さく、画像の表示も速くなり、利用者も快適に閲覧できるなどの利点があります。`ImageMagick` [^113]等のツールを導入することで、簡単に画像形式を変換することができますし、また、ネット上でオンラインで変換してくれるサイトもございますので、使って行きましょう。

[/note]


### 次世代画像形式のWebP（ウェッピー）
WebP（ウェッピー）について、より詳しく書かれた記事が、ウェブ制作会社ICS により公開されておりますので、引用してご紹介いたします。 [^114] (他にも HTML / CSS / JavaScriptを中心とした有益な記事が多数掲載されております。是非ご一読下さい。)


#### 次世代画像形式のWebP、そしてAVIFへ

長い間、Webの静止画に関しては「写真のJPEG、ロゴやイラストのGIF、透過画像のPNG」という明確な使い分けが確立されて来ました。WebPはこのすべてを置き換えることができる次世代のフォーマットです。


#### WebPはJPEG/GIF/PNG(APNG)をカバーする魅力的なフォーマット

WebPを使うことで、これまでは用途や画像の特徴ごとに使い分けが必要だったフォーマットの一本化が可能になります。主な特徴を簡単に紹介しましょう。

* 高い圧縮率：同等画質のJPEGと比較して20-30%のサイズ削減（JPEGの置き換え）
* 不可逆圧縮と透過アニメーションの併用（透過アニメーションでも画質を犠牲にしてサイズを削減できる）（GIF/PNGの置き換え）
* 画質劣化のない可逆圧縮もサポート（GIF/PNGの置き換え）

![](webp.png)


#### さらに次世代のフォーマット、AVIFも

* 多様な色空間やサンプリング方式をサポート
* WebPよりもさらに高画質でコンパクト（同じサイズでも画質が高く、JPEG特有のブロックノイズも発生しない）
* Amazon・Netflix・Google・Microsoft・Mozilla等の企業が共同で開発（FacebookやAppleも後から参画）

:::{.tip}
**　　　　　扇 に て 酒 く む か げ や ち る 櫻**
:::

春の桜の下で宴を楽しんでいる様子、そして散る桜にふっと淋しさを覚えるそんな心情が浮かびます。


それでは、ヒーローイメージに続けて、次のようにコードを書きましょう。

**▼index.html**


![](h1-non-css.png)

29行目の `<h1>`は `Heading（ヘディング）(見出し)`に由来するタグで、「大見出し」であることを示しています。 `<h2>`は「中見出し」、`<h3>`は「小見出し」と順に小さくなり、`<h6>`までが用意されています。

`<a>` は、 `anchor（アンカー）` (錨（いかり）)要素で、他の文書へのリンクを示す為に用いられる、ウェブサイトを特徴づけるタグです。
`<a>`タグをクリックすることで、どこへ跳ぶようにしたいのか、リンク先を示す必要があります。その為に `href="https://ja.wikipedia.org/wiki/松尾芭蕉"` と、`href`**属性**に**属性値**としてリンク先のURLを設定しています。
これにより「**松尾芭蕉**」をクリックすると、「**ウィキペディアの松尾芭蕉**に跳ぶ」というリンクが実現できます。

`<a>`タグに続く `<br>`タグは「改行」で、詠まれた俳句は次の行から表示が開始されます。

表示結果のように、「松尾芭蕉」の名前は「青字に下線が引かれた装飾」[^115] がされており、クリックするとリンクの効果によりウィキペディアへ跳ぶことが確認できます。

[^115]: ネット草莽期には様々な表現が考案されました。流行り廃りはありましたが、地の文とリンクとを区別するために、今でも見かけるなじみのある装飾表現です。

### さくらを詠んだ俳句

![](haiku.png)

続いて様々な俳句を紹介しましょう。


「さくらを詠んだ俳句」という見出しと
その下に、

* 桜の写真
* 詠んだ俳句
* 俳人名

が並ぶ構成です。

その生涯を知りたい方のために、ウィキペディアへのリンクも用意しましょう。
また、写真は今回も [ぱくたそ](https://www.pakutaso.com/) より使わせていただきましょう。


仕様は固まりましたので、実装しましょう。どのようなコードを書けば良いでしょうか。

写真の載せ方は、`<figure>`タグや`<img>`タグを用いれば良いと、すでに学習しています。名前やリンクも、`<p>`タグや`<a>`タグを使えば良さそうです。


見出しは、どうすれば良いでしょうか。

キャッチフレーズに 俳聖 松尾芭蕉の名句「扇にて 酒くむかげや ちる櫻」を紹介した際には、大見出しとして `<h1>`タグを使いました。

サイト全体の大見出しとして `<h1>`タグ を使いましたので、「さくらを詠んだ俳句」というこの「節」の見出しには、中見出しとして`<h2>`タグを、それぞれの俳句には、小見出しとして`<h3>`タグを使えば良さそうです。 [^116]


他に必要なタグとして、「写真と俳句、俳人名」が一塊（かたまり）であることを示しておくと都合が良さそうです。このような目的のために、`<article（アーティクル）>`タグが用意されています。「記事」という意味の英単語で、新聞や雑誌、ブログ記事など、それ自身でも独立して再利用できるものに使うことができます。

さらに、この六つの「記事」を束ねた「節」として、`<section（セクション）>` タグが用意されています。書籍や文章の「節」という意味の英単語が由来で、ウェブサイトの区分、区間として用いることができます。


以上を踏まえて、次のようにコードを書きましょう。

[^116]: 以下 `<h4>, <h5>, <h6>` まで用意されていますが、あまり使うことはないでしょう。

**▼index.html**

```

  <p class="code">
    <a href="https://github.com/Atelier-Mirai/sakura_fubuki_trial"
       class="ranbow button">
       <span>
        ソースコードは<br>こちら
      </span>
    </a>
  </p>

  <!-- 俳句の紹介 -->
  <section class="haiku">
    <!-- h2 は 中見出し -->
    <h2>さくらを詠んだ俳句</h2>

    <!-- それぞれの俳句の紹介 -->
    <article>
      <!-- ウィキペディアへのリンク -->
      <a href="https://ja.wikipedia.org/wiki/小林一茶">
        <!-- 桜の写真を載せる -->
        <figure class="rotate-3 frame">
          ![](images/ipponzakura.webp)
        </figure>

      </a>

</section>
```

![](haiku_non_css.png)

`<figure class="rotate-3 frame">` と `<figure>` タグにクラス名を付与しています。後々 CSS を設定することで、写真に枠を付けたり、傾けたり、セピア調やモノクロにするなどの効果を適用する為の準備です。
