## 図・表・ソースコードのナンバリングと相互参照仕様

### 1. 概要

本書籍では、図、表、ソースコード（リスト）といった非テキスト要素に対し、
**「章番号を含む階層型ナンバリング」を自動で付与**します。さらに、執筆者が番号を
手動で管理する手間をなくすため、**「参照ラベル」**を用いた相互参照機能をサポートします。

---

### 2. ナンバリング仕様

#### 2.1. 要素の名称と形式

| 要素の種類       | 推奨名称 | ナンバリング形式          | 例                             |
| :--------------- | :------- | :------------------------ | :----------------------------- |
| 図 (Figure)      | 図       | 図 [章番号]-[連番]        | 図 2-1（第2章の1番目の図）    |
| 表 (Table)       | 表       | 表 [章番号]-[連番]        | 表 3-5（第3章の5番目の表）    |
| ソースコード     | リスト   | リスト [章番号]-[連番]    | リスト 4-2（第4章の2番目のリスト） |

📌 **規則:** 各要素の連番は、章が変わるごとにリセットされます。

#### 2.2. キャプション（見出し）の配置

- **表/リスト:** キャプション（タイトルと番号）は要素の**上部**に配置します。
- **図:** キャプション（タイトルと番号）は要素の**下部**に配置します。

---

### 3. 相互参照（クロスリファレンス）仕様

ビルドシステム（vivlio-starter gem）は、Markdown 内の特定の構文を解析し、
自動で番号を計算・付与し、相互参照リンクを生成します。

#### 3.1. 参照ラベル（ID）の定義

各要素には、一意となる参照ラベルを **キャプション行** に記述します。キャプション行は、
行頭の太字見出しで、末尾に `@ラベルID` を含む形で記述します。

- ラベルIDの書式: 英数字・ハイフン・アンダースコアなどからなる識別子（例: `css-table-rotate`）
- 直後に続くブロック要素の種類から、対象が「リスト/表/図」のどれであるかを自動判定します。
  - 直後がコードブロック → **リスト**
  - 直後がテーブル（`|` で始まる表記） → **表**
  - 直後が画像（`![]()`） → **図**

**記述例（Markdown）**

リスト（CSSコード）の定義例:

```markdown
** 回転テーブルのCSS定義 @css-table-rotate **
```css:table.css
table {
    border-collapse: collapse;
    /* 略 */
}
```
```

表の定義例:

```markdown
** 回転テーブルのプロパティ一覧 @prop-list **
| プロパティ名 | 役割     |
| :----------- | :------- |
| --scale      | 縮尺率   |
| --shift-y    | Y軸シフト |
```

図の定義例:

```markdown
** 回転テーブルのイメージ図 @table-rotate-image **
![回転テーブルのイメージ図](table-rotate.png)
```

#### 3.2. 本文中での参照

本文中で要素を参照する際は、定義した参照ラベルを **`@ラベルID`** という形で記述します。
ビルド時に、ラベルIDに対応する要素の種別・番号を自動的に解決し、
`リスト 4-1` / `表 3-2` / `図 1-5` のような表示に置き換えます。

| 参照したい要素 | 本文中の記述構文     | 変換結果の例 |
| :------------- | :------------------- | :----------- |
| リスト         | `@css-table-rotate`  | `リスト 4-1` |
| 表             | `@prop-list`         | `表 3-2`     |
| 図             | `@table-rotate-image`| `図 1-5`     |

**記述例（Markdown）**

> 「横長表の調整方法については、**@css-table-rotate** を参照してください。
> ここで定義されているカスタムプロパティの一覧は、**@prop-list** にまとめています。
> レイアウト全体のイメージについては、**@table-rotate-image** を参照してください。」

#### 3.3. HTML変換時の動作

ビルドプロセスにおいて、参照構文は以下の要素に変換されます。

- **番号の自動計算と挿入:**
  記述されたラベルに対応する、自動計算された **ナンバリング**（例: `4-1`）に置き換えられます。
- **ハイパーリンクの付与:**
  参照元から参照先の要素へジャンプできる **リンク**（`<a>` タグ）が自動で付与されます。

なお、キャプション行に `@ラベルID` が一切含まれていない場合、そのブロックは
**番号付け・相互参照の対象にはなりません**。見出しとしての装飾（太字キャプション）は適用されますが、
「リスト 4-1」「表 3-2」「図 1-5」のような番号は付与されず、`@id` による参照もできません。

**番号付けを行わない例（CSSの簡単な例示のみ）**:

````markdown
** CSSの例 **
```css
h1 {
  color: red;
}
```
````

#### 3.4. 手動IDと自動IDの使い分け

本仕様では、ラベルIDの付与方法として **手動ID** と **自動ID** の2種類をサポートします。

##### 手動ID（推奨）

- 著者が意味を持ったIDを明示したい場合、キャプション行に `@css-table-rotate` のような形で
  **手動ID** を記述します。
- この場合、ビルドシステムはキャプション行に記述されたIDをそのまま採用し、本文中の
  `@css-table-rotate` などの参照と対応付けます。

##### 自動ID（@auto / @omakase）

- IDを自分で考えたくない場合、キャプション行に特別なキーワードとして `@auto`（または
  `@omakase`）を指定できます。

  ```markdown
  ** 回転テーブルのCSS定義 @auto **
  ```css:table.css
  table {
      border-collapse: collapse;
      /* 略 */
  }
  ```
  ```

- この場合、ビルドシステムは章番号・種別・連番などからなる **機械的で一意なID** を自動生成
  します（例: `list-4-1`, `table-4-2`, `fig-4-3` など）。
- 自動生成されたIDは **事前に推測できることを前提とせず**、後述のID一覧レポートやビルドログ
  から確認する運用を想定します。
- 同じ要素に手動IDと自動IDを同時に指定することはできません。手動ID（`@css-table-rotate` など）
  が記述されている場合は、常に手動IDを優先します。

#### 3.5. ID一覧レポートと確認方法

自動生成されたIDや、全ての図表・リストのラベルIDを著者が確認できるよう、ビルド時に
**ID一覧レポート**を生成します。

- 出力例（イメージ）:

  ```markdown
  # Cross Reference Map

  - 71-install.md
    - @css-table-rotate   (リスト 4-1, manual) 「回転テーブルのCSS定義」
    - @prop-list          (表 4-2, manual)   「回転テーブルのプロパティ一覧」
    - @table-rotate-image (図 4-3, manual)   「回転テーブルのイメージ図」

  - 93-appendix-c.md
    - @list-4-4           (リスト 4-4, auto)  「別のコード例（自動ID）」
  ```

- レポートには、少なくとも以下の情報を含めることを想定します。
  - 章ファイル名（例: `71-install.md`）
  - ラベルID（例: `@css-table-rotate`, `@list-4-4`）
  - 種別・番号（リスト/表/図 + `章番号-連番`）
  - 手動IDか自動IDかの区別（manual/auto）
  - キャプション行の本文（タイトル）

将来的には、CLI から `vs xref:list` のようなコマンドでID一覧を確認できるサブコマンドを
追加することも検討対象とします。

#### 3.6. ラベルIDの一意性と重複検出

ラベルID（`@ラベルID`）は、リスト・表・図といった種別をまたいで、**本書全体で一意であることを
必須条件とします**。

- 同じラベルIDを持つ定義が複数存在する場合（例: リストと図の両方で `@css-table-rotate` を定義
  している場合）、ビルドシステムは **重複エラー** として扱い、ビルドを中断します。
- エラー時には、少なくとも次の情報をメッセージとして出力することを想定します。
  - 重複しているラベルID（例: `@css-table-rotate`）
  - それぞれの定義が記述されている章ファイル名と行番号（例: `contents/71-install.md:123`）

著者は、重複エラーが発生した場合、ラベルIDをリネームすることで解消します。
意味のあるIDを種別ごとに使い分けたい場合は、`@css-table-rotate-css` / `@css-table-rotate-fig` の
ように、ID自体を明示的に区別してください。
