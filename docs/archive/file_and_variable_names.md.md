# 📋 命名規則の好ましい慣習一覧

原則として、ウェブ系のファイル名はケバブ、プログラミング言語の変数名はスネークが多く採用されています。

| 言語       | ファイル名 | 変数名 / メソッド名       | 命名規則の通称         |
| ---------- | ---------- | ------------------------ | ---------------------- |
| HTML       | ケバブ     | N/A                      | about-us.html          |
| CSS        | ケバブ     | ケバブ（クラス / ID 名） | style-guide.css        |
| JavaScript | ケバブ     | キャメル（推奨）         | helper-functions.js    |
| Ruby       | スネーク   | スネーク                 | user_processor.rb      |
| Markdown   | ケバブ     | N/A                      | getting-started.md     |
| JSON       | ケバブ     | キャメル（キー名）       | config-data.json       |
| YAML       | ケバブ     | スネーク（キー名）       | deployment-config.yaml |

## 📝 補足事項

### ファイル名（ケバブ推奨の理由）

- HTML / CSS / Markdown / JSON / YAML などは、URL やファイルパスの慣習に従い、ハイフン（-）で区切るケバブが推奨されます。
- アンダースコア（_）は一部環境で二重エンコードの問題を引き起こす可能性があるため避けられます。

### 変数名・キー名（言語ごとの慣習）

- **CSS（クラス / ID 名）**: CSS セレクタは伝統的にケバブ（例: `.primary-button`）。
- **JavaScript**: 変数名・関数名にはキャメル（例: `getInitialValue`）が一般的。
- **Ruby**: 変数名・メソッド名にはスネーク（例: `user_processor`）を使用。
- **JSON**: JavaScript オブジェクト構造に近いため、キー名はキャメルが主流。
- **YAML**: 設定ファイルとして使われることが多く、Ruby や Python の文化に合わせてキー名はスネークが好まれます。

プロジェクト内で一貫性を保つことが何よりも重要なので、この一覧を参考に、ご自身のプロジェクトのルールを統一されることをお勧めします。
