# Vivlio Starter ロゴの色と意味付け

Vivlio Starter のロゴは緑と青を基調としており、それぞれにブランドコンセプトと機能に沿った役割があります。

## 緑（Green）：成長、開始、自然な流れ

緑色が持つ代表的なイメージ

- 成長・進化: 開発、改善、そしてユーザーやプロジェクトの成長
- スタート・開始: 「Starter」という名称が示す、技術書づくりの第一歩のサポート
- 自然・環境: Markdown から PDF へ自然に移行できる、Web 標準技術（CSS 組版）の活用
- 安心・安全: 信頼性と使いやすさがもたらす安心感

Vivlio Starter における役割

- 緑は「Starter」の名前や Markdown というシンプルな入力形式を象徴します。
- VS ロゴの「V」に緑を配することで、活発さやスタート地点、入力の「始まり」を表現します。

## 青（Blue）：技術、信頼、効率

青色が持つ代表的なイメージ

- 技術・知識: テクノロジー、プログラミング、ソフトウェア開発
- 信頼・安定: 処理の正確性や config/book.yml による一元管理の安定性
- 知性・論理: 体系的な組版、CSS 組版技術の活用
- 効率・スピード: CLI によるワンコマンドビルドがもたらす生産性と高速化

Vivlio Starter における役割

- 青は CLI、自動化、CSS 組版といった「技術的な骨格」や、ブランドへの信頼感を象徴します。
- VS ロゴの「S」に青を配することで、安定性やシステム、PDF 出力という「ゴール」と技術的信頼性を表現します。

## 配色の組み合わせとメッセージ

緑と青の組み合わせは「持続的な成長（緑）と安定した技術（青）」を象徴する強力な配色であり、Vivlio Starter のブランドメッセージを視覚的に強調します。

| 色 | 担当する概念 | VS アイコンの構造と役割 |
| --- | --- | --- |
| 緑 (V の部分) | 成長とスタート、Markdown | 「V」は活発さやスタート地点。Markdown という入力の「始まり」とプロジェクトの成長を表現。 |
| 青 (S の部分) | 技術と信頼、PDF | 「S」は安定性やシステム。PDF という出力のゴールと、CLI や CSS 組版の技術的信頼性を表現。 |

この役割分担により、ロゴを見た人が「シンプルに（緑）、確かな技術で（青）本を作れる」という Vivlio Starter のブランドメッセージを直感的に受け取れるようになります。
