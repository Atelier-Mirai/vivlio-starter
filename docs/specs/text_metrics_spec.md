Vivlio Starter: Text Metrics Specification
1. 目的

本仕様は、執筆中の原稿を定量的に分析し、著者が内容の「難易度」「表現の多様性」「構成のバランス」を客観的に把握・改善するための指標を定義するものである。
2. 集計指標の定義
2.1 基本統計量 (Basic Stats)

最も基礎となる分量の指標。

    文字数 (Character Count): 改行・空白を除く純粋な文字総数。

    行数 (Line Count): 論理的な行数。

2.2 語彙難度 (Lexical Difficulty)

文章の「見た目の硬さ」と「専門性」を測定する。

    漢字比率 (Kanji Ratio): 全文字数に対する漢字（\p{Han}）の割合。

        基準: 25%〜35% が実用書として理想的。

    平均語長 (Average Word Length): 全文字数 ÷ トークン総数（MeCabによる分かち書き後）。

        意味: 熟語や長い専門用語が増えると数値が上昇する。

2.3 語彙多様度 (Vocabulary Diversity)

表現の豊かさ、単調さを測定する。

    TTR (Type-Token Ratio): 異なり語数（語彙のバリエーション）÷ 総語数。

        算出方法: MeCab の base_form（基本形）を用いて算出し、活用による重複を排除する。

        意味: 数値が高いほど語彙が豊富で、低いほど同じ表現が繰り返されていることを示す。

2.4 読解難度スコア (Readability Score)

日本語版 Flesch/Kincaid を応用した独自の「読みやすさ」指標。

    算出式 (プロトタイプ): Score=(平均文長×0.5)+(漢字比率×0.5)+(難読語カウント)

    評価ラベル:

        Easy: 小中学生向け。

        Standard: 一般的なビジネス・実用書。

        Professional: 専門家・技術者向け。

2.5 章・セクション単位の可視化 (Visual Balance)

構成の歪みを可視化する。

    算出単位: Markdown の H1 または H2 単位での文字数。

    表示形式: ターミナル上で、最大分量を 100% とした相対的な水平棒グラフで表示。

3. 実装要件 (Ruby 4.0 Standard)
3.1 解析エンジン

    Mecab: 索引用語抽出と共通の辞書設定を利用し、形態素解析を行う。

    Pattern Matching: case ... in 等を用い、品詞（助詞、名詞、動詞）ごとの比率計算を最適化する。

3.2 実行タイミング

    vs build 実行時のサマリとして出力。

    vs metrics コマンド（単独実行）による詳細レポートの提供。

4. 出力イメージ (CLI Output)

```text
[文章メトリクス: 原稿サンプル]

--- 基本統計 ---
総文字数    : 4,200 文字
総行数      : 120 行

--- 語彙分析 ---
漢字比率    : 32.5% (良好: 標準的な実用書レベル)
語彙多様度  : 0.62  (語彙が豊富です)
読解難易度  : 標準  (想定読者: 一般の読者)

--- 章ごとの分量バランス ---
第1章      [##########          ] 1,200 文字
第2章      [####################] 2,400 文字 (やや長い)
第3章      [#####               ]   600 文字 (加筆が必要かもしれません)
```
