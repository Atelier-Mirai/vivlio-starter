# 用語集バックリンク重複排除 仕様書

## 1. 概要

同一ページ内で同じ用語が複数回出現した場合、**最初の出現のみ**にバックリンクを表示し、重複するページ番号を排除する機能を実装する。

## 2. 現状の問題

### 2.1 現在の動作

```
用語集ページのバックリンク:
  p.i, v, 2, 4, 4, 5, 5, 10, 11, 11, 11, 15
```

- 同じページに同じ用語が複数回出現すると、それぞれにバックリンクが生成される
- 同一ページへの重複リンクが発生する

### 2.2 期待する動作

```
用語集ページのバックリンク:
  p.i, v, 2, 4, 5, 10, 11, 15
```

- 同じページ内で同じ用語が複数回出現しても、バックリンクは各ページにつき1件のみ

### 2.3 概要と技術的制約

#### 2.3.1 問題
- 同じ章内で用語が複数回出現 → バックリンクが `p.4, 4` と重複
- 本文中にも各出現に†が付与される

#### 2.3.2 現状の技術的制約
Vivliostyleの`target-counter`は**レンダリング時**にページ番号を解決するため：
- ❌ HTML生成時点では実際のページ番号が不明
- ❌ 「同一ページ」を100%正確に判断できない
- ✅ 「同一章（ファイル）」の判断は可能

となる。よって、glossary-linkについてCSSで同一ページの重複を排除する方法は不可能。

## 3. 改善方針
1. **解析パス**
  vivliostyle preview でレンダリングすると、次のようなdomが生成されていることが確認できる。
  （[raw.html](./raw.html)より抜粋）

  ```html
  <div data-vivliostyle-page-index="0">
    <a class="glossary-link" href="#gls-ウェブサイト">†</a>
  </div>
  <div data-vivliostyle-page-index="1">
    <a class="glossary-link" href="#gls-ウェブサイト">†</a>
  </div>
  ```

  これを用いることにより、同一ページ内で同じ用語が複数回出現した場合に、重複するバックリンクを排除することが可能になる。

  各用語がどのページに配置されたかのデータを取得する。(説明の都合上、pages.jsonとするが、ymlで描き出しても良いし、内部的にオブジェクトとして保持するだけでも良い)
  pages.json から、各用語の href と、その ID が所属するページ番号を取得する。

2. **HTMLの浄化**
   - 出力された pages.json と元の HTML を突き合わせ、Nokogiri 等を用いて以下の加工を行う。

    用語集エリア: .glossary-backlink の href（リンク先）と、その ID が所属するページ番号をチェックする。

    本文エリア: 本文中のマーク（例：† を含む <span> や <a> 等）も同様にチェックする。

    重複排除ロジック: **「同じページ番号内に、同じ用語への参照が複数ある」**場合、2回目以降の要素を HTML から物理的に削除する。

3. **整形** 削除後のクリーンアップ 要素を削除したことで不自然に残る「カンマ」や「余計な空白」を、正規表現または隣接ノードの操作で綺麗に掃除する。

4. **本番パス** PDFの生成 加工済みの HTML を使用して、最終的な PDF を出力するコマンドを実行する。

## 4. 実装上の注意

- pages.json 内の ID と HTML 内の要素を正確に対応させること。
- ページ番号がローマ数字（i, ii..）や算用数字（1, 2..）のどちらでも対応できるようにすること。
