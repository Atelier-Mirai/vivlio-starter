## 0. 経緯

`@nega`/`@posi`（`@vspace` の後方互換別名）と `@comment`/`@commend`（編集者コメント）を廃止し、残る `@` 記法は **`@vspace` のみ**になった（2026-07-12）。そこで「Objective-C / Java の `@` ディレクティブに倣って、他にあると便利な機能はないか」を検討する。**直前に3つ削ったばかり**なので、安易に増やさず「本当に割に合うものだけ」を見極めるのが本メモの目的。

## 1. `@` が割に合う条件（判断基準）

`:::{.class}`（ブロック装飾）と `{.aki}`／CSS クラス（見た目）で足りるものは `@` にしない、が大原則。`@` が向くのは次の2系統に限る（Objective-C の `@` 類推がそのまま当てはまる）:

- **定数リテラル系**（ObjC の `@"literal"` / `@123` 相当）＝ その場に**値を差し込む**。引数を取らない or 短い。例: `@version`。
- **組版プラグマ系**（`#pragma` / Java アノテーション相当）＝ その一点に**処理系ヒント**を与える。例: `@vspace`, `@pagebreak`。

共通条件は「**インラインの一点**に効く」「Markdown/CSS では素直に書けない」の2つ。これを満たさないものは `:::`／CSS／標準記法へ回す。

## 2. Tier 1 候補（足す価値が高い）

| 記法 | 用途 | なぜ `@` か / 実装感 |
|---|---|---|
| **`@pageref:id`** | ラベル/見出しの**ページ番号**を参照。「インストール（p.200）」 | §5 で深掘り。CSS `target-counter` で軽実装・印刷で確実に効く。**最優先候補** |
| **`@pagebreak:recto`** / `:verso` | 章・部を必ず**奇数ページ（右）始まり**に | `---`（＝ただの改ページ）では表現不可。印刷本の定番。実装は `@vspace` と同型で `<div style="break-before: recto"></div>`（※ `contents/61-developer.md:392` の「旧 `page-break-before` の方が確実」メモは要検証） |
| **`@version` / `@today` / `@title`** | 本文・コラム・奥付に book.yml／ビルド時刻由来の値を差し込む。「本書は v`@version` 時点の情報」 | ObjC の `@"…"` に最も近い“定数”。既存の本文変数展開は無い（`{{PLACEHOLDER}}` は `vs new` 生成専用）ので新規価値。実装は CONFIG から置換するだけで軽い |
| **`@qr:URL`** | サンプルコード/リポジトリへの導線（技術書の定番） | 既存 PLANNED「Web アプリ連携 QR」と一致。実装は SVG 生成ステップが要る（やや重め） |
| **`@hspace:N`** | `@vspace` の水平版（インライン字下げ・全角アキ） | `@vspace` と完全対称・実装も軽い。ただし需要は縦アキより薄いので優先度は中（Tier 1.5） |

## 3. Tier 2 候補（面白いが niche／重い）

- **`@nobr{…}`** — 禁則。数値+単位や固有名を**行またぎさせない**（`<span style="white-space:nowrap">…</span>`）。日本語組版でたまに効く。
- **`@fill`** — リーダー線。「目次 `@fill` 5」→「目次 …… 5」。CSS インラインでは書きづらい一方、実装はやや込み入る。
- **`@index:語{よみ}`** — 手動での索引マーキング（自動検出＋辞書の補完）。既存の索引システムと要すり合わせ。

## 4. `@abbr`（略語展開）の検討 — 「ニーズは○、ただし `@` は×」

**着想**: `@abbr:CERN(欧州原子核研究機構)` で定義し、本文の `@CERN` を「欧州原子核研究機構」に展開する。

**結論を先に**: 素の `@CERN`（`@`＋識別子）は **`@id` 参照と衝突する**ため採用しない。構文的にも概念的にも区別が付かず（解決器は「CERN は略語か、CERN という名のラベル参照か」を辞書引きしないと判定できない）、`@vspace` のような「引数を取る動詞」とも毛色が違う。**略語に `@` は充てない。**

やりたいことは実は2つの別機能に分かれる:

| 方式 | 挙動 | 向き先・課題 |
|---|---|---|
| **(a) 注釈型** `*[CERN]: 欧州原子核研究機構` | 本文は「CERN」のまま、`<abbr title="…">` でツールチップ | **Markdown 標準記法**（markdown-it-abbr 系）。sigil 不要＝**衝突ゼロ**。ただし**PDF ではツールチップが出ない**ので印刷向けには弱い（初出だけ展開するポリシーが要る） |
| **(b) 置換型** 定義した略語を正式名称へ差し込む | `@CERN` → 欧州原子核研究機構 | 着想イメージ。実体は `@version` と同じ「**ユーザー定義の定数**」ファミリー。素の `@名前` ではなく**明示マーカー**（例 `@abbr:CERN` や別 sigil）で `@id` 名前空間と分離が必須。生テキスト全置換は**コード内・部分一致の誤爆**リスク（置換ルールと同じマスキング問題）があり、マーカー必須 |

**さらに重要な論点**: この機能は既存の**用語集**（`config/index_glossary_terms.yml`・用語を定義して巻末一覧）と大きく重なる。新機構を足す前に、**用語集エントリに「略号／正式名称」フィールドを持たせて“用語は一箇所”に集約**できないかを先に検討するのが、本プロジェクトの「単一の真実」志向に合う。

**見立て**: ニーズは本物だが `@名前` の形は不採用。やるなら ①EPUB 重視なら標準 `*[…]:` の `<abbr>`、②印刷で正式名称を出すなら「初出だけ展開」ポリシー付きの**用語集連携**。どちらも `@id` を汚さない。

## 5. `@pageref:id` の深掘り（最優先候補）

現状、著者は「「インストール」の章を参照して下さい」と手書きしている。これを「**「インストール」（p.200）**」へ自動化する。

**実装方針（軽い）**: CSS の `target-counter(url, page)`（Vivliostyle ネイティブ）を使い、重いポスト処理なしで実現できる。

```
@pageref:install
   ↓ 前処理で（@id 参照が使うラベル/見出しアンカーを再利用）
<a href="#install-anchor" class="pageref"></a>
   ↓ CSS（組版時に Vivliostyle がページ番号を注入）
.pageref::after { content: "（p." target-counter(attr(href url), page) "）"; }
```

Step 8 の `/Dests` 解析のような後処理は不要。**既存 `@id` ラベル基盤に乗る**のが利点。

**設計上の勘所（唯一の要検討点）**: **EPUB／Kindle はリフロー型でページが無い**ため「p200」が意味を持たない。必ず優雅に劣化させる:

- **PDF**: 「インストール（p.200）」
- **EPUB/Kindle**: ページ番号を落とし、タイトルのみのリンク「[インストール](#…)」

理想は**タイトル＋ページを一体**で出すこと。よって対象はキャプションラベルだけでなく**見出し／章タイトルのアンカー**も指せる必要がある（章参照が主用途のため）。`@pageref`（ページ番号）と `@titleref`（タイトル文字列）を分けるか、1記法で両方出すかは要設計。

**見立て**: `target-counter` で軽実装・印刷で確実に効く・既存基盤に乗る。設計課題は「リフロー時の劣化」のみで方針は明確。**Tier 1 の筆頭。**

## 6. 作らない方がよい（既存で足りる＝ `@` にしない）

| やりたいこと | 既存の受け皿 |
|---|---|
| 改ページ全般 | `---`（`hr.pagebreak`） |
| 注意・コラム等の囲み | `:::{.notice}` / `:::{.column}` 等 |
| インライン装飾・フォント | `:::{.class}` + CSS / span クラス |
| ルビ | `{漢字|よみ}`（VFM・`vs furigana`） |
| メモ・TODO | HTML コメント `<!-- TODO: … -->`（`@comment` を畳んだ理由そのもの） |

## 7. 未決の論点 / 次アクション

- **どこまで増やすか**の総量感: 直前に3つ削った直後なので、増やすなら「1〜2個を厳選」が健全。乱立は避ける。
- **優先順位（現時点の私見）**: ① `@pageref`（＋タイトル参照）→ ② `@pagebreak:recto` → ③ `@version`/`@today` → ④ `@qr`（効くが重い）→ ⑤ `@hspace`。
- **`@abbr`**: 単独 `@` 記法としては見送り。「用語集への略号フィールド統合」または「標準 `*[…]:` `<abbr>`」として別途検討。
- **共通の実装土台**: 値差し込み系（`@version` 等）と参照系（`@pageref`）はエンジンが違う（前者は置換ルール／後者は `@id` 基盤＋CSS target-counter）。まとめて1記法族として設計するより、**参照系（`@pageref`/`@titleref`）と定数系（`@version` 等）を別トラック**で進める方が素直。
- **着手時**: 選んだ候補ごとに個別の `*-spec.md` を起こし、本メモの該当節を確定仕様へ昇格させる。
