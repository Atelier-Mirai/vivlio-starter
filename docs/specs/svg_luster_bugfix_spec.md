# svg_luster_bugfix_spec.md

## 1. 概要
Vivlio Starter (`vs`) によるPDF生成プロセスにおいて、Chromiumのレンダリング仕様に起因する**Type 3フォントの混入**を完全に排除し、同時に絵文字画像化に伴うデザイン上の視認性不備を解消する。

## 2. 修正が必要な課題と根本原因

### A. CSS擬似要素によるType 3混入
*   **事象**: `Page 13-16` 等で `♣` や `♦` が Type 3 フォントとして検出される。
*   **原因**: `chapter-common.css` 内の `--h3-marker` 等に直接文字が指定されているため。これは `EmojiReplacer` の置換対象（HTMLテキスト）外となり、Chromiumがフォントとして描画を試みる。

### B. 生成順序による置換漏れ
*   **事象**: `Page 2`, `Page 41`（タイトル・奥付）の波ダッシュ（`〜`）が置換されずに残存する。
*   **原因**: フロントページ生成（Step 9）が、既存の波ダッシュ置換処理（Step 5c）より後に実行されているため。

### C. 絵文字WebP化後の視覚的ノイズ
*   **事象**: 差し替えられた絵文字（WebP）の周囲に灰色の枠線がつき、視認性が低下する。
*   **原因**: 汎用的な `img` タグへのボーダー設定が、インラインの絵文字画像にも適用されている。

---

## 3. 実装要件

### 3.1 CSSマーカーの完全画像化
`processor.rb` の `inject_css` を拡張し、文字によるマーカー描画を廃止して WebP 参照へ切り替える。

*   `--h3-marker: "♣";` → `url("stylesheets/twemoji/2663.webp")`
*   `--h4-marker: "♦";` → `url("stylesheets/twemoji/2666.webp")`

### 3.2 パイプラインの再構成
フロントページ生成（Step 9）の直後に、以下の処理を再実行するフックを追加する。
1.  **波ダッシュ（U+301C）の置換**: 技術書として標準的な「〜」への統一。
2.  **Techbook 後処理**: 生成された全てのHTMLに対する画像参照の整合性チェック。

### 3.3 コンテキストに応じたCSS調整
絵文字画像と、著者が執筆した図版（SVG/WebP）を区別してスタイルを適用する。

*   **絵文字画像**: `.vs-emoji` クラスを付与し、`border: none !important;` を適用。
*   **著者執筆画像**: クラスが付与されていない画像については、既存の灰色枠を維持する。

## 4. 開発・デバッグ時の留意事項
*   **Ruby 4.0+ 開発標準スキル**: 処理の追加に伴い、Ruby 4.0+ の構文（パターンマッチング等）を維持し、過剰なクラス設計（Over-engineering）を避けること。詳細は[標準開発スキル](ruby-development-standard.md)を参照。
*   **冪等性の確保**: フロントページに対して複数回の置換処理が走る可能性があるため、二重変換が発生しないロジックとすること。
*  **`lib`変更に伴う`gem`の再インストール**: 現在ローカルで開発中であるが、開発中の`gem`は再インストールすることにより変更が反映される。詳細は[Vivlio Starter 開発ワークフロー](development-workflow.md)を参照。
