- ~~**EPUB の出力（† と索引リンク）が同時ビルドする target に依存する**~~（**2026-06-22 解消 ⑦**）: `pdf`/`print_pdf` を併せてビルドすると Step 8（backlink dedup）が「同一 PDF ページ内の 2 回目以降の重複」（† だけでなく `index-term` の出現アンカーも対象）を削除し、その結果が共有 HTML 経由で EPUB にも反映され、`targets: epub` 単体（Step 8 非実行で全残）と出力が食い違っていた（実測 2026-06-21: 索引リンク 単体 99 → 結合 73）。**解消**: Step 8 直前に章 HTML を退避し（Step 7b・`@pre_dedup_snapshot`）、`run_step_epub` 冒頭で復元してから EPUB を生成（PDF は dedup 済みで生成済み）。`test_epub_consistent_across_single_and_combined` を † を除かず本文完全一致＋† 数一致で検査するよう強化。詳細 `docs/specs/epub-backlink-dedup-isolation-spec.md`。
- **EPUB(Kindle) の表内インライン数式のサイズが不安定**（既知の制限）: 外部 SVG を `<img>` 参照する数式は、Kindle のリフロー（特に表内）で本文フォントに追従せず、極大・極小に振れることがある。`em`→巨大、`px`→拡大非追従の本質的限界で、px 固定で安定化しつつ既知の制限とする（`epub-kindle-target-split-spec.md` §3）。表内に数式を置かない運用回避を推奨（ディスプレイ `$$` は正常）。
- **EPUB(Kindle) のコード行番号と行の対応がずれる/見栄えが悪い**（将来対応）: リフロー型 EPUB ではクリーン EPUB の Prism 行番号が折返し行とずれ、Kindle のテーブル方式では 2 桁行番号が縦に折り返り行間も不均一になる。方式の選択肢（テーブル化／番号非表示等）と現状不具合を `docs/specs/epub-code-line-numbers-spec.md` に整理（将来対応）。
- **EPUB の表紙画像が一回り小さく見える**（軽微・要実機確認）: 表紙埋め込み自体は解消済み。EPUB カバー（1600×2560, 比 1:1.6）と本文 A4（比 1:1.414）のアスペクト比差による見え方で、カバー画像の設計上の比率。実害が無ければクローズ可。

## `rake test:manual`（RC 品質保証・実マニュアルフルビルド）の残課題（2026-06-21 切り分け）
> いずれも `Common.resolve_page_size` の判型修正（同日・奥付前空白ページ B5 問題）とは無関係で、`test:layout` 中断のため従来は到達していなかった既存問題。各失敗を「環境/上流要因・既知課題・フレーキー・設計仕様」に層別した結果を記録する。明確なコード回帰（安全に修正できるもの）は検出されなかった。

- **【環境/上流要因】Type 3 フォントの再発（FT-01 / `manual_build_test.rb`）**: 生成 PDF の約 25 ページに Type 3 フォントが混入。原因は `@vivliostyle/cli` のピンを 2026-06-13（commit `30e2e44c`「RC版へ向け…package.json も最新版に更新」）で **`^10.5.0` → `^11.0.2`**（core 2.41→2.43.2）へ更新したこと。Type 3 解消（`svg_luster_bugfix_technical_notes.md`）は 10.5.0 の Chromium で検証されており、11.0.2 同梱 Chromium が CSS generated content / SVG / 特殊記号を再び Type 3 で出力していると見られる（FT-01 のメッセージ自体が「Vivliostyle/Chromium 更新による再発の可能性」と明記）。なお CHANGELOG では canary で 11.0.1 は Type3 再発なしと記録しており、11.0.2 で再発した形。**対応の選択肢**: (a) `@vivliostyle/cli` を Type3 クリーンな 10.5.0 系へ戻す（最も確実・「最新版へ」の意図は後退）、(b) 11.x 向けに Type3 緩和（WebP 化等）を拡張（重い・不確実）、(c) 上流の追従待ち。印刷入稿品質に関わるためリリース判断時に要決定。
- ~~**【環境/上流要因】ノンブル由来の Helvetica 非埋め込み（FT-02）**~~（**2026-06-23 解消・上記選択肢(a)**）: 隠しノンブルが PDF 標準 14 書体 `Helvetica`（非埋め込み）で描画され、入稿で非埋め込みフォント事故になっていた。**解消**: ノンブル描画を同梱 `stylesheets/fonts/hackgen35/HackGen35ConsoleNF-Regular.ttf` のサブセット埋め込みへ変更。MIT 経路（`StandardProvider`・Prawn）と拡張プラグイン経路（`vivlio-starter-pdf` 1.1.1 / `EnhancedProvider`・HexaPDF `font.map`）の双方を修正。フォント未配置時は `Helvetica` にフォールバック。実ビルド（`targets: print_pdf`）の入稿用 PDF で非埋め込み 0・Type3 0・ノンブル HackGen サブセット埋め込みを確認。回帰テスト: 本体 `nombre_stamper_test`・プラグイン `enhanced_provider_test`。
- **【既知課題】索引ページの epubcheck RSC-012（EP / `epub_validation_test.rb`）**: `_indexpage.xhtml` のページ番号リンクが、本文側に対応するフラグメント（アンカー id）を持たず「フラグメント識別子が定義されていません」を多数出す。索引→EPUB 変換でのアンカー生成が必要で、`epub-kindle-target-split-spec.md` 系で「索引 RSC-012 は全章ビルドでは発生せず」とされていたが、本フルビルドでは発生する（前提の再確認が必要）。用語集 RSC-005 は解消済みだが索引側は未対応。
