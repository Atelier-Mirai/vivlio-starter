# Vivlio Starter 堅牢性テスト仕様書

**対象バージョン:** vivlio-starter 1.0.0-alpha
**作成日:** 2026-04-20
**最終更新:** 2026-04-20（実コード照合版 v2）
**目的:** 通常の `rake test` では検出しにくい、境界条件・異常系・予期しない入力に対するシステムの堅牢性を検証する。本仕様書は、単なる机上のチェックリストではなく、**実コードで既にガードされている項目・そもそも発生しえない項目を除外し**、本当に確認すべき穴を洗い出すことに主眼を置く。

---

## 前提：Vivlio Starter の CLI 設計

読者（テスト実施者）が誤認識を持たないよう、先に重要な設計を明記する。

- `vs build` の引数は **Markdown ファイルパスではなく「章トークン」**（例: `11`, `11-21`, `11-install`）。`TokenResolver`（`lib/vivlio/starter/cli/token_resolver.rb`）でゼロ埋め・範囲展開・カタログ突合が行われる。**ファイルパスを直接指定する API は存在しない**。
- 設定ファイルは `vivlio.config.yml` ではなく **`config/book.yml`** + `config/catalog.yml` + `config/page_presets.yml` + `config/post_replace_list.yml`（`REQUIRED_YAML_FILES`, `common.rb`）。必須ファイルが 1 つでも欠けるとプロジェクト外とみなされる（`version`, `help`, `new`, `doctor` のみ動作可）。
- 原稿は `contents/*.md` 固定配置。`catalog.yml` に列挙された章のみフルビルド対象となり、列挙外は `UNCATALOGED` / `NEW` として扱われる。
- `vs new` は対話プロンプトで書籍情報を収集し、`lib/project_scaffold/` を展開する。プロジェクト名は `VALID_NAME_PATTERN = /\A[a-zA-Z0-9_-]+\z/` でバリデート済み。

## テストの分類と優先度

| 優先度 | 記号 | 意味 |
|--------|------|------|
| 高 | 🔴 | データ破損・クラッシュ・セキュリティリスクに直結 |
| 中 | 🟡 | ユーザー体験を著しく損なう可能性がある |
| 低 | 🟢 | あれば望ましい品質改善 |

**期待される結果の表記**

- ✅ 正常終了（exit 0）
- ⚠️ 適切なエラーメッセージを出して終了（exit 非0）
- ❌ この挙動は不合格（クラッシュ・無限ループ・無言終了・セキュリティ事故）

**既ガード欄の表記**

- 🛡️ 実コードで既にガードされており、テストは現状挙動の回帰確認目的
- 🆕 現状未ガード。テストで実挙動を把握し、必要なら将来対応
- ➖ 該当する API が存在せず、そもそも発生しえない（仕様書から削除）

## 2. Markdown 内容の異常系

### 2-1. Frontmatter

**実コード**: `frontmatter_generator.rb` の `apply_frontmatter` は `rescue StandardError` でパース失敗を捕捉し、`report_frontmatter_error` で警告ログを出してから**元のテキストをそのまま返す**（非 exit）。`find_frontmatter_end` は行単位走査でコードフェンス内の `---` を誤認識しない（設計済み）。

## 改善候補（本仕様書の副産物）

テスト作成中に判明した、**実装改善の候補**を記録する。

1. **SIGINT/SIGTERM トラップ** — 8-1, 8-2, 3-1-8, 5-6-2 共通。`trap(:INT) { cleanup! }` を build/lint の入口に設置し、`.cache/vs/` と一時変換ファイルを消してから exit する。
2. **並列ビルドのロックファイル** — 4-3-1。`.cache/vs/.build.lock` を `File.flock` で取得する簡易実装を検討。
3. **book.yml プレースホルダのエスケープ** — 3-2-1, 3-2-2。ユーザー入力を YAML リテラルに埋め込む際は `.to_yaml` 経由で安全化する。

5. **book.yml 必須キーのバリデーション** — 9-2。`ensure_configured!` で `book.main_title` 等の存在確認を追加。
6. **frontmatter 未クローズ時の明示ログ** — 2-1-1。`apply_frontmatter` 入口で検出しログを出す。
