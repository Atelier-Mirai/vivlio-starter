# 初心者向け 実践プロンプトエンジニアリング技法

### 1. 具体的な条件指定プロンプト
作りたいものの詳細を明確に伝える。
```
HTMLとCSSで、青い背景（#3498db）に白い文字のボタンを作ってください。
- ボタンのテキストは「送信する」
- 角を少し丸くする（border-radius: 8px）
- マウスを乗せると少し色が濃くなる
- クリックできる感じが伝わるよう、カーソルをpointerにする
```

**効果**: 曖昧さがなくなり、期待通りの結果が得られやすい。

### 2. 文脈（コンテキスト）提供プロンプト
背景情報を伝えて、より適切な提案を得る。
```
今、カフェのWebサイトを作っています。落ち着いた雰囲気のデザインで、メインカラーは深緑（#2d5a27）です。

このサイトのヘッダーのCSSを書いてください。ロゴが左、ナビゲーションが右に配置される構成です。
```

**効果**: サイト全体のトーンに合った提案が得られる。

### 3. 役割付与プロンプト
AIに特定の立場や役割を与える。
```
あなたはWeb制作の経験豊富なエンジニアです。初心者にもわかりやすいように、以下のコードを説明してください。
```

**効果**: その視点からの専門的な回答が得られる。

### 4. 出力形式指定プロンプト
コードの出力形式を明確に指定する。
```
HTMLとCSSは別々のコードブロックで出力してください。

コードの各部分に、何をしているか説明するコメントを付けてください。

初心者向けに、コードの後に簡単な解説も付けてください。
```

**効果**: 使いやすく、理解しやすい形式で回答が得られる。

### 5. Few-shot（例示）プロンプト
望む形式の例を見せて、同じ形式で出力してもらう。
```
以下の形式で、HTMLタグの説明を5つ作ってください。

【例】
- `<h1>` … 最も大きな見出し。ページのタイトルに使う。
- `<p>` … 段落。本文のテキストを囲む。

【お願い】
上記の形式に従って、`<a>`, `<img>`, `<ul>`, `<li>`, `<div>` を説明してください。
```

**効果**: 統一された形式で情報が得られる。

### 6. 対話的プロンプティング
段階的に質問を重ねていく方法。
```
【1回目】ナビゲーションメニューのHTMLを作ってください。ホーム、サービス、会社概要、お問い合わせの4項目です。

【2回目】このメニューをCSS Gridで横並びにするCSSを追加してください。

【3回目】マウスを乗せたときに下線が表示されるようにしてください。アニメーションは0.3秒でお願いします。
```

**効果**: 各段階で確認しながら進められる。方向転換も柔軟にできる。

### 7. Chain of Thought（思考過程）プロンプト
ステップバイステップで考えてもらう。
```
レスポンシブ対応の複雑なレイアウトを作りたいです。
ステップバイステップで考えて、順番に実装方法を教えてください。

【要件】
- PCでは3カラム、タブレットでは2カラム、スマホでは1カラム
- サイドバーはPCでは左側、タブレット以下では非表示
- メインコンテンツは常に中央寄せ
```

**効果**: 複雑な問題を段階的に理解できる。

### 8. 参考URL提示プロンプト
参考サイトやドキュメントのURLを提示する。
```
このサイトのようなカード型のレイアウトを作りたいです。
参考: https://example.com/gallery

同じような見た目をCSS Gridで実現するにはどうすればいいですか？
```
```
MDNのCSS Gridのページを参考に、以下のレイアウトを作ってください。
参考: https://developer.mozilla.org/ja/docs/Web/CSS/CSS_grid_layout

3カラムで、中央のカラムだけ幅を広くしたいです。
```

**効果**: 正確な仕様に基づいた回答が得られる。ハルシネーション防止にも有効。

### 9. スクリーンショット活用プロンプト
画像を見せて状況を正確に伝える。
```
添付のスクリーンショットを見てください。ナビゲーションメニューを横並びにしたいのですが、縦に並んでしまっています。どこを直せばいいですか？

（スクリーンショットを添付）
```

**効果**: 言葉より正確に状況を伝えられる。視覚的な問題の解決に強力。

### 10. 否定プロンプト（制約指定）
やってほしくないことを明確に伝える。
```
シンプルなナビゲーションメニューを作ってください。
ただし、以下は**使わないでください**：
- JavaScriptは使わない（CSSのみで実装）
- 外部ライブラリ（Bootstrap等）は使わない
- 複雑なアニメーションは不要
```

**効果**: 期待と違う回答を防げる。

### 11. コード説明要求プロンプト
生成されたコードの説明を求める。
```
さっき作ってもらったコードについて、各部分が何をしているか、初心者にもわかるように詳しく説明してください。
```

**効果**: AIの意図を確認でき、自分の理解も深まる。論理的な矛盾の発見にも役立つ。

### 12. 「なぜ」を問うプロンプト
実装方法の理由を深掘りする。
```
`addEventListener` を使う方法と、HTMLに `onclick` を直接書く方法がありますよね。なぜ `addEventListener` のほうがいいのですか？
```

**効果**: 本質的な理解が得られる。ベストプラクティスの背景を学べる。

### 13. 比較プロンプト
複数の方法を比較してもらう。
```
配列の要素を順番に処理する方法として、`for`文と`forEach`がありますよね。それぞれの違いと、どちらを使うべきか教えてください。
```
```
この機能を実装する方法を3つ提案してください。
それぞれについて：
- 実装の複雑さ
- パフォーマンス
- メンテナンス性
- ブラウザ互換性
の観点から比較してください。
```

**効果**: 選択肢を理解した上で適切な方法を選べる。

### 14. エラーメッセージ提示プロンプト
エラーメッセージをそのまま見せる。
```
以下のエラーが出ました。原因と解決方法を教えてください。

`Uncaught TypeError: Cannot read properties of null (reading 'addEventListener')`
```

**効果**: 的確な原因特定と解決策が得られる。

### 15. 状況説明プロンプト
期待と現実のギャップを明確に伝える。
```
ボタンをクリックしたら、下のテキストが赤くなるようにしたいです。

**期待する動作**: ボタンを押すと、id="message"の文字が赤くなる
**実際の状態**: ボタンを押しても何も起きない。コンソールにエラーは出ていない。

【HTML】
（コードを貼り付け）

【JavaScript】
（コードを貼り付け）

何が問題でしょうか？
```

**効果**: 問題を特定しやすくなる。

### 16. 改善提案要求プロンプト
より良いコードへの改善を求める。
```
このコードは動いていますが、もう少し簡潔に書くことはできますか？

このCSSは同じことを何度も書いている気がします。改善点があれば教えてください。
```

**効果**: コードの質を高められる。自分では気づかない改善点を学べる。

### 17. 代替案要求プロンプト
別の実装方法を探る。
```
このJavaScriptを、別の方法で書くとしたらどうなりますか？

このレイアウトをCSS Gridで作る場合、他にどんな書き方がありますか？
```

**効果**: プログラミングの幅が広がる。

### 18. 仕様改善プロンプト
作成した仕様書のレビューを依頼する。
```
以下の仕様書を確認してください。抜け漏れや改善できる点があれば教えてください。

【お問い合わせフォーム仕様】
- お名前：必須
- メールアドレス：必須
- お問い合わせ内容：必須
- 送信ボタン
```

**効果**: 見落としていた要素に気づける。より完成度の高い仕様書になる。

### 19. ビジネス視点レビュープロンプト
技術面だけでなくビジネス面からの提案を求める。
```
このお問い合わせフォームの仕様について、ビジネス的な観点から見て改善点はありますか？コンバージョン率（問い合わせの送信率）を上げたいです。
```

**効果**: ユーザー体験やビジネス目標を考慮した設計ができる。

### 20. 質問誘導プロンプト
AIに質問してもらう。
```
お問い合わせフォームを作りたいのですが、何を決めればいいかよくわかりません。必要な項目を質問してもらえますか？
```

**効果**: 自分では気づかなかった必要事項が明確になる。

### 21. ハルシネーション防止プロンプト
情報の正確性を確認する。
```
その情報の出典を教えてください。公式ドキュメントのURLがあれば示してください。

この関数は本当に存在しますか？使用例と一緒に説明してください。

自信がない部分があれば、「確実ではない」と教えてください。
```

**効果**: 誤った情報の使用を防げる。

### 22. 練習問題生成プロンプト
学習内容を定着させる。
```
CSSグリッドの練習問題を3問作ってください。回答も一緒にお願いします。

今日学んだ `addEventListener` の理解度を確認するクイズを5問出してください。
```

**効果**: 学んだことを実践で確認できる。

### 23. コードレビュープロンプト
自分で書いたコードの評価を求める。
```
以下のコードをレビューしてください。改善できる点があれば教えてください。

（自分のコードを貼り付け）
```

**効果**: 実践的なフィードバックが得られる。

### 24. 構造化プロンプトテンプレート
情報を整理して伝える。
```
【作りたいもの】○○
【使用技術】HTML / CSS / JavaScript
【見た目】
- 色：
- サイズ：
- 配置：
【動作】
- クリックしたら：
- ホバーしたら：
【その他の条件】
- レスポンシブ対応：必要 / 不要
```

**効果**: 漏れなく情報を伝えられる。AIが理解しやすい。

### 25. 指示を復唱させる（確認プロンプト）
自分の意図が正しく伝わっているか確認する。
```
今から○○を作りたいと思います。
まず、私の依頼内容をあなたの言葉で要約してください。
認識が合っていたら、コードを書いてください。
```

**効果**: 誤解による手戻りを防げる。初心者は自分の説明が曖昧になりがちなので、AIに要約してもらうことで認識のズレに気づける。

### 26. 「まず見本を見せて」プロンプト
いきなり自分のコードを書く前に、お手本を見る。
```
○○を作りたいです。
まず、シンプルな見本コードを見せてください。
その後、私の要件に合わせてカスタマイズしていきます。
```

**効果**: ゼロから作るより、見本を見て理解してから修正するほうが学習しやすい。

### 27. 「一緒に考えて」プロンプト
AIに答えを出させるのではなく、一緒に考えるパートナーとして使う。
```
○○を作りたいのですが、何から始めればいいかわかりません。
まず、どんな要素が必要か、一緒に考えてもらえますか？
質問形式で聞いてください。
```

**効果**: 受け身にならず、自分で考える習慣がつく。

### 28. 「これとあれの違いは？」プロンプト
似たような技術や書き方の違いを理解する。
```
CSSで中央寄せする方法として、
- margin: 0 auto;
- text-align: center;
- display: grid; justify-content: center;
がありますが、それぞれいつ使うのか、具体例で教えてください。
```

**効果**: 「どっちを使えばいいの？」という初心者の疑問を解消できる。

### 29. 「動いているけど、なぜ？」プロンプト
動作しているコードの仕組みを理解する。
```
このコードは動いていますが、なぜ動いているのか説明してください。
特に○○の行が何をしているのかわかりません。
```

**効果**: コピペで終わらず、理解しながら進められる。

### 30. 「失敗例も見せて」プロンプト
よくある間違いを先に知っておく。
```
○○を作るとき、初心者がよくやってしまう間違いを
3つ教えてください。それぞれ、なぜダメなのかも説明してください。
```

**効果**: 同じ失敗を避けられる。エラーが出ても「よくある間違いだ」と冷静に対処できる。

### 31. 「ゼロから順番に」プロンプト
何もない状態から、手順を追って作る。
```
HTMLファイルの新規作成から、○○が動くまで、
1ステップずつ順番に教えてください。
各ステップで、何のファイルに何を書くかも明示してください。
```

**効果**: 「どこに何を書けばいいかわからない」という初心者の不安を解消。

### 32. 「もっと簡単に言うと？」プロンプト
専門用語を平易な言葉に翻訳してもらう。
```
「DOMを操作する」という説明がよくわかりません。
もっと簡単な言葉で、具体例を使って説明してください。
```

**効果**: 専門用語の壁を越えられる。

### 33. 「コメントをたくさん」プロンプト
コードにたくさん説明を入れてもらう。
```
このコードに、各行が何をしているか、
初心者にもわかるように詳しいコメントを追加してください。
「なぜこう書くのか」も説明してください。
```

**効果**: コードを読むだけで学習できる。後で見返したときも理解しやすい。

### 34. 「動かして確認」プロンプト
段階的に動作確認しながら進める。
```
まず、最小限の機能だけで動くコードを作ってください。
それが動いたら、次の機能を追加していきます。
各段階で「ここまでで動くか確認してください」と言ってください。
```

**効果**: どこで問題が起きたか特定しやすい。成功体験を積み重ねられる。

### 35. 「この部分だけ教えて」プロンプト
全体ではなく、わからない部分だけピンポイントで質問する。
```
このコード全体は理解できましたが、
15行目の `addEventListener` の第2引数の
`function() { ... }` の部分だけ、もっと詳しく説明してください。
```

**効果**: 必要な情報だけを効率よく得られる。

### 36. 「作りながら学ぶ」プロンプト
実際に手を動かしながら理解を深める。
```
フォトギャラリーを作りながら、HTMLとCSSを学びたいです。
まず最初に作る部分と、そこで学べることを教えてください。
一つずつ作って、動作確認しながら進めたいです。
```

**効果**: 理論だけでなく、実践を通じて学べる。

### 37. 「見た目で確認」プロンプト
視覚的に確認しながら進める。
```
今こういう見た目になっています（スクリーンショット添付）。
これを○○のようにしたいのですが、
CSSのどこをどう変えればいいですか？
```

**効果**: 言葉で説明するより、画像で見せるほうが正確に伝わる。

### 38. 「お手本サイトを参考に」プロンプト
既存のサイトを参考にして学ぶ。
```
このサイト（URLまたはスクリーンショット）のような
ナビゲーションメニューを作りたいです。
似たような見た目と動きを実現するには、
どんなHTMLとCSSを書けばいいですか？
```

**効果**: 「こういうのを作りたい」が明確に伝わる。

### 39. 「できた！次は？」プロンプト
一つの機能が完成したら、次のステップを提案してもらう。
```
○○が完成しました！
これをもっと良くするために、次に追加すると良い機能や
改善点を3つ提案してください。
難易度も教えてください。
```

**効果**: 自然な流れでスキルアップできる。次に何を学ぶべきかわかる。
