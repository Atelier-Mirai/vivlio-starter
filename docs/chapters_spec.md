# 📖 `book.yml` `chapters` 設定仕様

`chapters` 設定は、ビルド対象とする章（Markdown ファイル）の範囲と順序を定義します。

---

## 1. 📚 基本構造と目的

| 設定項目 | データ型            | 必須/任意 | 目的                                   |
| :------- | :------------------ | :------- | :------------------------------------- |
| `chapters` | String または Array | 必須     | ビルド対象とする章ファイルを指定します |

---

## 2. 📝 章の指定方法（ファイル名ベース）

### 形式 1: 全ての章を対象（デフォルト）

ディレクトリ内のすべての Markdown ファイルを、ファイル名順にビルド対象とします。

```yaml
chapters: all
```

### 形式 2: ファイル名による配列指定（従来の形式）

特定の章ファイルを配列で指定します。拡張子（`.md`）は省略可能です。

```yaml
chapters:
  - 02-preface
  - 11-install
  - 12-tutorial
```

---

## 3. 🔢 章の指定方法（番号ベース）

ファイル名の先頭にある章番号（例: `02-preface.md` の `02`）を利用して指定する形式です。

### 形式 3: 配列による番号指定

章番号を配列形式で指定します。

```yaml
chapters: [02, 11, 12, 91]
```

### 形式 4: カンマ区切りによる番号指定

章番号をカンマ区切りの文字列で指定します。

```yaml
chapters: 02, 11, 12, 91
```

### 形式 5: 範囲指定（ハイフン区切り）

開始章番号と終了章番号をハイフンで区切ることで、連続した範囲を指定できます。他の番号指定と組み合わせることも可能です。

```yaml
chapters: 02-12, 91
```

（※ この場合、`02`～`12` および `91` の章番号に対応するファイルがビルド対象となります。）

**注意**: 番号指定とファイル名指定を混在させることはできません。混在形式を指定した場合はエラーとなります。

---

## 4. 💡 処理ロジックの優先順位

`vs build` の処理は、次のロジックに基づきビルド対象を決定します。

1. **章番号リストの生成**: `chapters` の設定値を解析し、`all`指定、ファイル名、配列・カンマ区切り・範囲指定などを展開して単一の章番号リスト(`keep`)へ変換します。
2. **ファイル検索とソート**: 上記で得た章番号に対応する Markdown ファイルをプロジェクト内で検索し、章番号順に整列させてビルド対象とします。

## 5. 🚨 エラー処理

### 重複章番号

同一章番号で複数のファイルが存在する場合（例: `02-preface.md` と `02-introduction.md`）、エラーメッセージを表示してビルドを中止します。

### 混在形式

番号指定とファイル名指定を混在させた場合（例: `chapters: 02-12, 21-customize, 91`）、エラーメッセージを表示してビルドを中止します。

### 存在しない章番号

指定された章番号に対応するファイルが `contents/` ディレクトリ内に存在しない場合、その番号は自動的にスキップされます（エラーにはなりません）。

## 6. 📌 特別な扱い

### 目次（03-toc）

`03-toc` は目次を表す特別な章番号として扱われ、フルビルド時には常に自動生成されます。

- **自動生成**: ビルドプロセスの Step 6 で、指定された章の見出しを元に目次を自動生成します。
- **章指定での扱い**: `chapters` 設定に `03` を含めても含めなくても、目次は必ず生成されます。`03` を指定した場合は自動的に無視されます。
- **生成内容**: 目次には、`chapters` 設定で指定された章の見出しのみが含まれます。
