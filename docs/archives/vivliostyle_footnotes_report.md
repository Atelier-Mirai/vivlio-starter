# Footnotes（脚注）関連のバグ修正と挙動改善

@vivliostyle/cli は 10.3.1 → 11.0.2に更新されました。今回のメジャーアップデートの裏側（内包するコアエンジン Vivliostyle.js のアップデート）では、日本の書籍組版でも特によく使われ、かつバグの温床になりやすかった脚注（Footnotes）周りの挙動が大幅に強化・修正されています。

具体的には、W3Cの CSS GCPM（CSS Generated Content for Paged Media）仕様への準拠度が上がり、レイアウトのバグが徹底的に潰されました。主な修正・改善内容は以下の5点です。

1. footnote-display の新規サポート（inline / compact）
従来の挙動: 各脚注は必ず改行され、個別のブロック（段落）として縦に並ぶしかありませんでした。

改善後: CSS GCPM 3 の footnote-display: inline; および compact がサポートされました。これにより、短い脚注が複数ある場合に「1行の中に横並びで詰めて配置する」という、新書や学術誌などでよく見られる高度な紙面レイアウトが可能になりました。

2. ページをまたぐ分割（Fragmentation）と領域制御の修正
従来の挙動: 1ページ内に大量の脚注がある、あるいは1つの脚注が非常に長い場合、ページをまたいで脚注を綺麗に分割する（文脈を維持したまま次ページへ送る）挙動が不安定でした。また、脚注エリアの max-height や box-sizing が正しく計算されず、ページ下部に謎の巨大な空白ができたり、本文と重なるバグがありました。

改善後: 脚注エリアのサイズ計算ロジックが修正され、領域が溢れた際のページ分割（フラグメンテーション）の信頼性が大幅に向上。本文の版面設計が崩れるリスクが激減しています。

3. ::footnote-marker の配置（ぶら下げインデント）の修正
従来の挙動: 脚注エリアの先頭につく番号（マーカー）を制御する擬似要素 ::footnote-marker に対して、インデント制御が効きにくい、またはリストアイテム（<li>）内で脚注を呼んだ際にマーカーの描画がバグる問題がありました。

改善後: list-style-position: outside などのプロパティが ::footnote-marker に対しても正しく機能するようになりました。これにより、2行以上にわたる長い脚注がある場合に、2行目以降のテキストを番号の右側に綺麗に揃える（ぶら下げ）制御が標準CSS通りに動くようになっています。

4. セマンティックなマークアップ（DPUB-ARIAロール）へのネイティブ対応
従来の挙動: float: footnote を適用するために、特定のテーマや専用のCSSクラス（例: .footnote）に頼る必要がありました。

改善後: EPUBやアクセシブルなWebドキュメントで使われる role="doc-noteref"（参照元） および role="doc-footnote"（注本文） の属性をコアエンジンが直接認識するようになりました。これにより、テーマ側のCSSに依存せずとも、マークアップ構造だけで自動的に脚注エリアへ配置され、本文側の参照インデックス（::footnote-call）が生成されます。

5. ページスコープによるカウンターリセット（ページ毎の番号振り直し）の安定化
従来の挙動: 脚注番号を「本全体で通し番号」にするのではなく、「ページごとに 1 から振り直す」という処理を行う際、ページ移動や要素の再配置（レイアウト計算）のタイミングでカウンターの数値がズレたり、消失したりする現象がありました。

改善後: ページスコープを跨いだカウンターの利用が最適化され、何度リレイアウトが走っても正確な脚注番号が維持されるよう内部コードが修正されました。
