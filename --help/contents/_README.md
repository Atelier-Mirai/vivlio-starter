# contents/ — 原稿ディレクトリ

書籍の本文となる Markdown ファイルを配置するディレクトリです。

## ファイル命名規則

```
NN-slug.md
```

- `NN` は2桁の章番号（例: `01`, `11`, `90`）
- `slug` は章の識別名（英数字とハイフン）

## 章番号の範囲

| 範囲 | 用途 |
|------|------|
| `00` | 前書き・はじめに |
| `01-89` | 通常の章 |
| `90-98` | 付録 |
| `99` | 後書き・おわりに |

## 章の作成・管理

```bash
vs create 11-intro     # 章ファイルと画像ディレクトリを生成
vs delete 11-intro     # 章を削除
vs rename 11 12        # 章番号を変更
vs renumber            # 章番号を一括で整列
```

## 注意

- このディレクトリの `_README.md`（このファイル）は `vs build` / `vs lint` / `vs metrics` の対象外です
- ビルド対象は `config/catalog.yml` に登録された章のみです
