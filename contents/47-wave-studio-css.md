# Wave-Studio トップページを CSS で彩る

:::{.chapter-lead}
前の章では、小さな練習用の HTML を使って、CSS の文法や「セレクター＋ルール」という考え方をひととおり味わいました。ここからは、いよいよ本編である `wave-studio` のトップページに戻り、前章までに組み上げた `work/index.html` に色と余白、レイアウトをていねいに重ねていきます。

この章のゴールは、ひとつの `style.css` を書き上げて、まずはスマートフォンサイズでトップページが「ひとまず完成です」と胸を張れるところまでたどり着くことです。より広い画面向けのレイアウト調整や、さらに凝った装飾・動きの追加は、続く章にゆずりつつ、まずはモバイルファーストの土台となる CSS を落ち着いた足取りで仕立てていきましょう。
:::

## この章で仕上げる<br>トップページ

:::{.section-lead}
まずは、この章で書き上げる `style.css` によって、`wave-studio` のトップページがどのような姿にまとまるのかを、一枚のスクリーンショットで眺めておきましょう。
:::

:::{.sideimage-right}
この章で書き上げる `style.css` を適用したあとのトップページです。スマートフォンサイズの画面いっぱいに、ヒーローイメージやキャッチコピー、最新記事のカードたちが気持ちよく並ぶところをゴールにしていきます。

![](after_css.webp){width=25%}

モバイルファーストでデザインを組み立てるときは、「小さな画面で読みやすいか」「指先で触れやすいか」を出発点に考えます。Wave-Studio では、まずスマートフォンの縦長のキャンバスの上で、見出し・写真・ボタン・余白のバランスを整え、その上で必要に応じて大きな画面向けの表現を足していきます。

たとえば、次のようなことを心がけています。

- **情報を絞る**  
  ひとつの画面の中に置く要素を欲張りすぎず、「この段では何を伝えたいか」を一つに絞ります。ヒーローイメージの段では写真とキャッチコピーだけ、最新記事の段では記事カードに集中できるようにします。
- **色数を抑える**  
  和色のカスタムプロパティを使いつつも、1 画面の中で主役になる色は 1〜2 色にとどめ、残りは背景やアクセントとしてそっと添える程度にします。そうすることで、リンクやボタンなど「触ってほしい場所」が自然と目に留まります。
- **余白を味方にする**  
  文字サイズや行間だけでなく、ブロック同士の間隔をしっかりとことで、指でタップしやすく、読み進めやすいレイアウトになります。CSS グリッドで行と列を決めておくと、この余白もあとから調整しやすくなります。

:::

前の章で作成した `work/index.html` の骨組みが、色や余白、レイアウトの力を借りてどのように立ち上がっていくのかは、このあとコードとあわせて、ゆっくり追いかけていきます。

## CSSファイルの構造

:::{.section-lead}
最初に、`wave-studio` プロジェクト全体で CSS ファイルがどのように役割分担されているのかを眺めておきます。

いきなり `work/style.css` を書き始めるのではなく、「色の定義」「リセット」「レイアウト」などの役者たちがどこに住んでいるのかを知っておくことで、後からデザインを見直すときの道しるべがぐっと分かりやすくなります。

ここで把握する「CSS ファイルの地図」は、この章で仕上げる `style.css` だけでなく、後の章で Wave-Studio に機能や装飾を足していくときにも、頼りになる手がかりになってくれます。
:::

`index.html` では、

```html
<head>
  ...
  <link rel="stylesheet" href="stylesheets/master.css">
</head>
```

のように、ひとつのスタイルシート `master.css` を読み込んでいます。`master.css` の中では、ページ全体のレイアウトやヘッダー、最新記事一覧、フッターなどを分担して記述した複数の CSS ファイルを `@import` で束ねています。

たとえば `master.css` は、次のように役割ごとのスタイルシートをまとめて読み込むだけの「親ファイル」になっています。

```css
/* 部品別にスタイルシートを記述しておき、
   個々のスタイルシートを読み込むようにすると、
   管理がしやすくお勧めです。 */
@import "_colors.css";  /* 日本の伝統色など、色の定義 */
@import "_reset.css";   /* ブラウザ既定スタイルのリセット */
@import "_body.css";    /* ページ全体の配置 */
@import "_header.css";  /* ヘッダー */
@import "_hero.css";    /* ヒーローイメージ */
/* ...(このあとも、ボタンやフッターなどのCSSを同じように読み込みます) */
```

このように、実際のデザインや細かな指定は `_colors.css` や `_header.css` などの「部品ファイル」に分割し、`master.css` ではそれらを一覧するだけの薄いファイルにしておくと、

- サイト全体の中で、どの役割の CSS がどこにあるかが一目で分かる
- ヘッダーだけ・ボタンだけといった部分的なデザイン変更の際に、該当ファイルだけを開けばよい
- `_colors.css` など、他のサイトでも使い回したい CSS を別プロジェクトからも再利用しやすい

といったメリットがあります。`wave-studio` でも同じ考え方で、ページ全体のスタイルを組み立てています。

実際の Wave-Studio プロジェクトでは、`stylesheets/` フォルダの下に、ここで見たような `_colors.css` や `_header.css` などが並んでいますが、この章で扱う練習用の `work` フォルダでは、まずは学びやすさを優先して、ひとまず 1 本の `style.css` にまとめて書いていきます。CSS の書き方になじんできたら、「色だけ別ファイルに切り出してみる」「ヘッダーまわりだけ `_header.css` に分けてみる」といった具合に、ここで紹介した分割スタイルにも少しずつ挑戦してみてください。

### リセットCSS


ブラウザは、見出しや段落、リストなどがそれなりに読みやすく見えるよう、あらかじめ既定のスタイルを用意してくれています。ただ、そのままではブラウザごとに余白の付き方やインデントの深さが少しずつ異なり、細かなデザインを整えたいときには少々扱いづらく感じられることもあります。

そこで、いったん「真っさらなキャンバス」に戻してから、自分の手で余白や配置を描き込んでいくのがリセットCSSです。

`_reset.css`では次のように書いています。

```css
* {
  margin: 0;  /* すべての要素の外側の余白を 0 にする */
  padding: 0; /* すべての要素の内側の余白を 0 にする */
}
```

ここで使っている `*` はユニバーサルセレクタと呼ばれ、「このページに登場するすべての要素」を指し示す合図です。`html`, `body`, `h1`, `p`, `img`, `ul`, `li` など、ページ内のあらゆる要素に同じルールを届けてくれます。

`margin: 0;` は、要素のまわりにふんわりと置かれていた外側の余白（上下左右の「外側の空き」）をいったん取り払う指定です。ブラウザが親切心から `<body>` の上下に足してくれていた余白や、見出しや段落ごとにまちまちについていた上下の空きも、ここでフラットな状態に戻します。

`padding: 0;` は、枠線の内側から内容までの「内側の余白」を 0 にそろえる指定です。たとえばリストの `ul` / `ol` が標準でもっている左側のインデントも、この指定で一度リセットされます。

こうして外側と内側の余白をいったん 0 にそろえておくことで、ブラウザや環境ごとにばらついていた初期状態から離れ、自分のデザインの都合に合わせて `body` や `h1`, `p` などに必要な余白だけを、改めて丁寧に設計していくことができます。

### box-sizing で幅の計算をそろえる

あわせて、要素の「幅の計算の仕方」を最初にそろえておくと、レイアウトづくりがぐっと扱いやすくなります。`box-sizing` というプロパティでは、「CSS で指定した 幅（`width`） や 高さ（`height`） の中に、内側の余白（`padding`）や枠線（`border`）まで含めて計算するか否か」を選べます。

```css
*, *::before, *::after {
  box-sizing: border-box; /* 幅や高さに padding と border を含めて数える */
}
```

何も指定しないときの既定値は `content-box` で、指定した幅の「外側」に `padding` や `border` が足されていく挙動になります。このままデザインを進めると、「あとから余白や枠線を足したらカードの実寸がどんどん膨らんでしまい、列からはみ出してしまった……」ということが起こりがちで、縦横の寸法を頭の中で引き算しながらレイアウトを考えなければならず、ミスの温床にもなってしまいます。

たとえば、横幅 300px のカードに、左右 16px の余白と 1px の枠線を足したとしましょう。`content-box` の場合は、300 + 16 × 2 + 1 × 2 = 334px までカードの実寸がふくらみ、用意していた列からはみ出してしまうことになります。一方、`border-box` を指定すると、同じ指定でもカードの横幅は 300px のままです。「書いた数値どおりの幅」がそのまま箱の大きさになるので、寸法の見通しが立てやすく、レイアウトの計算もぐっと直感的になります。
そのため、最近のレイアウトでは、最初にページ全体を `border-box` にそろえておくのがお作法のようになっています。

### img で幅を指定する

```css
img {
  width: 100%;
  height: auto;
}
```

ページづくりをしていると、写真はいつも、文章のあいだをそっと橋渡ししてくれる存在です。同じ 1 枚の画像ファイルでも、レイアウトの都合にあわせて「どれくらいの大きさで見せるか」はケースごとに変わってきます。そこでまずは、画像そのものの横幅を「親要素いっぱい」に広げつつ、縦横比だけはきちんと守ってもらう、という約束を先に結んでおきます。

`width: 100%;` は、「この画像が入っている箱（`<figure>` や `<article>` など）の横幅にぴったり合わせてね」という指示です。いっぽう `height: auto;` は、高さをこちらから決め打ちせず、元の写真の比率を保ったまま自動で伸び縮みしてもらう指定になります。この 2 行をそっと添えておくだけで、画面の横幅が変わっても、写真が縦にぐにゃりとつぶれたり、横にびよんと伸びたりせず、きちんと「写真らしい姿」のままレイアウトの中に収まってくれるようになります。

wave-studio のトップページで使っているヒーローイメージや、最新記事カードに並ぶ小さなサムネイルも、すべてこの考え方の上に立っています。まずは「入れ物（親要素）の幅」に素直に寄り添ってもらい、そのうえでグリッドレイアウトや余白の指定を重ねていくことで、どの画面サイズでも破綻しにくい、やわらかな写真まわりのデザインが育っていきます。

### CDN からリセットCSSを読み込む

先人の方々が書いたリセットCSSを読み込む、という選び方もあります。

```html
<!-- Reset CSS -->
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/@acab/reset.css@0.11.0/index.min.css">
```

リセットCSSは、世界中の開発者がそれぞれの美意識や設計哲学にもとづいて公開してくれているため、どこまで強くリセットするか、フォームやタイポグラフィをどこまで面倒みるかといったカバー範囲も少しずつ異なります。プロジェクトの規模や目指したい雰囲気に合わせて「この子と歩んでみよう」と思えるものを選び、必要に応じて自分用に小さな手直しを重ねていくとよいでしょう。そうしたリセットCSSを手早く読み込む手段としてよく使われるのが、CDN（Content Delivery Network：コンテンツデリバリーネットワーク）と呼ばれる仕組みです。世界各地に用意されたサーバーから、閲覧している人に近い拠点を経由してファイルを届けてくれるため、スタイルシートを高速かつ安定して配信してくれます。

ここで例として挙げている `@acab/reset.css` は、最近のブラウザ環境を前提にしたモダンなリセットCSSで、見出しや本文、フォーム要素などの初期状態をそっと整えながら、アクセシビリティにも目配りした、やわらかなリセットです。`margin: 0;` や `padding: 0;`、`box-sizing` の指定など、レイアウトの前提となる土台づくりもひと通り含まれているので、その上に自分のサイトの雰囲気に合わせたスタイルを少しずつ塗り重ねていく、という使い方がしやすくなっています。先ほど見た `img { width: 100%; height: auto; }` と同じ考え方で、画像が親要素からはみ出さず、縦横比を保ったまま収まるような下ごしらえも、あらかじめ済ませてくれているのが心強いところです。

### トップページ全体のレイアウトを整える

CSS グリッドレイアウトを用いて、トップページのレイアウトを整えていきましょう。

前章で作成した `work/index.html` の概要コードです。

```html
<body>
  <header>WAVE</header>
  <nav></nav>
  <main>
    <figure class="hero">
      <img src="images/blue_sea_1920x1280.webp" alt="世界で一番の観光地">
    </figure>
    <h1 class="glow text">Best place to visit in the world</h1>
    <p class="sub_title">
      <img src="images/circle.svg" alt=""><br>
      <a href="#">爽やかな青い空と海</a>
    </p>
    <section>
      <h2>最新記事</h2>
    </section>
  </main>
  <footer>(有) WAVE商会</footer>
</body>
```

:::{.sideimage-right}
![](before_css.webp)

このコードをブラウザで開くと右側の画像のように表示されます。

ここまでの章では、こうした要素たちを「縦にブロックを積み重ねていく」イメージで並べてきました。ここからは少し発想を変えて、真っ白なキャンバスに縦横の補助線をすっと引き、そのマス目の上にヘッダーやヒーローイメージ、キャッチコピーやサブタイトル、記事一覧、フッターを載せていくつもりでレイアウトを考えてみましょう。これを手助けしてくれるのが **CSS グリッドレイアウト** です。

ページを真上から見下ろしたときに、「一番上の段にはサイト名、その下の大きな段には写真とキャッチフレーズ、さらにその下に最新記事、そのいちばん下にフッター」といった具合に、行と列の区画を先に決めてしまう──CSS グリッドは、そんな誌面づくりに似た考え方で配置を組み立てられる仕組みです。

:::

`work/style.css` では、ページ全体を囲む `<body>` 自体をグリッドコンテナにして、ヘッダー・メイン・フッターの置き場所を次のように指定しています。

```css
/* ページ全体の設定 */
body {
  display: grid;
  grid-template:
    ".    head .   " 100px
    "main main main" auto
    "foot foot foot" 140px /
    20px  1fr  20px;
  row-gap: 20px;
}
```

`display: grid;` で `<body>` を「グリッドコンテナ」に変え、その中に並ぶ要素たち（`header`, `main`, `footer` など）をマス目の上に載せていきます。`grid-template:` の 3 行は、それぞれ 1 行分のレイアウトを表していて、`"head"` や `"main"`, `"foot"` といったラベルで「この行のどの列に、どの役割のブロックを置くか」を描き分けています。行末の `100px` や `auto`, `140px` は行の高さ、最後の `/` 以降の `20px  1fr  20px` は左・中央・右の 3 列の幅を表しています。

ここでは、まだ細かな書式をすべて暗記する必要はありません。「ページ全体に、上から順にヘッダーの段・メインコンテンツの段・フッターの段という 3 行を作り、左右には 20px の余白をあけて中央に本文を通しているのだな」というイメージだけつかんでおけば十分です。のちほど、`header`, `main`, `footer` 側のスタイルとあわせて眺めることで、「どのブロックがグリッドのどのマス目に立っているか」が、少しずつ輪郭を帯びて見えてきます。

### CSS グリッドで「main」を並べる

`<body>` 全体の上に大きなグリッドを敷いたら、その中にある `<main>` の中身も、もう一段だけ細かいマス目に分けてあげましょう。`work/style.css` では、ヒーローイメージとキャッチコピー、サブタイトル、最新記事一覧を次のように並べています。

```css
body main {
  grid-area: main;
  display: grid;
  grid-template:
    ".    title  .   " auto
    ".    sub    .   " auto
    ".    recent .   " auto /
    20px  1fr    20px;
  row-gap: 20px;
}
```

`grid-area: main;` で、この `<main>` が先ほどの `body` のグリッドのうち、`"main main main"` と書かれていた 2 行目のマス目一帯にすっぽり収まることを示しています。そのうえで、`display: grid;` を指定することで、今度は `<main>` 自身が小さなグリッドコンテナとしてはたらき始めます。

`grid-template:` の 3 行は、ヒーローイメージのキャッチコピー用の行（`title`）、その下のサブタイトル用の行（`sub`）、最新記事一覧を置く行（`recent`）を表しています。左右にはそれぞれ 20px の余白列をとり、中央の `1fr` の列にタイトルや記事一覧を通している、という設計です。`row-gap: 20px;` は、これら 3 つの行どうしのあいだに 20px の縦方向の余白をあける指定です。

このように、**グリッドは入れ子（ネスト）にして使うことができます**。ページ全体を見下ろす大きなグリッドで「ヘッダー・メイン・フッター」の段取りを決め、その中の `<main>` や `#recent` などでもう一段細かいグリッドを敷くことで、「海辺の誌面」のレイアウトを、無理なく段階的に組み立てていくことができます。

### ヘッダーからフッターまで、マス目の上に立たせる

ここまでで、ページ全体の大きなグリッドと、その中にある `<main>` の小さなグリッドという、二重のキャンバスが用意できました。あとは、このマス目の上に「ヘッダー」「ナビゲーション」「ヒーローイメージ」「キャッチコピー」「サブタイトル」「最新記事」「フッター」という登場人物たちを、ひとりずつ立たせていきます。
 
```css
/* ヘッダー */
header  {
  grid-area: head;
  justify-self: center;
  align-self: center;
}
```

`header` には `grid-area: head;` が指定されています。`body` のグリッドで `"head"` と名付けた 1 行目のマス目の上に、サイト名がそっと腰掛けるイメージです。`justify-self: center;` と `align-self: center;` によって、そのマス目の中央にふわりと配置され、ページの一番上で穏やかに訪問者を迎えます。

```css
/* ナビゲーションメニュー */
nav {
  grid-area: head;
  justify-self: center;
  align-self: end;
}
```

`nav` も同じく `grid-area: head;` を持っており、ヘッダーと同じ行を分け合っています。こちらは `align-self: end;` によって、ヘッダー行の下寄りに寄り添うように立ち、メニュー項目がヘッダーのすぐ下に並ぶような配置になります。ひとつの行のなかで、ロゴとナビゲーションが上下に静かに住み分けている、と捉えると雰囲気がつかみやすいでしょう。

```css
/* ヒーローイメージ */
figure.hero {
  grid-column: 1 / -1;
  grid-row: 1 / span 2;
}
```

`figure.hero` は、ヒーローイメージ（大きな写真）を表すブロックです。`grid-column: 1 / -1;` で「1 本目の列線から、いちばん右端の列線（`-1`）まで」横いっぱいに広がり、`grid-row: 1 / span 2;` で「1 行目から数えて 2 行ぶんの高さ」を占めています。メインコンテンツの最上段から 2 段目にかけて、海辺の写真がどん、と背景のように横たわり、その上にキャッチコピーやサブタイトルが乗る、誌面らしい構図になります。

```css
/* タイトル（キャッチフレーズ） */
h1.catch_phrase {
  grid-area: title;
  justify-self: center;
  align-self: center;
}
```

`h1.catch_phrase` は、`body main` のグリッドの中で `grid-area: title;` を与えられています。つまり、「タイトル行」として用意したマス目の中央に、キャッチコピーがすっと立っている状態です。`justify-self: center;` と `align-self: center;` のおかげで、ヒーローイメージの中央付近に、ポスターの文字のように印象的なフレーズが浮かび上がります。

```css
/* サブタイトル */
p.sub_title {
  grid-area: sub;
  text-align: center;
}
```

`p.sub_title` は、そのすぐ下の行、`grid-area: sub;` のマス目に配置されています。ここには短い説明文や導入の一行が入ります。`text-align: center;` によって行全体を中央寄せにしつつ、ヒーローイメージとキャッチコピーを受け止めるように、やわらかな一文が横に広がります。

```css
/* 記事一覧 */
section#recent {
  grid-area: recent;
  display: grid;
  grid-template-columns: 1fr 1fr;
  gap: 20px 10px;
}
```

`section#recent` は、最新記事の一覧を包むセクションです。`grid-area: recent;` で `<main>` のグリッドにおける「記事一覧の行」に腰を下ろしつつ、自身の中でも `display: grid;` と `grid-template-columns: 1fr 1fr;` を指定して、2 列組の小さなグリッドをもう一段用意しています。`gap: 20px 10px;` は、上下方向に 20px、左右方向に 10px の余白を取り、カードどうしが窮屈にならないように間をあけています。雑誌の紙面で、記事カードが等間隔に並んでいる様子を思い浮かべるとよいでしょう。

```css
/* フッター */
footer {
  grid-area: foot;
  justify-self: center;
  align-self: center;
}
```

最後の `footer` は、`grid-area: foot;` によって、`body` のグリッドでいちばん下に用意したフッター行にそっと収まります。ここでも `justify-self: center;` と `align-self: center;` を指定することで、ページの最下段の中央に、波打ち際の住所や連絡先が静かに並ぶかたちになります。

こうして眺めてみると、CSS グリッドは「どの段・どの列に、どのブロックを立たせるか」を静かに語る設計図のような役割を果たしていて、`grid-area` や `grid-column` / `grid-row` の指定をたどるだけで、コードの上から誌面のレイアウトを頭の中に思い浮かべられるようになるでしょう。

## コラム：海辺の情報誌レイアウトを覗いてみよう

この章では、`index.html` の「最新記事」まで CSS で彩り、グリッドレイアウトで誌面らしく並べるところまでたどり着きました。

同じ CSS グリッドを少しだけ発展させると、`wave-studio/sample01/index.html` のような、海辺のライフスタイル誌をイメージした紙面レイアウトも作れます。

`sample01` では、タイトルやリード文、メインビジュアル、途中の見出しや本文、SNS のリンクなどを、紙の雑誌の 1 ページをイメージしながらグリッドの上に配置しています。どのブロックがどの列・どの行に載っているか、余白の取り方や行間の詰め方などを眺めてみると、「CSS グリッドで誌面をデザインする」という感覚が少しずつつかめてくるはずです。

HTML 側を少しだけ覗いてみると、見た目には現れない「伝わりやすさ」の工夫として `aria` 属性も使われています。たとえば、ページ全体を囲む `<main>` 要素に `aria-labelledby="post-title"` を付けて見出しと結びつけたり、SNS ボタンのリンクに `aria-label="X（旧Twitter）でシェアする"` のようなラベルを付けて、「これは何のボタンか」を画面読み上げソフトに伝えています。時計アイコンのような純粋な飾りには `aria-hidden="true"` を付けて、読み上げ対象から外す、といった工夫もしています。

CSS 側では、この章のコード例では分かりやすさを優先して長さの単位を px に揃えましたが、`sample01` の `style.css` ではフォントサイズなどに `rem` という単位も登場します。ざっくり言うと、`px` は画面上の絶対的なピクセル数、`em` や `rem` はフォントサイズを基準にした相対的な長さです（`1rem` はページ全体の標準フォントサイズ、`1em` はその要素自身のフォントサイズ）。本文では詳しくは立ち入りませんが、「文字に合わせて自動的にバランスが変わる柔らかい単位」と覚えておくと、あとで `sample01` の CSS を読むときに腑に落ちやすくなるはずです。

さらに `sample01` の `style.css` では、最近のブラウザで使えるようになってきた **ネストした CSS** の書き方も使っています。

```css
.sns {
  grid-area: 9 / 2;

  ul {
    display: grid;

    a {
      border-radius: 50%;
    }
  }
}
```

この書き方は、従来の CSS に書き下すと次のようなイメージになります。

```css
.sns {
  grid-area: 9 / 2;
}

.sns ul {
  display: grid;
}

.sns ul a {
  border-radius: 50%;
}
```

親セレクタの中に子要素用のスタイルをインデントしてまとめておくことで、「この `ul` や `a` は `.sns` 内の要素だな」とひと目で分かるようにしつつ、最終的にはブラウザが従来のセレクタに展開して解釈してくれます。

ネスト CSS では、`&` という記号で「いまネストしている元のセレクタ」を参照することもできます。たとえば、次のような書き方です。

```css
.sns {
  a {
    &.is-active {
      /* .sns a.is-active に展開される */
    }

    &:hover {
      /* .sns a:hover に展開される */
    }
  }
}
```

このように、`.sns a.is-active` や `.sns a:hover` のような「元のセレクタ＋クラス名／擬似クラス」を、読みやすい形でまとめて書けます。本書ではここまで詳しくは踏み込みませんが、「`&` はいまのセレクタを差し込む記号」とだけ覚えておくと、後で `sample01` の CSS を眺めるときに役立ちます。

ネストした CSS は、比較的新しい仕様です。現在の主要ブラウザでは動きますが、かなり古いブラウザでは無視されてしまうことがあります。その場合は、ここで紹介したような「従来の書き下し版」のセレクタで書いておくと安全です。本書の `sample01` は、ある程度新しいブラウザを前提にした「モダンな書き方の例」として掲載しています。

本書では `sample01` のコード全文は追いませんが、`index.html` で学んだことを応用すると「ここにも grid が使われていそうだな」「ここで `aria` や `rem`/`em`、ネストした CSS が使われているな」というポイントがいくつも見えてきます。時間に余裕があるときに、海辺の情報誌をパラパラめくるような気持ちで眺めてみてください。
