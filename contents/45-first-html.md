## ウェブサイト作成の為の三つの言語

ウェブサイトを作成する為に用いられる言語には、大きく次の三つがあります。

* `HTML(HyperText Markup Language（ハイパーテキストマークアップランゲージ）)` <br>
  　HyperText Markup Language を日本語で表すなら、「ハイパーテキストに目印をつける言語」くらいの意味になります。<br>
  　目印をつける(Markup)というのは、文書の各部分（見出し・段落・表・リストなど)が果たしている役割が分かるようにすることです。そのために専用の「タグ」を使います。 <br>
  　コンピュータに理解できるよう文書構造を定義することが、HTMLの最も重要な役割です。

* `CSS(Cascading Style Sheet（カスケーディング・スタイルシート）)` <br>
  　ウェブサイトの見た目を飾り付け、彩る為の言語です。文字の大きさや色の指定から画像の配置など、様々な装飾を行うことができます。<br>
  　HTMLのタグが親子関係(包含関係)にある場合、多くの設定値は親要素に指定されたものが子要素、孫要素に引き継がれていきます。このように設定値が上から下へ伝播（でんぱ）していく様子を、階段状の瀧を意味する cascade（カスケード）になぞらえて、命名されました。

* `JavaScript（ジャバスクリプト）` <br>
  　ウェブサイトに、「双方向性・相互作用性（インタラクティブ）」を齎（もたら）すために用いられるプログラミング言語です。 <br>
  　例えば、閲覧者の操作に反応して表示が書き換わったり、ページが表示される際に写真などの要素に動きや効果を加えたり、サーバと通信してデータを取得したりするなど、現在のウェブサイトには欠かせないプログラミング言語となっています。

日々更新される各言語の詳細を知悉（ちしつ）することは大変ですが、良く使う基本的なコードのみで十分にウェブサイトを作成することが出来ます。こつこつ学んでいきましょう。


## HTMLの基本

テキストエディタ Windsurf（ウィンドサーフ） の使い方をご説明する過程で、すでに以下のHTML を作成しています。とても短いコードですが、HTMLの基本が詰まっていますので、ご説明いたします。

**▼index.html**

```html
<!DOCTYPE html>
<html>
  <head>
    <meta charset="utf-8">
    <title>HTML学習</title>
  </head>
  <body>
    <h1>初めてのHTML</h1>
    <p>
      HTMLを学習して、素敵なサイトを作れるように 成ります。
    </p>
  </body>
</html>
```


### ファイル名について

ファイル名を、`index.html` としました。
`index`は、「索引」「指針」「指標」などの意味を持つ英単語です。
ウェブサイトにアクセスした際、特にページの指定がされなければ、ブラウザは `index.html`を表示します。例えば、`https://example.com/` にアクセスすると、ブラウザは`https://example.com/index.html`のファイルを表示します。

`.html`は、**拡張子**と呼ばれます。ファイルの種類が `HTML` であることを示しています。
拡張子が `.html`であるファイルを開くと、**OS**により既定のブラウザが開きます。


### DOCTYPE

HTML文書の先頭に書かれている `<!DOCTYPE html>` は、**DOCTYPE宣言**と呼ばれます。その唯一の役割は **HTML5[^107]で書かれている旨をブラウザに伝えることです。今日では無意味な記述ですが、歴史的な経緯により、先頭に書く必要があります。

[^9999]: ウェブの発明者であるバーナーズリー氏が主催するW3Cの規格HTML5と、ブラウザ開発企業であるMozilla, Apple, Operaによって設立された団体WHATWGが推進する規格HTML Living Standard がありましたが、HTML Living Standard に合流する形で、HTML5は廃止されています。


### `<html>` と `<head>` と `<body>` について

`<html>` 全体は、大きく二つの部分 `<head>` と `<body>`  に分かれます。
`<head>`  は、ウェブページに関する設定を書く場所で、以下のことを記述しています。

* 文字コードとして、`utf-8` を使用すること
* 文書の表題が、「HTML学習」であること

---
`<body>` は、コンテンツ(内容・出し物)を書く場所で、以下を記述しています。

* 大見出し(h1)に、「初めてのHTML」
* 段落(p)に、「HTMLを学習して(略)」


:::{.note}
**文字コードと「文字化け」について**

かつてコンピュータが英数字しか扱えなかった時代、アルファベットの「`A`」は十六進数の`41`(十進数の`65`)、「`B`」は`42`と番号を割り振っていました。これを ASCII（アスキー）コードと言います。

その後、日本語を扱えるように拡張され、例えば「`日`」は 十六進数の `93 FA`と番号付けされました。これを Shift JIS（シフトジス）コードと言います。

Shift JIS コードは長らく使われてきましたが、現在ではほぼ世界中の文字を扱えるようになった UTF-8（ユーティーエフエイト） が標準となっており、例えば「`日`」には `E6 97 A5` が割り振られています。

同じ「`日`」という文字ですが、割り当てられている文字コードが違うため、古くからあるウェブサイトでは Shift JIS の `93 FA` が、新しく作られたウェブサイトでは UTF-8 の `E6 97 A5` という文字コードで書かれています。

そのため、古いサイトを閲覧した際などに、正しく文字が読めないことが起こります。これがいわゆる「文字化け」です。

`<meta charset="utf-8">` は、今から作成するサイトが UTF-8 で書かれていることを示すコードで、文字化けを防ぐ大事な役割を担っています。

:::

### 要素とは

![](html11.png)

`HTML`では、**コンテンツ(内容)** を **開始タグ** と **終了タグ** で **マークアップ** (印付け)することにより、その種類を明確にします。マークアップした部分は左のような構造になり、全体を要素と呼びます。

この例では、開始タグ`<h1>`と、終了タグ`</h1>`で囲まれた範囲「初めてのHTML」が「大見出し」であることを示しています。

人が見ると、「見出し」が「始めてのHTML」、「本文」が「HTMLを学習して・・・」であることは一目瞭然ですが、コンピュータにとってそれが何であるかは自明ではありません。そこで、`<h1>始めてのHTML</h1>`と書くことで、「始めてのHTML」が「大見出し」であることを「印付け（マークアップ）」します。

それが何であるかを示す適切なタグを使うことが、良いウェブサイト作りの第一歩です。  [^108]

[^108]: 全部で100以上のタグが用意されていますが、良く用いるタグは十数個です。少しずつ学んでいきましょう。


### タグは入れ子にする

![](right_html.png)

複数のHTMLタグでマークアップする場合、開始タグと終了タグは、他のタグの中に完全に入った状態(入れ子)にしましょう。

![](wrong_html.png)

上は正しいHTMLで、下は誤ったHTMLです。タグの対応状態を確認してください。


### 親要素と子要素

![](html14.png)

正しく入れ子の形で記述すると、階層構造が作られます。このとき、上位階層の要素を「親要素」、下位階層の要素を「子要素」と呼びます。

例えば、`<body>` は `<h1>` の親要素、 `<h1>` は `<body>` の子要素です。


### コメント(覚書・説明書き)

![](comment.png)

ソースコードを分かり易くするための覚書、説明書きが出来るよう、**コメント** と呼ばれる構文があります。

作成したウェブサイトを閲覧する際にはないものとして扱われますので、文法上、絶対に書かなければならないものではありませんが、適切なコメントを書くことにより、自らの助けともなります。

また、不要となったコードを一時的に非表示にしたり、参考となるコードを残しておく為にも使うことが出来ます。

`<!--  -->` の間にコメントを書いていきますが、Windsurf など ほとんどのテキストエディタでは、`command + /` (コマンドキーと `/`キーを同時に押す)でコメントにしたり、もう一度押すとコメントを解除できます。エディタの支援機能を活用して、効率的にコーディングしていきましょう。


### 字下げ(インデント)

HTMLでは、空白や改行は意味を成さない為、任意に字下げや改行が行えます。そして、タグの親子関係(入れ子関係)が分かりやすいよう、綺麗に字下げ(インデント)することを心がけましょう。
自分が書きたいことを明確にできるとともに、コードの不具合も発見しやすくなります。

![](good_html1.png)

**好ましい字下げ 1**<br>
まったく字下げをせず一行に書いていますが、短いコードなのでひと目で分かります。


![](good_html2.png)

**好ましい字下げ 2**<br>
タグごとに字下げをした例です。タグの中にタグが配置されているという、抱合関係・親子関係が明確で、これも好ましいコードです。


![](good_html3.png)

**好ましい字下げ 3**<br>
全てのタグを字下げすることにより、行数の増加や、深すぎる字下げが視認性の妨げとなることもあります。`<h1>`タグをひと目で理解できるため字下げせずに書いた例です。これも好ましいコードです。

![](bad_html.png)

**不適切な字下げ**<br>
字下げが滅茶苦茶になっています。これは少し極端な例ですが、慣れないうちはつい目茶苦茶になりがちです。上の三つの例のように見易く綺麗な字下げを心がけましょう。

== HTMLを記述する

作成したいウェブサイトのイメージができたので、実際に HTML を書いて、`wave-studio` のトップページを作り始めていきましょう。


### ビューポート

作成した`index.html`を編集し、次のように更新しましょう。

**▼index.html**

```html
<!DOCTYPE html>
<html>
  <!-- サイトに関する設定事項を記述します -->
  <head>
    <!-- 文字コードの指定 -->
    <meta charset="utf-8">

    <!-- ページのタイトル -->
    <title>WAVE</title>

    <!-- ビューポート(視点)の指定をすると、
         iPhone でも きれいに見せることができる -->
    <meta name="viewport" content="width=device-width">
  </head>

  <!-- 表示したい内容・出し物・コンテンツを記述します -->
  <body>

  </body>
</html>
```

HTMLの文法で解説したように、分かりやすくするために `<!-- -->` で囲まれた「コメント」を入れています。  [^109]

ページのタイトルを、

```html
<title>WAVE</title>
```

としましたので、ブラウザのタブに「WAVE」と表示されます。

[^109]: 作成例は説明の為に多くのコメントを記入していますが、冗長だと感じる方もいらっしゃることでしょう。ご自身の読みやすさに合わせて、適宜、調整されてください。


ここで、着目したいのは次の記述です。

```html
<meta name="viewport" content="width=device-width">
```

先に述べたように、iPhone で見る人が多いので、この短いコードを入れておくことで、後々、それぞれの端末の表示幅に応じた配置ができるようになります。

### サイト名

HTML は、ウェブサイトの「文書構造」を記述するための言語です。そして、サイトに表示する文字が、「見出し」なのか「本文」なのかなど、それぞれの要素に適した「タグ」を使って「印付け（マークアップ）」を行うことが、肝要でした。

ウェブサイトの完成図を見ると、いくつかの部品に分けられそうです。それぞれの部品に適した「印付け（マークアップ）」を行っていきましょう。

![](header.png)

まずは、サイト名から作っていきましょう。

サイト名を表すための専用のタグはありませんが、`<header>` タグを使って書くのが良さそうです。現在、空っぽになっている `<body>` タグの中に書き加えて、次のようにしましょう。

**▼index.html**

```html
<!-- 表示したい内容・出し物・コンテンツを記述します -->
<body>
  <!-- ヘッダー(サイト名) -->
  <header>WAVE</header>
</body>
```


:::{.note}
**入力支援機能の活用**

`<header>WAVE</header>` をすべて手入力してもよいですが、Windsurf には HTML タグを楽に入力できる「入力支援機能」が備わっています。

![](shien1.png)

`he` と入力すると、左のように入力したいタグの候補が表示されます。キーボードの下矢印キーで`<header></header>` を選び、Enter キーを押しましょう。

![](shien2.png)

すると、`<header>` と `</header>` が自動で入力され、間にカーソルが点滅します。そのまま `WAVE` と入力すれば、`<header>` タグは完成です。

このような入力支援機能を活用することで、タイピングミスを減らしながら効率的にコーディングを進められます。
:::

**ピクセル数について**

スマートフォンの画面サイズを説明するとき、「横 375px × 縦 667px」といった表現をよく目にします。ここでの `px`（ピクセル）とは何でしょうか？

> ピクセルとは、デジタル画像や画面などを構成する最小単位である、色のついた微細な点のこと。また、その数を表す単位。単位を表す場合は “px” と略記されることもある。

### ヒーローイメージ

![](hero.png)

「ヒーローイメージ」とは「サイトを印象づける大きな画像」のことです。

`wave-studio` では、爽やかな青い空と海の写真を、ページ最上部いっぱいに配置して、見る人に「ここは海辺のライフスタイルを紹介するサイトなんだ」と直感的に伝えることにしましょう。

写真そのものは、自分で撮りに行ってもよいですし、巻末の参考リンクにも掲載したように、多くのサイトで豊富な写真やイラストが提供されていますので、そちらを利用するのも便利です。

それでは、先ほどのヘッダーのあとに続けてコードを書いていきましょう。

**▼index.html**

```html
  <!-- ヘッダー(サイト名) -->
  <header>WAVE</header>

  <!-- ヒーローイメージ（サイトを印象づける大きな画像） -->
  <figure class="hero">
    <img
      src="images/blue_sea_1920x1280.webp"
      alt="爽やかな青い空と海">
  </figure>
</body>
</html>
```

ここでの要点は、何といっても次の一行です。

**▼index.html**

```html
<img
  src="images/blue_sea_1920x1280.webp"
  alt="爽やかな青い空と海">
```

`<img>` は、画像を表示するために用意されているタグです。
先に登場した `<header>` タグと比べると、大きく二つの違いがあります。

:::{.notice}
**`<img>` と `<header>` の比較**

`<img src="images/blue_sea_1920x1280.webp" alt="爽やかな青い空と海">`<br>
`<header>WAVE</header>`

* **コンテンツ(内容物)**や **終了タグ`</img>**`がない。 <br>
　`<header>` タグの場合には、印付け(マークアップ)した「WAVE」の前後を `<header>` と `</header>` で囲んでいました。<br>
　`<img>`は、画像の表示を目的としているため、**開始タグ**のみで目的を達成できます。そのため`<header>`タグにはあった**コンテンツ(内容物、WAVE)**や**終了タグ**`</header>`がありません。<br>
このように、コンテンツを持たない要素のことを**空要素**と言います。

* `src`と`alt`の二つの**属性**を持っている。 <br>
  　**属性**は、**要素への追加情報を指定する**ためのもので、必須の属性と任意の属性があります。<br>
  　`src` 属性は、source(源泉)の略で表示したい画像の所在を与えます。これは必須の属性です。ここでは、`src="images/blue_sea_1920x1280.webp"` と書くことで、`images` ディレクトリ(フォルダ)の中にある海の写真を表示することを示しています。 <br>
  　もう一つの、`alt` 属性は、alternative({代替|だいたい})の略で何らかの理由で画像が見つからないときに表示する代替テキストを指定します。これも(一部の例外はありますが)ほぼ必須の属性です。目の見えない方が利用する音声読み上げソフトも、代替テキストを利用して画像の内容を伝えます。`alt`属性には、その画像に相応しいテキストを記述しましょう。

もう一つのポイントは、この`<img>`タグを囲むように書かれた `<figure>`タグです。

**▼index.html**

```html
<figure class="hero">
  (略)
</figure>
```

![](hero-non-css.png)

`<figure>`タグは、その内容物（コンテンツ）が、「図版」であることを示すために用いるタグです。写真の他にも、イラストや図、詩やプログラムコードなど、さまざまなものを含むことができます。

また`<figure>`タグを、単に `<figure>` と書くのではなく `<figure class="hero">` と、**属性(要素への追加情報)**を記述しています。`class`は**属性名**と呼ばれ、`hero`は**属性値**と呼ばれます。属性名`class`は、特に**クラス名**とも呼ばれます。

**クラス名**を付けると「特定の要素を他の要素から区別する、異なる要素を同じものとして取り扱う」ことが容易になります。

例えばこのサイトでは、いずれ複数の写真を掲載していきます。ヒーローイメージの写真に `class="hero"` と `hero` クラスを付与しておくと、多数の写真の中から「トップに大きく表示する 1 枚」を区別できるようになります。

さて、結果を確認してみましょう。右のようになります。
完成形のヒーローイメージと比べると、写真が少し見切れています。
写真が大きく画面幅に全てを収めきれていないためですが、後ほど CSS を用いて見栄え良く出来ますので、今は先へ進むことと致しましょう。

**画像形式について**

コンピュータで画像を扱うための主な形式には、**ラスター形式**と**ベクター形式**があります。大まかに言うと、写真のようなピクセルの集まりを扱うのがラスター画像、ロゴやアイコンのように線や図形で表現するのがベクター画像です。

### ラスター形式
* JPEG（ジェイペグ） - フルカラー・非可逆圧縮・写真用途
* PNG（ピング） - フルカラー・可逆圧縮・半透明化用途
* GIF（ジフ）  - 256色の図やイラスト・簡易アニメーション用途

とくに写真やイラストなどの「ピクセル画像（ラスター画像）」では、最近は WebP（ウェッピー）を使うと便利です。
WebP は JPEG / PNG / GIF を置き換えられる次世代の画像形式で、同じ画質ならファイルサイズを小さくできるため、表示も速くなります。

画像の形式を変換したいときは、`ImageMagick` [^113] などのツールを使うと簡単です。
WebP について詳しく知りたい方は、制作会社 ICS による解説記事 [^114] も参考になります。

ここまでは主にラスター画像について見てきましたが、ロゴやアイコンなど「どのサイズでもくっきり見せたい」ものには、次に紹介するベクター画像もよく使われます。

### ベクター形式
* SVG（エスブイジー） - ベクター形式の図やアイコン用途。HTML に似たタグのテキストで書かれており、拡大縮小しても画質が劣化しない。例えば `circle.svg` の中身は次のようになります。

```html
<svg width="100" height="100">
  <circle cx="50" cy="50" r="40" fill="skyblue" />
</svg>
```

ここでは、円の中心座標を表す `cx`（x 方向）, `cy`（y 方向）と、半径 `r` を数値で指定して円を描いています。ピクセルを並べるのではなく「円」という図形として記録しているため、何倍に拡大してもきれいな円のままです。

### ソースコードへのリンク

![](link_button.png)

続いては、`wave-studio` のソースコード一式を GitHub 上で公開しておき、そこへジャンプできるリンクをページ上に用意しましょう。完成形のイメージは右のようになります。

虹色のグラデーションなど、見た目の装飾は CSS の章でじっくり扱いますので、ここでは HTML 側の「文書構造」を整えることに専念します。

それでは、キャッチフレーズのすぐ下に、次のようなコードを書き足しましょう。

**▼index.html**

```html
<!-- キャッチフレーズ -->
<h1 class="glow text">
  Best place to visit in the world
</h1>

<!-- サブタイトルとソースコードへのリンクボタン -->

  </a>
</p>
```

ここで登場した `<p>` タグは、`paragraph（パラグラフ）` (段落) を表すためのタグです。通常の文章などを記述する際に広く用いることができるタグです。

`<p>` タグの中には、装飾用の小さな画像 `<img>` と、リンク用の `<a>` タグが含まれています。`<br>` タグは改行で、「小さな丸いアイコン」と「リンクの文字列」を別の行に分けるために使っています。


#### class 属性と id 属性

HTML では、`class="..."` や `id="..."` のように、タグに名前と値のペアで「属性」を付け加えることで、意味や役割をより詳しく指定できます。

`class` 属性は「グループ名」のようなもので、同じクラス名を複数の要素に付けられます。例えば、このページでは `<h1 class="glow text">` や `<p class="sub_title">` と書き、後から CSS で `.glow` や `.sub_title` にまとめて装飾を指定します。同じクラス名を付けておけば、別の場所でも同じスタイルを簡単に再利用できます。

一方、`id` 属性はページの中で 1 つだけに付ける「固有の名前」です。少し後に登場する `<section id="recent">` のように書いておくと、CSS や JavaScript から `#recent` という目印でそのブロックを指し示せますし、ナビゲーションメニューからその位置までジャンプするリンクも作れます。


#### `<a>`タグ (Git（ギット） / GitHub（ギットハブ）)

`<a>` タグはリンクを作るための要素です。`href` 属性にリンク先の URL を指定します。

`<a href="https://github.com/Atelier-Mirai/wave-studio">wave-studio のソースコードはこちら</a>` のように書くと、「wave-studio のソースコードはこちら」という文字列全体がリンクになり、クリックすると GitHub 上の `wave-studio` リポジトリに移動します。

それでは、実際にリンク先の [https://github.com/Atelier-Mirai/wave-studio](https://github.com/Atelier-Mirai/wave-studio) に訪れてみましょう。

![](github_download.png)

`Git（ギット）` や `GitHub（ギットハブ）` は、ソースコードのバージョン管理や、コードの公開・共有を助けてくれる、開発に欠かせないサービスです。

長い HTML の旅もいよいよ最後です。フッターを作っていきましょう。

ヘッダー(サイト名)を表示する際に `<header>` タグを用いましたので、`<footer>` タグは、対になる存在として覚えやすいと思います。

また「段落」というほどの分量ではありませんが、「© WAVE」や住所、メールアドレスのために それぞれ `<p>` タグを用いています。

## この章のまとめ

ここまでで、`wave-studio` トップページの HTML を一通り組み上げました。

- `<!DOCTYPE html>` から `<html>`, `<head>`, `<body>` まで、HTML 文書の基本的な枠組みを確認しました。

## コラム：次は CSS でレイアウトの衣装をまとわせる

この章で書いてきた HTML は、いわば「骨組み」と「テキストの内容」をきちんと並べた状態です。まだ余白や色づかいは素朴かもしれませんが、ヘッダー・ヒーローイメージ・最新記事・フッターといった区切りは、すでに HTML のうえでできあがっています。

次の章からは、この骨組みに CSS を重ねていきます。文字の大きさや色を変えたり、余白を整えたり、画像とテキストの配置をそろえたりしながら、「海辺のライフスタイル誌らしい」レイアウトに近づけていきましょう。

CSS では、フォントや色だけでなく、グリッドレイアウトといったレイアウトの仕組みも使えるようになります。ちょうど雑誌の紙面レイアウトを考えるときのように、縦横2次元のマス目の上に文章や写真を自由に配置しながら、「海辺のライフスタイル誌」らしいページ構成を試せるのが、とても楽しいところです。同じ HTML でも、スタイルを変えるだけで驚くほど雰囲気が変わる様子を、これから一緒に体験していきます。
「タグの意味を考えて素直にマークアップする」という、この章で身につけた癖は、そのまま CSS からも扱いやすい HTML づくりにつながります。次の章では、その HTML をキャンバスに見立てて、CSS という絵の具で少しずつ色とリズムを乗せていきましょう。
:::
