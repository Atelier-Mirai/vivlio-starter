# クロスリファレンス（相互参照）の使い方

:::{.chapter-lead}
本章では、本書で定義している **図・表・リストのナンバリングと相互参照（クロスリファレンス）** の
書き方を説明します。

「この図を参照してください」「詳しくは前の表を参照してください」といった記述を、章番号付きの
番号（図 3-2, 表 4-1 など）とリンク付きで自動生成するための仕組みです。
:::


## ラベル付きキャプションの書き方

クロスリファレンスは、次の二段階で構成されています。

1. **定義側**: 図・表・リストに「ラベルID（`@id`）」を付けてキャプションを記述する
2. **参照側**: 本文中から `@id` を使ってその要素を参照する

ビルド時に、各章の図・表・リストに「章番号＋連番」の番号が自動で割り振られ、
本文中の `@id` は「図 3-2」「表 4-1」「リスト 5-3」のような番号付きのテキスト（＋リンク）に
置き換えられます。

### 共通ルール

図・表・リストには、次のような **キャプション行** を付けます。

- 行頭に太字 (`** ... **`) でタイトルを書く
- タイトルの末尾に `@ラベルID` を付ける
- 直後に続くブロック要素（コードブロック / テーブル / 画像）がその対象になります

ラベルIDは、本文中で参照するときに使う名前です（例: `@css-table-rotate`）。

### コード（リスト）の例

CSS コード例を「リスト」として扱い、相互参照したい場合は次のように書きます。

````markdown
** 回転テーブルのCSS定義 @css-table-rotate **
```css:table.css
table {
  border-collapse: collapse;
  /* 略 */
}
```
````

- キャプション行: `** 回転テーブルのCSS定義 @css-table-rotate **`
- 直後のブロック: ```css:table.css ～ ``` → **コードブロック** なので「リスト」として扱われます。

### 表の例

テーブル（表）をナンバリングの対象にしたいとき:

```markdown
** 回転テーブルのプロパティ一覧 @prop-list **
| プロパティ名 | 役割     |
| :----------- | :------- |
| --scale      | 縮尺率   |
| --shift-y    | Y軸シフト |
```

- キャプション行の直後が Markdown テーブルなので、「表」として扱われます。

### 図の例

画像を図として扱い、参照できるようにしたい場合:

```markdown
** 回転テーブルのイメージ図 @table-rotate-image **
![回転テーブルのイメージ図](table-rotate.png)
```

- 直後が画像（`![]()`）なので、「図」として扱われます。

### 番号を付けない例（ただのコード例など）

`@id` を書かなければ、その要素は **番号も相互参照も付かない単なる例** として扱われます。

````markdown
** CSSの例 **
```css
h1 {
  color: red;
}
```
````

このように書くと、見出しとして太字にはなりますが、

- 「リスト 4-1」などの番号は付かない
- 本文から `@id` で参照することもできない

という扱いになります。

## 本文での参照方法

### `@id` による参照

本文からは、定義したラベルIDを **`@id` という形でそのまま書くだけ** で参照できます。

例:

```markdown
横長表の調整方法については、@css-table-rotate を参照してください。
ここで定義されているカスタムプロパティの一覧は、@prop-list にまとめています。
レイアウト全体のイメージについては、@table-rotate-image を参照してください。
```

ビルド結果では、次のような形になります（例）:

- `@css-table-rotate` → 「リスト 4-1」へのリンク
- `@prop-list` → 「表 4-2」へのリンク
- `@table-rotate-image` → 「図 4-3」へのリンク

### 表示されるテキスト

番号部分（リスト 4-1 / 表 3-2 / 図 1-5）は、

- **章番号**（第n章）
- **その章の中での連番**

から自動的に計算されます。著者が手動で番号を管理する必要はありません。

## ラベルIDの扱い（手動ID・自動ID・一意性・命名規則）

### 手動ID（@id）

意味のある名前でIDを付けたい場合は、キャプション行に直接 `@id` を書きます。

```markdown
** 回転テーブルのCSS定義 @css-table-rotate **
```

- 本文では `@css-table-rotate` と書くだけで参照できます。
- ID は本書全体で一意である必要があります（同じIDを複数の図表リストに使うことはできません）。

### 自動ID（@auto / @omakase）

ID を自分で考えたくない場合は、キャプション行に `@auto` または `@omakase` を書きます。

````markdown
** 別のコード例 @auto **
```ruby
def prime?(n)
  # ...
end
```
````

- この場合、システムが「リスト 4-4」のような一意のIDを内部的に割り当て、図表番号を自動で付けます。
- 内部IDの名前は本文からは分かりません。本文中で **番号付きリンクとして参照したいもの** には `@auto` ではなく、`@css-table-rotate` のように自分で意味のあるIDを付けてください。`@auto` は「番号は振りたいが、本文からは参照しない」要素に向いています。

### システム予約ID（予約語）

`@id` という記法は、本章で説明したクロスリファレンスのほかに、システム組み込みの **ビルド時マクロ** でも使われます。以下の `@id` は **システム予約語** として扱われ、クロスリファレンス側では「未定義のラベルID」として警告されません。

#### 自動採番用の予約ID

| 予約ID | 用途 |
| :--- | :--- |
| `@auto` | 自動ID割り当て（前掲「自動ID」） |
| `@omakase` | 同上（別名） |
| `@id` | 同上（別名） |

#### 組み込みマクロの予約ID（完全一致）

| 予約ID | マクロの役割 |
| :--- | :--- |
| `@vspace` | 縦方向の余白挿入（`@vspace:1.5lh` / `@vspace:10`（単位省略時は mm）。負値で上に詰める） |

:::{.tip}
**著者が新しいラベルIDを付けるときの注意**

上記の予約語と衝突する ID（例: `@auto` という図を定義する等）は使用しないでください。予約語扱いとなるため、本文から参照してもクロスリファレンスとして展開されません。
:::

### IDの一意性（重複禁止）

ラベルIDは、リスト・表・図をまたいで **本書全体で一意でなければなりません**。たとえば、リストと図の両方で `@css-table-rotate` を定義することはできません。

同じIDが複数箇所で定義されていると、ビルド時に重複エラーとして検出され、次のようなメッセージが表示されます。

```
🔴 71-install.md:42 - ラベルID 'CSS定義 @css-table-rotate' は重複しています
          重複箇所: 71-install.md: 42, 88
```

どのファイルの何行目で重複しているかが示されるので、いずれかのIDを変更してください。なお、重複があってもビルドは最後まで実行されますが、意図したリスト番号等が付かない{虞|おそれ}があるため、原稿を見直すことをおすすめします。

意味のあるIDを種別ごとに分けて使いたい場合は、次のようにID自体を変えてください。

- `@css-table-rotate-css` （リスト用）
- `@css-table-rotate-fig` （図用）

このようにしておくと、本文中の参照も明確になり、後から読んだときにも意味が分かりやすくなります。

### ラベルIDの命名規則（推奨）

ラベルID（`@id`）に厳密な制約はありませんが、次のように付けておくと、IDを見ただけで「どの章の何を指すか」が分かり、重複も避けやすくなります。

- **英小文字＋数字＋ハイフン**（kebab-case）で統一する
- 内容を表す英単語を選ぶ（コード例なら `@ruby-sample`、表なら `@basic-table`、図なら `@einstein` など）
- 同種のものが章をまたいで増える場合は、**章番号を末尾に添える**と整理しやすい

```markdown
# 図が多い章
@einstein-11, @diagram-11, @diagram-12

# 表が多い章
@basic-table, @data-table-25
```

どの種別（リスト／表／図）の番号になるかは、ID名ではなく **キャプション行の直後にあるブロックの種類** から自動で判定されます。そのため、ID自体に `fig-` や `table-` といった接頭辞を付ける必要はありません（付けても構いません）。

## 実践的な使用例

### 図の定義と参照

```markdown
**アインシュタインの写真 @einstein**

![](Einstein.png){width=40% align=center}
```

このように記述すると、次のように表示されます。

:::{.output}
**アインシュタインの写真 @einstein**

![](Einstein.png){width=40% align=center}
:::

この図を参照したい場合には、`@einstein`というIDを使用して、次のように記述します。

```markdown
@einstein は、アインシュタインが自室で研究しているイメージ画像です。
```

このように記述すると、次のように表示されます。{.aki}

:::{.output}
@einstein は、アインシュタインが自室で研究しているイメージ画像です。
:::

`@einstein` が、「図1-1」に自動で変換されています。自分で図や表の番号を管理する手間が省けるので便利です。

```markdown
**テスト図（自動ID） @auto**

![](Einstein.png){width=30% align=right}
```

`@auto` または `@omakase` を用いても図や表の番号を自動付与することが出来ます。本文中から参照することは出来ませんが、`@id`を考える手間を省けるので便利です。

### 表の定義と参照


```
**基本的な表 @basic-table**
| 名前 | 年齢 | 職種 |
|:------:|:------:|:------:|
| 太郎 | 25   | エンジニア |
| 花子 | 30   | デザイナー |
| 次郎 | 35   | マネージャー |

@basic-table は、基本的な表の一例です。
```

と記述すると、次のように表示されます。

:::{.output}
**基本的な表 @basic-table**
| 名前 | 年齢 | 職種 |
|:------:|:------:|:------:|
| 太郎 | 25   | エンジニア |
| 花子 | 30   | デザイナー |
| 次郎 | 35   | マネージャー |

@basic-table は、基本的な表の一例です。
:::


### コード（リスト）の定義と参照

````
**Rubyサンプルコード @ruby-sample**

```ruby
def hello(name)
  puts "Hello, #{name}!"
end

hello("World")
```
````

このように記述すると、次のように表示されます。

:::{.output}
**Rubyサンプルコード @ruby-sample**

```ruby
def hello(name)
  puts "Hello, #{name}!"
end

hello("World")
```
:::

`@ruby-sample` が、「リスト1-1」に自動で変換されています。
書籍内で、ソースコードを元に解説をする際に、

```
@ruby-sample は、Rubyで画面表示を行なうサンプルコードです。
```

と記述すると、

:::{.output}
@ruby-sample は、Rubyで画面表示を行なうサンプルコードです。
:::

と表示されますので、リスト1-1であったか、リスト1-2であったか、著者自身で管理する手間が省けますので、便利です。

### コード（リスト）の取り込みと参照

````markdown
**Prime @prime**
```include:prime.rb```

**Prime2範囲指定 @prime-range**
```include:prime.rb:14-17```
````

のように、コード（リスト）を取り込んで参照することもできます。
ビルドすると、次のように表示されます。

:::{.output}
**Prime @prime**
```include:prime.rb```

**Prime範囲指定 @prime-range**
```include:prime.rb:14-17```
:::

コードの参照をしたい場合にはこちらも、先ほどと同様、`@id` を指定することで行なえます。

```
@prime は、Rubyで素数を表示するコードです。
@prime-range は、その一部を指定したコードです。
```

と記述すると、

:::{.output}
@prime は、Rubyで素数を表示するコードです。
@prime-range は、その一部を指定したコードです。
:::

と表示されます。

### 章間参照

参照は章をまたいでも同じ書き方です。前章で定義した `@einstein` や `@basic-table`、`include` で取り込んだ `@prime` / `@prime-range` も、これまでと同じく `@id` と書くだけで参照できます。ビルド時に書籍全体からラベルが解決され、「図 2-3」「リスト 5-1」のように正しい章番号付きのリンクになります。

## トラブルシューティング

### よくある問題

#### IDが見つからない
- 原因: タイプミス、または参照先のラベルが定義されていない
- 解決: 定義側のキャプション行（`@id`）のスペルを確認する

#### 重複エラー
- 原因: 同じIDを複数の要素で使用
- 解決: IDを一意になるように変更

#### 番号がおかしい
- 原因: ビルドのキャッシュ残留
- 解決: `vs clean` でクリーンアップ

### エラーメッセージの例

`vs preflight` や `vs build` で表示される主なエラーメッセージと対処法です。

#### ラベルIDの重複

```
🔴 25-querystream.md:25 - ラベルID '画像(左寄せ) @img-left' は重複しています
        重複箇所: 25-querystream.md: 25, 30
```

同じ `@id` が複数箇所で定義されています。IDは書籍全体で一意にする必要があります。いずれかのIDを変更してください。

#### 孤立ラベル（未参照の警告）

```
🟡 24-cross-reference.md:329 - 孤立ラベル 'Prime @prime' は未参照です
```

定義されたラベルが本文中のどこからも `@prime` で参照されていません。参照を追加するか、不要であればラベル定義を削除してください。これは警告（🟡）であり、ビルドは正常に完了します。

### デバッグ方法

1. **個別ビルド**: `vs build [章番号]` で特定章のみテスト
2. **バリデーション**: `vs lint` で基本的な問題を検出

## まとめ

クロスリファレンス機能を使いこなすことで：

- **参照の一貫性**: 章番号付きで正確な参照
- **リンク機能**: PDF内でジャンプ可能な相互参照
- **自動管理**: 番号付けの自動化と重複防止
- **柔軟性**: 手動IDと自動IDの使い分け

これらの機能を活用して、読者にとって使いやすく、著者にとって管理しやすい技術書を作成しましょう！
