# Unix シェル超ダイジェスト

:::{.chapter-lead}
本書の本文だけでも、Windsurf とブラウザを使って Web サイトを作り切ることはできます。ただ、将来クラウドサーバや他の開発環境に触れるときのことを考えると、「Unix シェル」と呼ばれるコマンドラインの世界を少し知っておくと、あとあと効いてくる場面が多くあります。

ここでは、ほんの触りだけをコンパクトにご紹介いたします。
:::

## シェルって何ですか

:::{.section-lead}
「シェル（shell）」とは、キーボードからコマンドを入力して、OS にいろいろな仕事をしてもらうための対話窓口のような存在です。

GUI（マウスでクリックする画面）と違って、文字だけで操作しますが、そのぶんスクリプトにして自動化しやすい、という大きな利点があります。
:::

- macOS / Linux では、最初から Unix 系シェル（`zsh` や `bash` など）が用意されています。
- Windows でも、WSL（Windows Subsystem for Linux）や Git Bash を使うことで、Unix に近い環境を手元に用意できます。


## どこから触ればいいですか

### macOS の場合

- 「アプリケーション > ユーティリティ > ターミナル」からターミナルを起動します。
- 何も設定していなければ、`zsh` というシェルが立ち上がるはずです。
- プロンプト（`%` などで終わる行）が表示されたら、そこにコマンドを入力して Enter キーを押します。

### Windows の場合

- まずは **WSL（Ubuntu など）** か **Git Bash** を導入しておくと、Unix 系のコマンドにそのまま触れます。
- WSL の Ubuntu を起動すると、`$` で終わるようなプロンプトが表示されます。
- Git Bash も同様に、`$` で終わるプロンプトに対してコマンドを入力していきます。

### Windsurf の内蔵ターミナル

Windsurf には内蔵ターミナルがあり、エディタの下部にシェルを表示できます。

- メニューやショートカットからターミナルを開くと、現在開いているプロジェクトフォルダをカレントディレクトリとして、シェルが起動します。
- macOS なら `zsh`、Windows なら WSL のシェルや Git Bash などを既定として選んでおくと、学んだコマンドを他の環境でもそのまま使い回しやすくなります。


## 習得したい基本的なターミナルコマンド

ここでは、本書の学習で特に出番がありそうな基本コマンドだけを挙げておきます。

まずは、次のような流れを何度か試してみると、雰囲気がつかみやすくなります。

```zsh
% pwd
/Users/yourname/projects
% ls
study  taskleaf
% cd study
% ls
index.html  style.css  images
```

- `pwd` で「いまどこにいるか」を確認し、
- `ls` で中身を眺め、
- `cd` で目的のディレクトリに移動し、
- 必要であれば `mkdir` で新しいディレクトリを作る、というのが基本パターンです。

ここまでで登場した `pwd` / `ls` / `cd` / `mkdir` に加えて、

- `cp`
- `mv`
- `rm`（削除コマンドなので要注意）

を含めた 7 つを、「賢いプログラマがよく使う 7 つの基本コマンド」として覚えておくとよいでしょう。  
この付録では、まずこれら 7 つを中心に紹介しつつ、余裕があれば役に立つコマンドもあわせて一覧にしています。

このイメージをつかんだうえで、代表的なコマンドを表形式で整理しておきます。

### ls (LiSt)
カレントディレクトリにあるファイルやディレクトリを表示

| コマンド | 意味 |
|:---------|:-------------------------------------------------|
| `ls` | ファイルやディレクトリを表示 |
| `ls -l` | ファイルやディレクトリを詳細表示 |
| `ls -a` | ファイルやディレクトリを表示(隠しファイル含む) |
| `ls -la` | ファイルやディレクトリを詳細表示(隠しファイル含む) |

### cd (Change Directory)
カレントディレクトリを移動

| コマンド | 意味 |
|:---------|:-------------------------------------------------|
| `cd source` | sourceディレクトリに移動|
| `cd ~/` | ホームディレクトリに移動 |
| `cd ..` | 親ディレクトリに移動|

### pwd (Print Working Directory)
カレントディレクトリを表示

| コマンド | 意味 |
|:---------|:-------------------------------------------------|
| `pwd` | カレントディレクトリを表示|

### touch
タイムスタンプ更新や空ファイル作成

| コマンド | 意味 |
|:---------|:-------------------------------------------------|
| `touch index.html` | index.html ファイルを作成 |

### mkdir (MaKe DIRectory)

新しいディレクトリを作成
| コマンド | 意味 |
|:---------|:-------------------------------------------------|
| `mkdir stylesheets` | stylesheets ディレクトリを作成 |

### mv (MoVe)
ファイル名変更や移動

| コマンド | 意味 |
|:---------|:-------------------------------------------------|
| `mv master.css style.css` | master.cssからstyle.cssにファイル名変更 |
| `mv style.css stylesheets`| style.cssをstylesheetsディレクトリに移動 |

### cp (CoPy)
ファイルをコピーする

| コマンド | 意味 |
|:---------|:-------------------------------------------------|
| `cp master.css style.css` | master.cssをstyle.cssにコピー|
| `cp style.css stylesheets`| style.cssをstylesheetsディレクトリにコピー |
| `cp -r stylesheets /tmp/`| stylesheetsディレクトリを/tmp/ディレクトリに<br>再帰的(Recursive)にコピー|

### rm (ReMove)
ファイル削除

| コマンド | 意味 |
|:---------|:-------------------------------------------------|
| `rm master.css` | master.cssを削除 |
| `rm -rf stylesheets`| stylesheetsディレクトリを削除(要注意) |

### open
ターミナルからFinderでファイルを開く

| コマンド | 意味 |
|:---------|:-------------------------------------------------|
| `open .` | カレントディレクトリをFinderで開く |
| `open ~/` |ホームディレクトリFinderで開く|
| `open -a "Windsurf" ./` |カレントディレクトリをWindsurfで開く|

### history
コマンド履歴を表示

| コマンド | 意味 |
|:---------|:-------------------------------------------------|
| `history` | コマンド履歴を表示 |

### cat (conCATenate)
ファイル内容表示及び連結

| コマンド | 意味 |
|:---------|:-------------------------------------------------|
| `cat a.txt` | 内容を表示 |
| `cat -n a.txt` | 行番号付きで内容を表示 |
| `cat a.txt b.txt > c.txt` | 結合してc.txtに書込|


## ちょっとした注意点

- `rm` コマンドはファイル削除用で、とても強力です。誤って重要なファイルを消してしまうと、元に戻せない場合があります。
- とくに `rm -rf` のようなコマンドは、使い方をよく理解するまでは安易に実行しないようにしてください。

本書では、危険度の高いコマンドには踏み込みません。まずは `pwd` / `ls` / `cd` / `mkdir` だけを丁寧に使えるようになる、くらいの気持ちで十分です。


## コマンドの調べ方

コマンドの詳しい使い方を知りたくなったときは、ターミナルの中で次のように入力してみてください。

- `ls --help` のように、`--help` オプションを付ける
- `man ls` のように、`man` コマンドでマニュアルページを開く（WSL や macOS など）

どちらも英語中心ではありますが、「どんなオプションがあるのか」「よく使われる使い方は何か」を知る手がかりになります。あわせて、ブラウザで「ls コマンド 使い方」などと検索してみるのも有効です。


## どこまで学べばいいですか

Web 開発を進めるうえで、Unix シェルは「最初から完璧に覚えなければならないもの」ではありません。

- 最初は GUI だけで作業していてもかまいません。
- 必要を感じたタイミングで、少しずつシェルに触れていけば大丈夫です。

将来、クラウド上のサーバを扱ったり、自動化スクリプトを書いたりしたくなったときに、ここで触れたコマンドがきっと役に立つはずです。興味が湧いた方は、外部の入門書やオンラインチュートリアルもあわせて参照してみてください。
