# シンプルスタイルのテーマ設定

:::{.chapter-lead}
Vivlio Starter では、画像を使わないシンプルなデザインのテーマスタイルを選択できます。`theme.style = simple` を設定することで、章扉や節見出しに背景画像を使わず、色とタイポグラフィを中心とした洗練されたデザインを実現できます。
:::

## シンプルスタイルとは

:::{.section-lead}
シンプルスタイル（`simple`）は、背景画像を使用せず、テーマカラーとタイポグラフィを活用したミニマルなデザインスタイルです。読みやすさを重視し、コンテンツに集中できる環境を提供します。
:::

### 特徴

- **背景画像なし**: 章扉や節見出しに画像を使用しません
- **テーマカラー中心**: `theme.color` で指定した色を効果的に活用
- **タイポグラフィ重視**: 文字の美しさと読みやすさを追求
- **ミニマルデザイン**: 装飾を抑えた洗練された見た目
- **高速ビルド**: 画像処理が不要なため、ビルド時間が短縮されます

### 画像スタイルとの違い

| 項目 | シンプルスタイル | 画像スタイル |
|:---|:---|:---|
| 章扉背景 | なし（色のみ） | frontispiece 画像 |
| 節見出し装飾 | なし（色のみ） | ornament 画像 |
| ビルド時間 | 高速 | やや時間がかかる |
| 設定の複雑さ | シンプル | やや複雑 |
| デザインの印象 | ミニマル・洗練 | 華やか・個性的 |

## 基本的な設定方法

:::{.section-lead}
`config/book.yml` の `theme.style` を `simple` に設定するだけで、シンプルスタイルが適用されます。frontispiece や ornament の設定は不要です。
:::

### 最小限の設定

```yaml
theme:
  style: simple  # シンプルスタイルを使用
  color: teal    # テーマカラーを指定
```

この設定だけで、以下が自動的に適用されます：

- 章扉の背景画像が無効化（`--frontispiece-image: none`）
- 節見出しの装飾画像が無効化（`--section-bg-image: none`）
- テーマカラーを使った美しいグラデーションとボーダー
- `simple_header.css` の読み込み

### 詳細な設定例

```yaml
theme:
  style: simple
  color: navy      # 落ち着いた紺色
  
  # frontispiece と ornament の設定は無視されます
  # （設定しても背景画像は表示されません）
  frontispiece:
    image: himawari     # この設定は無効
    padding: 10mm       # この設定は無効
  ornament: himawari    # この設定は無効
```

## テーマカラーの効果的な活用

:::{.section-lead}
シンプルスタイルでは、テーマカラーが章扉や節見出しの主要な装飾要素となります。適切な色を選択することで、書籍全体の印象を大きく左右できます。
:::

### 推奨カラー

**ビジネス・技術書向け**
- `navy` - 信頼感のある深い紺色
- `indigo` - 知的な印象の藍色
- `teal` - モダンで洗練された青緑色

**学術・教育書向け**
- `purple` - 学術的で上品な紫色
- `blue` - 清潔感のある青色
- `green` - 安心感のある緑色

**創作・エッセイ向け**
- `coral` - 温かみのあるサンゴ色
- `amber` - 親しみやすい琥珀色
- `plum` - 上品で個性的な紫色

### カラー設定例

```yaml
# 技術書の例
theme:
  style: simple
  color: navy

# 教育書の例  
theme:
  style: simple
  color: teal

# エッセイの例
theme:
  style: simple
  color: coral
```

## デザインの詳細

:::{.section-lead}
シンプルスタイルでは、`simple_header.css` によって章扉と節見出しが美しくスタイリングされます。テーマカラーを基調とした洗練されたデザインが自動的に適用されます。
:::

### 章扉のデザイン

- **枠線**: テーマカラーの1mm枠線
- **背景**: 白いグラデーション
- **影**: 立体感のあるドロップシャドウ
- **リボン**: 上下にテーマカラーのアクセントリボン
- **角丸**: 優雅な角丸処理

### 節見出しのデザイン

- **番号**: テーマカラーの背景に白文字
- **左枠**: テーマカラーの太い左ボーダー
- **背景**: 白いグラデーション
- **影**: 軽やかなドロップシャドウ
- **角丸**: 統一感のある角丸処理

## 画像スタイルからの移行

:::{.section-lead}
既存の書籍を画像スタイルからシンプルスタイルに変更する場合の手順と注意点を説明します。
:::

### 移行手順

1. **`book.yml` の変更**
   ```yaml
   theme:
     style: simple  # image から simple に変更
     color: teal    # 適切なテーマカラーを選択
   ```

2. **ビルドの実行**
   ```bash
   vs build
   ```

3. **結果の確認**
   - 章扉から背景画像が消えることを確認
   - 節見出しから装飾画像が消えることを確認
   - テーマカラーが適切に反映されることを確認

### 注意点

- **frontispiece/ornament 設定**: 設定が残っていても無害ですので、いつでも `image`スタイルに戻せるよう、残しておくと便利です。
- **テーマカラー**: 画像がない分、色の選択がより重要になります。
- **ビルド時間**: PDF圧縮に要する処理時間は、大幅に短縮されます。
- **ファイルサイズ**: PDF のファイルサイズは、ほぼ同じです。

## トラブルシューティング

:::{.section-lead}
シンプルスタイル使用時によくある問題と解決方法を説明します。
:::

### よくある問題

**Q: 章扉や節見出しが地味すぎる**

A: より鮮やかなテーマカラー（`coral`、`amber`、`magenta` など）を試してください。{.aki}

**Q: 以前の画像が表示される**

A: `vs clean --purge --cache` で PDFなどの生成物やキャッシュを削除して、再びビルドしてください。{.aki}

**Q: テーマカラーが反映されない**

A: `book.yml` の `theme.color` 設定を確認し、有効な色名（yellow / amber / orange / peach / coral / red / magenta / plum / purple / indigo / navy / blue / cyan / teal / mint / green / lime）を指定してください。{.aki}
