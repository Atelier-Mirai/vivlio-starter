**はじめての技術書づくり ～Vivlio Starter 実践ガイド～**

本書の内容
本書は、CSS組版と Vivliostyle の基本から、Ruby 製ツール「vivlio-starter」でサクッと本を作るまでを、やさしく案内します。むずかしい理屈より「まず触ってみる」を大切に、手順やコツを実例とともに紹介します。

- CSS組版と Vivliostyle とは？
  Web の技術（HTML/CSS）で紙のように美しくレイアウトする考え方と、Vivliostyle の仕組みを、図や例を交えながら紹介します。

- vivlio-starter ってなに？
  Vivliostyle をもっと扱いやすくする Ruby の gem です。プロジェクトのテンプレート作成、ビルド、リネームなど、面倒な作業をコマンド一発でこなせます。

- まずは触ってみましょう（かんたん使い方）
  ターミナルで次の3行だけで体験できます。

      vivlio-starter new mybook
      cd mybook
      vivlio-starter build

  これで PDF/プレビューができるところまで進めます。

- インストール手順
  Ruby と必要ツールの入れ方、gem の導入、動作確認までを丁寧に案内します。

- コマンド紹介
  よく使うサブコマンドを実例付きで解説します。

      vivlio-starter build   … 原稿とスタイルから成果物を生成
      vivlio-starter create  … 章やテンプレートの追加
      vivlio-starter delete  … 章やファイルの削除
      vivlio-starter rename  … ファイル名や章番号の変更 ほか

- Markdown記法の紹介 
  基本的な文法に加え、Vivliostyle Flavored Markdown（VFM）の便利な記法や、Vivlio Starter が提供する拡張機能・テンプレート活用のコツにも触れます。
  
- もっと拡張したい方へ
  HTML / CSS / Ruby の超入門と、テーマやビルドフローの拡張方法を紹介。自分らしい本づくりに一歩踏み込みます。技術的には、Ruby／Thor で実装された `vivlio-starter` 本体のコード解説も扱います。

  最後にひとこと――むずかしい準備より「まず一冊」を。手を動かし、少しずつ整えていきましょう。本書があなたの最初の一歩をやさしく後押しできればうれしいです。
  
  (なお、本書の執筆およびコード例の作成には、生成AIを活用しています。AIの出力は著者が検証・編集し、最終的な内容については著者が責任を負います。倫理と透明性の観点から、ここに明記いたします。)
    
  ## 対象読者
  本書は、**はじめて技術書を書いてみたい方**を主な対象としています。構成づくりから環境セットアップ、ビルド、PDF出力までを“まず一冊仕上げる”ことにフォーカスして案内します。
 
  ### 主要対象読者
  - **技術書執筆が初めての方**：最短で原稿を書き始め、PDFやプレビューまで到達したい方
  - **Vivliostyle / CSS組版に興味がある方**：Web技術で紙のようなレイアウトを実現したい方
  - **執筆環境を手早く整えたい方**：テンプレート作成からビルドまでをコマンドで一気通貫に回したい方
  - **小規模チームや個人制作の方**：同じ環境をすぐ共有し、迷わず執筆を進めたい方
 
  ### 具体的には
  - Markdown中心で原稿を書きたい／ブログ記事の冊子化に挑戦したい
  - コマンドラインは最低限なら触れる、もしくはこれを機に慣れていきたい
  - 同人誌・社内ドキュメント・勉強会資料など、配布物を自分の手で仕上げたい
 
  ### 前提知識
  - パソコンの基本操作と、ターミナルの超基礎がわかれば十分です
  - Rubyが入っているとスムーズ（未導入でも本書で案内します）
  - HTML/CSSの初歩がわかると、レイアウト調整がより楽しめます（必須ではありません）
 
  ### 読後の期待
  - `vivlio-starter`で新規プロジェクトを作成し、ビルドまで自力で行える
  - 章の追加・削除・リネームなど、日々の編集作業を迷わずこなせる
  - テーマやスタイルを少しカスタマイズして、自分らしい体裁に近づけられる
  - 次の一冊に向け、目次設計や作業フローを自分で回せるようになる
