# 開発環境の準備

:::{.chapter-lead}
プログラミングを始めるにあたり、まずは土台となる開発環境を整えていきます。
本書では、AI を搭載したコードエディタである `Windsurf` を作業机として、書いたコードの結果を確かめたり、開発者ツールで中身を観察したりする窓として `Firefox` をご紹介いたします。  
Windsurf と Firefox を左右に並べれば、「コードを書く場所」と「結果を見る場所」が一度に開ける、小さな開発スタジオのような環境が整います。

## 開発環境の整備

:::{.section-lead}
これから本書で学んでいくあいだ、ずっとお世話になる道具は、大きく分けて二つです。「エディタ」と「ブラウザ」です。  
プログラミングそのものは、iPad や iPhone からでも [cloud9](https://aws.amazon.com/jp/cloud9/) や [paiza](https://paiza.io/ja)、[CodePen](https://codepen.io) などのオンライン環境を使って行うことができますが、この本では Mac / Linux / Windows が動くデスクトップやラップトップコンピュータを前提に、ローカル環境での開発を進めていきます。
:::

エディタとは、プログラミングが快適に行えるよう様々な支援機能を備えたコーディングのためのソフトウェアです。さまざまなエディタがありますが、本書では `Windsurf（ウィンドサーフ）` をおすすめのエディタとしてご紹介いたします。

「ブラウザ」は、ウェブサイトを閲覧するためのソフトウェアです。自分の書いたコードの詳細を確認する為の「開発者ツール」や、iPhone や iPad など異なる端末での表示結果を確認する機能も備わっています。さまざまなブラウザがありますが、ここでは `Firefox（ファイヤフォックス）` をご紹介いたします。

他にも、ソースコードのバージョン管理システム `Git（ギット）`, `GitHub（ギットハブ）` や、画像編集ソフト `Affinity（アフィニティ）` など、サイト作成の為の様々なツールがございますので、適宜学習されて下さい。

### コラム：バージョン管理と画像編集ツール

Web サイトを少し本格的に作っていこうとすると、ソースコードの履歴を残せる **バージョン管理システム** がとても心強い味方になります。たとえば `Git（ギット）` / `GitHub（ギットハブ）` を使っておくと、

- いつどのような変更をしたかを後から見返せる
- 変更前の状態に戻しやすい
- 別の案をブランチとして試したうえで、良い方だけ採用できる

といったメリットがあり、一人で開発するときにも十分役に立ちます。本書では Git の詳細までは扱いませんが、「将来もう一歩進めたくなったら、バージョン管理を覚えると便利」ということだけ頭の片隅に置いておいてください。

あわせて登場する `GitHub` は、Git で管理しているソースコードの履歴（リポジトリ）をインターネット上に保存し、バックアップを取ったり、複数人で共有しながら開発できるようにするためのサービスです。最初のうちは名前だけなんとなく見知っておけば十分ですが、「Git で記録した変更履歴を、GitHub を通じてクラウドに置いている」といった関係だとイメージしておくとよいでしょう。

また、画像を扱う場面では、最近は生成 AI でイラストや写真を生成することもできるようになりましたが、ちょっとしたトリミングやリサイズ、文字入れなどを自分でできると、細かな調整がしやすくなります。Affinity シリーズのような画像編集ソフトや、お絵かきツールに軽く触れておくと、Web サイトづくりの表現の幅が、少しだけ広がっていきます。

## AI支援のモダンなエディタ<br>`Windsurf（ウィンドサーフ）`

Windsurf は、AI アシスタントを組み込んだモダンなコードエディタです。VS Code に近い操作感を持ちながら、「ペアプログラミング（ペアプロ）」のように画面の中に相棒としての AI を住まわせて、いつでも相談しながらコードを書けるのが大きな特長です。この本で登場する AI アシスタント `Cascade` も、その相棒のひとりです。

ここでいう `Cascade` は、本書の中で一緒にコードや文章を考えてくれる AI アシスタントの「名前」です。英単語としての *cascade* には、「小さな滝が段々に連なって流れ落ちるようす」や「出来事や処理が次々に伝わっていく」というイメージがあります。技術的には、ChatGPT や Gemini、Claude（Sonnet シリーズ）といった大規模言語モデルを使ったサービスと同じ種類のしくみを土台にしていますが、この本では細かな製品名の違いには立ち入りません。「エディタの横に座って、自然文で相談できる AI の相棒」としてイメージしてもらえれば十分です。同じ単語は、後の章で扱う <span id="idx-3ghtaxxv1hvr-66" class="index-term" data-yomi="しーえすえす">CSS</span> の正式名称 *Cascading Style Sheets*（カスケーディング・スタイルシート）にも登場しますが、ここではまず「滝のように、上から下へルールや提案が伝わっていく感じ」として軽く覚えておいてください。

- <span id="idx-5376xxg0dgl-40" class="index-term" data-yomi="えいちてぃーえむえる">HTML</span> / <span id="idx-3ghtaxxv1hvr-67" class="index-term" data-yomi="しーえすえす">CSS</span> / <span id="idx-ufy6mn67o2id-38" class="index-term" data-yomi="じゃばすくりぷと">JavaScript</span> などのコードを、タグやキーワードごとに色分けして読みやすくし、タイプミスやカッコの閉じ忘れにも気づきやすくする「シンタックスハイライト」を利用できる。
- AI によるコード補完や文章のリライトで、書きかけのコードの続きを提案してもらったり、日本語の説明文の言い回しを少し丁寧に整えてもらったりできる。
- 複数ファイルにまたがる検索や、動作は変えずにコードの構造を整える「リファクタリング」を行うことができる。
- 少し慣れてきたら、内蔵ターミナルから `cd` や `mkdir` といった簡単なコマンドを実行して、フォルダの移動や作成といった作業も行うことができる。
- Git と連携し、前の版とどこが変わったのかを色付きで見比べる「差分表示」で確認することができる。

といった機能が、最初からひとまとめに用意されています。

### 初期設定のポイント

細かな設定は後から少しずつ触っていけば十分ですが、はじめに次のあたりを整えておくと快適です。

- **テーマやフォントサイズの調整**  
  長時間眺めても疲れにくい配色と文字サイズにしておきましょう。最初のうちは、コントラストがはっきりして見やすい `Monokai` テーマなどがおすすめです。
- **日本語表示の確認**  
  メニューやファイル名など、日本語が正しく表示されているか確認します。
- **ターミナルのシェル**  
  macOS であれば `zsh` など、ふだん使っているシェルが選ばれているかを確認しておくと安心です。

### コラム：ちょっとだけ Unix シェル

本書の内容は、基本的にはマウスやキーボード操作だけで完走できますが、長い目で見ると「シェル」と呼ばれるコマンドライン環境にも少しずつ慣れておくと役に立ちます。

macOS や Linux では、標準で Unix 系のシェル（`zsh` や `bash` など）が用意されていて、`ls` や `cd` のようなコマンドでファイル操作ができます。クラウドサーバやレンタルサーバでも同じコマンドがそのまま通用することが多く、「一度覚えるといろいろな場面で使い回せる」知識です。

Windows の場合も、PowerShell やコマンドプロンプトだけでなく、WSL（Windows Subsystem for Linux）や Git Bash などを使うことで、Unix 系のシェルに近い環境を手元に用意できます。将来クラウド開発などに進んでいきたい方は、少しずつ Unix 系のコマンドにも触れておくとよいでしょう。

この本の本文では、シェルの細かい使い方までは立ち入りませんが、「ちょっと試してみたい」という方のために、巻末に簡単な入門編を付けてあります。興味が湧いたら、付録の「Unix シェル超ダイジェスト（97-unix.md）」を覗いてみてください。

### おすすめ拡張機能

Windsurf には、見た目や作業効率を高めてくれる拡張機能（Extensions）がいくつか用意されています。ここでは、最初に入れておくと便利な系統を三つだけご紹介いたします。

- **日本語化（Language Pack）**  
  メニューやエラーメッセージなどを日本語で表示できる言語パックがある場合は、インストールしておくと安心です。拡張機能パネルで`Japanese Language Pack for VS Code`と検索し、有効化しておきましょう。
- **ローカルプレビュー（Live Server）**  
  <span id="idx-5376xxg0dgl-41" class="index-term" data-yomi="えいちてぃーえむえる">HTML</span> ファイルを保存するたびにブラウザを自動でリロードしてくれる拡張機能です。Windsurf にも `Live Server` が用意されていますので、本書ではこれを導入してローカルプレビュー用として使っていくことにいたしましょう。
- **ファイルアイコンテーマ**  
  ファイルの種類ごとにアイコンを変えてくれる拡張機能です。<span id="idx-5376xxg0dgl-42" class="index-term" data-yomi="えいちてぃーえむえる">HTML</span> や <span id="idx-3ghtaxxv1hvr-68" class="index-term" data-yomi="しーえすえす">CSS</span>、<span id="idx-ufy6mn67o2id-39" class="index-term" data-yomi="じゃばすくりぷと">JavaScript</span>、画像ファイルなどが一目で判別しやすくなり、プロジェクトの全体像をつかみやすくなります。Windsurf で利用可能なアイコンテーマには`Material Icon Theme`などがあります。お好みのものをひとつ選んでおくとよいでしょう。

拡張機能は「便利そうだから全部入れる」というよりも、「いまの自分の作業スタイルに本当に役立ちそうかどうか」を基準に、少しずつ増やしていくのがおすすめです。

### プロジェクトフォルダを開く

Windsurf では、プロジェクトごとにフォルダをひとつ開き、その中のファイルを編集していきます。

1. メニューから「File > Open Folder...」を選びます。  
2. 作業用のフォルダ（例：`projects/studio-wave`）を選択します。  
3. 左側のエクスプローラに、そのフォルダ内のファイルやサブフォルダが一覧表示されます。

あとは、新規ファイルを作成したり、既存のファイルを開いたりしながら、<span id="idx-5376xxg0dgl-43" class="index-term" data-yomi="えいちてぃーえむえる">HTML</span> や <span id="idx-3ghtaxxv1hvr-69" class="index-term" data-yomi="しーえすえす">CSS</span>、<span id="idx-ufy6mn67o2id-40" class="index-term" data-yomi="じゃばすくりぷと">JavaScript</span> を編集していくことができます。

## セキュリティ重視のブラウザ<br>`Firefox（ファイヤフォックス）`
Firefox は、Chrome や Edge、Safari などと同じモダンブラウザのひとつですが、本書では**学習用ブラウザの主役**として Firefox をおすすめいたします。

その理由はいくつかあります。

- Web 標準仕様に忠実で、レイアウトや <span id="idx-3ghtaxxv1hvr-70" class="index-term" data-yomi="しーえすえす">CSS</span> の挙動を学びやすいこと  
- <span id="idx-3ghtaxxv1hvr-71" class="index-term" data-yomi="しーえすえす">CSS</span> グリッドや Flexbox の状態を視覚的に確認できる、学習者向きの開発者ツールを備えていること  
- Firefox を開発している Mozilla が Web 技術ドキュメントサイト **MDN Web Docs** を運営しており、エラーやプロパティ名から MDN にすぐたどり着きやすいこと  
- 追跡防止機能など、プライバシーとセキュリティを重視した設計になっていること  
- 最近のバージョンでは、Claude などの AI チャットサービスへの導線も用意されており、わからない点をその場で質問しやすいこと

`Firefox` 自体は、[公式サイト](https://www.mozilla.org/ja/firefox/new/) から無料でダウンロードできます。

<!-- ![](firefox_download.png) -->

「Firefox をダウンロード」ボタンを押すとダウンロードが始まるので、落としたファイルをダブルクリックしてインストールしてください。インストールが終わったら、Dock やスタートメニューに登録しておくと、Windsurf と並べて起動しやすくなります。

実際の開発現場では、Chrome や Edge、Safari など複数のブラウザで表示確認することが一般的ですが、本書では説明をわかりやすくするために、主に Firefox の画面を使って解説していきます。

## 基本的な開発方法

Windsurf と Firefox を導入できたので、ウェブサイトの基本的な作り方をご案内します。

0. 画面の左側に Windsurf を、右側に Firefox を配置します。
1. Windsurf で、プログラムをいろいろ書いていきます。
2. Firefox で、作ったウェブサイトの表示や動作を確認します。
3. 出来上がりが良ければ、ウェブサイトを公開して完了です。<br>
   もう少し改良したければ、1. に戻ります。


### 手順のご案内

作業ディレクトリの作成方法や、Windsurf や Firefox の簡単な使い方の紹介です。 ^106

^106: Mac利用者向けです。Windowsは適宜読み替えてください。

1. `Users/利用者名` ディレクトリの直下に `projects` ディレクトリを作成し、その中に今回の学習用フォルダとして `studio-wave` ディレクトリを作ります。`projects` はよく使うので、`Finder` のサイドバーに登録しておくと便利です。Web 関連のフォルダ名やファイル名は、`studio-wave` のようにハイフン（-）で区切る「ケバブケース」にしておくと、URL と相性がよく、本書で扱う他のファイル名とも統一しやすくなります。

2. Windsurf と Firefox のアイコンを Dock に登録しておくと、いつでもすぐ起動できて便利です。

3. Windsurf と Firefox を左右に並べて表示し、左に Windsurf、右に Firefox を配置しておくと、編集と表示確認を行き来しやすくなります。

4. Windsurf では、左側のエクスプローラに作業用フォルダを追加して管理します。メニューの「ファイル > フォルダをワークスペースに追加...」から、先ほど作成した `projects/studio-wave` フォルダを選びましょう。

5. `Finder` が開いたら、サイドバーから `projects` を選び、`studio-wave` フォルダを開きます。エクスプローラのツリービューで `studio-wave` を選び、右クリックから「新しいファイル」をクリックします。

6. 名前が空っぽのファイルが作られるので、`index.html` と入力してエンターキーを押します。左側のエクスプローラに `index.html` が現れるので、それをクリックします。すると右側の編集領域でいろいろ文字を入力して編集できるようになります。

7. `index.html` には、まず <span id="idx-5376xxg0dgl-44" class="index-term" data-yomi="えいちてぃーえむえる">HTML</span> の基本的な雛形を用意します。`html` と入力してエンターキーを押すと、Windsurf が雛形を自動挿入してくれます。あるいは、AI アシスタント（この本で登場する Cascade）に「<span id="idx-5376xxg0dgl-45" class="index-term" data-yomi="えいちてぃーえむえる">HTML</span> の基本的な雛形を書いてください」と依頼し、自動生成されたコードを貼り付けてもかまいません。

8. 挿入された雛形をベースに、見出しや本文を自分用の内容に書き換えていきます。色付きの表示（シンタックスハイライト）は、<span id="idx-5376xxg0dgl-46" class="index-term" data-yomi="えいちてぃーえむえる">HTML</span> の構造を分かりやすくするための機能です。

9. ひととおり書けたら、`Command + S` で `index.html` を保存します。

10. Firefox を開き、「ファイル > ファイルを開く」から `index.html` を選んで表示を確認します。

11. もし直したいところがあれば、Windsurf に戻って `index.html` を編集し、再度 `Command + S` で保存します。そのあと Firefox に戻り、「再読み込み」ボタンか `Command + R` で画面を更新して、修正結果を確認します。

### カーソル移動
|キーバインド|動作|意味|
|:--|:--|:-- |
|`Ctrl + F`|カーソルを右へ移動 |Forward 先へ進んで|
|`Ctrl + B`|カーソルを左へ移動|Backward 後方へ |
|`Ctrl + P`|カーソルを上へ移動|Previous 以前の行へ|
|`Ctrl + N`|カーソルを下へ移動|Next 次の行へ|
|`Ctrl + A`|カーソルを行頭へ移動    |Ahead 前方へ|
|`Ctrl + E`|カーソルを行末へ移動    |End 行末へ|

なお、`Command + Z` / `Command + X` / `Command + C` / `Command + V` は、Undo・切り取り・コピー・貼付を指を大きく動かさずに続けて行えるよう、キーボードの同じ段に横一列で並んだキーに割り当てられています。Z の文字自体に特別な意味があるというより、「X・C・V とセットで覚えるキー」として設計されている、と考えると覚えやすいでしょう。また、`Ctrl + H` は ASCII の「Backspace (BS)」制御文字、`Ctrl + D` は「End Of Transmission (EOT)」、`Ctrl + K` は「Vertical Tab (VT)」という制御文字に対応しており、Unix 系の環境ではその上に「左側削除」「右側削除」「行以降を削除」といった編集操作が重ねて割り当てられています。

なお、`Command + ←` と `Command + →` でも行頭・行末に移動できます。

Windsurf の画面上部には、統合ターミナルや画面分割のためのアイコンも用意されています。これらはまずはアイコンをクリックして操作し、慣れてきたらコマンドパレットからショートカットを検索してみる、という進め方でも構いません。

複数の場所を同時に編集したいときは、行番号のあたりを `Option + クリック` すると、標準設定ではマルチカーソルを使うことができます。Atom キーバインドを入れている環境では、`Command + クリック` に割り当てられている場合もあります。

## Firefox の基本操作

### ページのオープンと再読み込み

|`Command + O`|ローカルの <span id="idx-5376xxg0dgl-47" class="index-term" data-yomi="えいちてぃーえむえる">HTML</span> ファイルを開く|Open|

### Live Server で localhost を開く

Windsurf 側で Live Server（開発用の簡易 Web サーバー）を起動すると、ブラウザから `http://localhost:ポート番号/` にアクセスしてページを確認できます。
一度 Firefox で `localhost` を開いておけば、そのあとは <span id="idx-5376xxg0dgl-48" class="index-term" data-yomi="えいちてぃーえむえる">HTML</span> を保存してから `Command + R`（必要に応じて `Shift + Command + R`）で、変更が反映された画面をすぐに確認できます。

## よくあるつまずき Q&A

### Q. <span id="idx-5376xxg0dgl-50" class="index-term" data-yomi="えいちてぃーえむえる">HTML</span> を保存したのに、Firefox の画面が変わりません。

A. 次の点を順番に確認してみてください。

1. Windsurf でファイルを本当に保存できているか、`Command + S` をもう一度押してみます。
2. Firefox のアドレスバーが `file:///` から始まるパスなのか、`http://localhost:` から始まる URL なのかを確認します。Live Server を使っているときは、`localhost` 側を更新する必要があります。
3. `Command + R` で再読み込みしても変わらない場合は、`Shift + Command + R` でスーパーリロードを試してみます。

それでも変わらないときは、<span id="idx-5376xxg0dgl-51" class="index-term" data-yomi="えいちてぃーえむえる">HTML</span> ファイルを別の場所に保存していないか、もう一度パスを見直してみましょう。

### Q. Live Server で開いたページが真っ白です。

A. まずは Windsurf 側で Live Server が起動しているかを確認します。ステータスバーにポート番号が表示されていないときは、もう一度コマンドパレットから Live Server を起動してみてください。

それでも真っ白な場合は、

- URL が `http://localhost:ポート番号/` になっているか
- <span id="idx-5376xxg0dgl-52" class="index-term" data-yomi="えいちてぃーえむえる">HTML</span> ファイルにまだ何も書かれていないだけではないか

といった基本的なところを一緒に確認してみると、原因が見つかることが多いです。

### Q. 日本語入力が思ったように切り替わりません。

A. この章で紹介したように、`Ctrl + Space` は日本語入力と英数入力を切り替えるためのショートカットです。
もし効きが悪いと感じたら、macOS の「システム設定」→「キーボード」→「キーボードショートカット」で、入力ソース関連のショートカットが他のアプリと競合していないかも確認してみてください。

慣れるまでは、画面右上のメニューバーから現在の入力モードを目で確認しつつ、`Ctrl + Space` を押して切り替える、という二段構えでもかまいません。
