## ウェブサイト作成の為の<br>三つの言語

ウェブサイトを作成する為に用いられる言語には、大きく次の三つがあります。

* `HTML（HyperText Markup Language ハイパーテキストマークアップランゲージ）` <br>
  　HyperText Markup Language を日本語で表すなら、「ハイパーテキストに目印をつける言語」くらいの意味になります。<br>
  　目印をつける(Markup)というのは、文書の各部分（見出し・段落・表・リストなど)が果たしている役割が分かるようにすることです。そのために専用の「タグ」を使います。 <br>
  　コンピュータに理解できるよう文書構造を定義することが、HTMLの最も重要な役割です。

* `CSS（Cascading Style Sheet カスケーディング・スタイルシート）` <br>
  　ウェブサイトの見た目を飾り付け、彩る為の言語です。文字の大きさや色の指定から画像の配置など、様々な装飾を行うことができます。<br>
  　HTMLのタグが親子関係(包含関係)にある場合、多くの設定値は親要素に指定されたものが子要素、孫要素に引き継がれていきます。このように設定値が上から下へ伝播（でんぱ）していく様子を、階段状の瀧を意味する cascade（カスケード）になぞらえて、命名されました。

* `JavaScript（ジャバスクリプト）` <br>
  　ウェブサイトに、「双方向性・相互作用性（インタラクティブ）」を{齎|もたら}すために用いられるプログラミング言語です。 <br>
  　例えば、閲覧者の操作に反応して表示が書き換わったり、ページが表示される際に写真などの要素に動きや効果を加えたり、サーバと通信してデータを取得したりするなど、現在のウェブサイトには欠かせないプログラミング言語となっています。

日々更新される各言語の詳細を{知悉|ちしつ}することは大変ですが、良く使う基本的なコードのみで十分にウェブサイトを作成することが出来ます。こつこつ学んでいきましょう。

## HTMLの基本

テキストエディタ `Windsurf`（ウィンドサーフ） の使い方をご説明する過程で、すでに以下のようなHTMLを作成しています。[^1]とても短いコードですが、HTMLの基本が詰まっていますので、ご説明いたします。

[^1]:生成AIの出力結果は実行する度に異なります。出力されたHTMLの内容は異なっていても構いません。

**▼index.html**

```html
<!DOCTYPE html>
<html>
  <head>
    <meta charset="utf-8">
    <title>HTML学習</title>
  </head>
  <body>
    <h1>初めてのHTML</h1>
    <p>
      HTMLを学習して、素敵なサイトを作れるように 成ります。
    </p>
  </body>
</html>
```

### ファイル名について

ファイル名を、`index.html` としました。
`index`は、「索引」「指針」「指標」などの意味を持つ英単語です。
ウェブサイトにアクセスした際、特にページの指定がされなければ、ブラウザは `index.html`を表示します。例えば、`https://example.com/` にアクセスすると、ブラウザは`https://example.com/index.html`のファイルを表示します。

`.html`は、**拡張子**と呼ばれます。ファイルの種類が `HTML` であることを示しています。
拡張子が `.html`であるファイルを開くと、**OS**により既定のブラウザが開きます。


### DOCTYPE

HTML文書の先頭に書かれている `<!DOCTYPE html>` は、**DOCTYPE宣言**と呼ばれます。昔の HTML では、どの時代の仕様に従って書かれている文書なのかを宣言し、その内容に応じてブラウザが表示方法（いわゆる「互換表示モード」か「標準モード」か）を切り替えるために使われていました。その名残として、今の HTML でも、最初の 1 行に `<!DOCTYPE html>` と書いておくことが「お作法」になっています。

現在の HTML の仕様は、かつての **HTML5** から受け継がれたあと、WHATWG という団体が策定する **HTML Living Standard（HTML リビングスタンダード）** へと一本化されています。`<!DOCTYPE html>` 自体の書き方は変わっておらず、「この文書は最新の HTML（HTML Living Standard）に従って書かれていますよ」とブラウザに伝える印のようなものだと考えておけば十分です。


### `<html>` と `<head>` と `<body>` について

先ほど見たシンプルな HTML では、全体が `<html>...</html>` で包まれていました。この `<html>` の中身は、大きく二つの部屋、`<head>` と `<body>` に分かれています。

`<head>` は、「このページがどんなページなのか」をブラウザに伝えるための設定を書く場所です。今回のサンプルでは、次のような情報が入っていました。

* 文字コードとして、`utf-8` を使用すること (`<meta charset="utf-8">`)
* 文書の表題が「HTML学習」であること (`<title>HTML学習</title>`)

実際のサイトでは、このほかにも検索エンジン向けの説明文や、後の章で登場する CSS ファイルへのリンクなど、ページの「裏側のプロフィール」のような情報をここに集めていきます。多くは画面には直接表示されませんが、ページ全体のふるまいをそっと支えてくれる、舞台裏のような場所です。

一方の `<body>` は、読者の目に見えるコンテンツ（内容・出し物）を書くための部屋です。今回の例では、次のような要素が入っていました。

* 大見出し（`<h1>`）に、「初めてのHTML」
* 段落（`<p>`）に、「HTMLを学習して……」という本文

ここに、これから学んでいく画像やリンク、ボタン、記事一覧などを少しずつ増やしていくことで、ひとつのウェブページが形になっていきます。「設定を書く `<head>` と、見える部分を書く `<body>`」という役割の違いを、まずはやさしく押さえておきましょう。


:::{.note}
**文字コードと「文字化け」について**

かつてコンピュータが英数字しか扱えなかった時代、アルファベットの「`A`」は十六進数の`41`(十進数の`65`)、「`B`」は`42`と番号を割り振っていました。これを ASCII（アスキー）コードと言います。

その後、日本語を扱えるように拡張され、例えば「`日`」は 十六進数の `93 FA`と番号付けされました。これを `Shift JIS`（シフトジス）コードと言います。

Shift JIS コードは長らく使われてきましたが、現在ではほぼ世界中の文字を扱えるようになった `UTF-8`（ユーティーエフエイト） が標準となっており、例えば「`日`」には `E6 97 A5` が割り振られています。

同じ「`日`」という文字ですが、割り当てられている文字コードが違うため、古くからあるウェブサイトでは `Shift JIS` の `93 FA` が、新しく作られたウェブサイトでは UTF-8 の `E6 97 A5` という文字コードで書かれています。

そのため、古いサイトを閲覧した際などに、正しく文字が読めないことが起こります。これがいわゆる「文字化け」です。

`<meta charset="utf-8">` は、今から作成するサイトが `UTF-8` で書かれていることを示すコードで、文字化けを防ぐ大事な役割を担っています。

:::

### 要素とは

:::{.sideimage-right}
![](html11.png)

`HTML`では、**コンテンツ(内容)** を **開始タグ** と **終了タグ** で **マークアップ** (印付け)することにより、その種類を明確にします。マークアップした部分は左のような構造になり、全体を要素と呼びます。

この例では、開始タグ`<h1>`と、終了タグ`</h1>`で囲まれた範囲「初めてのHTML」が「大見出し」であることを示しています。

人が見ると、「見出し」が「始めてのHTML」、「本文」が「HTMLを学習して・・・」であることは一目瞭然ですが、コンピュータにとってそれが何であるかは自明ではありません。そこで、`<h1>始めてのHTML</h1>`と書くことで、「始めてのHTML」が「大見出し」であることを「印付け（マークアップ）」します。

それが何であるかを示す適切なタグを使うことが、良いウェブサイト作りの第一歩です。
全部で100以上のタグが用意されていますが、良く用いるタグは十数個です。少しずつ学んでいきましょう。
:::

### 親要素と子要素

:::{.sideimage-right}
![](html14.png)

正しく入れ子の形で記述すると、階層構造が作られます。このとき、上位階層の要素を「親要素」、下位階層の要素を「子要素」と呼びます。

例えば、`<body>` は `<h1>` の親要素、 `<h1>` は `<body>` の子要素です。
:::

### コメント(覚書・説明書き)

:::{.sideimage-right}
![](comment.png)

ソースコードを分かり易くするための覚書、説明書きが出来るよう、**コメント** と呼ばれる構文があります。

作成したウェブサイトを閲覧する際にはないものとして扱われますので、文法上、絶対に書かなければならないものではありませんが、適切なコメントを書くことにより、自らの助けともなります。
:::

また、不要となったコードを一時的に非表示にしたり、参考となるコードを残しておく為にも使うことが出来ます。

`<!--  -->` の間にコメントを書いていきますが、Windsurf など ほとんどのテキストエディタでは、`command + /` (コマンドキーと `/`キーを同時に押す)でコメントにしたり、もう一度押すとコメントを解除できます。エディタの支援機能を活用して、効率的にコーディングしていきましょう。

### 字下げ(インデント)

HTMLでは、空白や改行は意味を成さない為、任意に字下げや改行が行えます。そして、タグの親子関係(入れ子関係)が分かりやすいよう、綺麗に字下げ(インデント)することを心がけましょう。
自分が書きたいことを明確にできるとともに、コードの不具合も発見しやすくなります。

右は iPhone で見たときの、次頁は iPad や Mac で見たときの完成形イメージです。表示している文字や写真は同じですが、画面幅に応じて配置を変えている点にも注目してください。

iPhone、iPad、Mac と、それぞれ閲覧する端末に適した配置となるようにウェブサイトを作ることを、**レスポンシブデザイン（レスポンシブウェブデザイン）** と呼びます。

## HTMLを記述する

作成したいウェブサイトのイメージができたので、実際に HTML を書いて、`wave-studio` のトップページを作り始めていきましょう。


### ビューポート

作成した`index.html`を編集し、次のように更新しましょう。

**▼index.html**

```html
<!DOCTYPE html>
<html>
  <!-- サイトに関する設定事項を記述します -->
  <head>
    <!-- 文字コードの指定 -->
    <meta charset="utf-8">

    <!-- ページのタイトル -->
    <title>WAVE</title>

    <!-- ビューポート(視点)の指定をすると、iPhone でも きれいに見せることができる -->
    <meta name="viewport" content="width=device-width, initial-scale=1">
  </head>

  <!-- 表示したい内容・出し物・コンテンツを記述します -->
  <body>

  </body>
</html>
```

HTMLの文法で解説したように、分かりやすくするために `<!-- -->` で囲まれた「コメント」を入れています。  [^109]

ページのタイトルを、

```html
<title>WAVE</title>
```

としましたので、ブラウザのタブに「WAVE」と表示されます。

[^109]: 作成例は説明の為に多くのコメントを記入していますが、冗長だと感じる方もいらっしゃることでしょう。ご自身の読みやすさに合わせて、適宜、調整されてください。


ここで、着目したいのは次の記述です。

```html
<meta name="viewport" content="width=device-width">
```

先に述べたように、iPhone で見る人が多いので、この短いコードを入れておくことで、後々、それぞれの端末の表示幅に応じた配置ができるようになります。

### サイト名

HTML は、ウェブサイトの「文書構造」を記述するための言語です。そして、サイトに表示する文字が、「見出し」なのか「本文」なのかなど、それぞれの要素に適した「タグ」を使って「印付け（マークアップ）」を行うことが、肝要でした。

ウェブサイトの完成図を見ると、いくつかの部品に分けられそうです。それぞれの部品に適した「印付け（マークアップ）」を行っていきましょう。

<!-- ![](header.png) -->

まずは、サイト名から作っていきましょう。

サイト名を表すための専用のタグはありませんが、`<header>` タグを使って書くのが良さそうです。現在、空っぽになっている `<body>` タグの中に書き加えて、次のようにしましょう。

**▼index.html**

```html
<!-- 表示したい内容・出し物・コンテンツを記述します -->
<body>
  <!-- ヘッダー(サイト名) -->
  <header>WAVE</header>
</body>
```

#### 入力支援機能の活用

`<header>WAVE</header>` をすべて手入力してもよいですが、Windsurf には HTML タグを楽に入力できる「入力支援機能」が備わっています。
`he` と入力すると、入力したいタグの候補が表示されます。また暫く待つとさらに補完されたコードも提示されます。補完されたコードをそのまま使用する場合は、`Tab` キーを押すと、`<header>` タグは自動で完成します。逆に補完されたコードが気に入らない場合には、エスケープキーを押して、自分で入力することができます。

このような入力支援機能を活用することで、タイピングミスを減らしながら効率的にコーディングを進められます。

:::{.note}
**ピクセル数について**

スマートフォンの画面サイズを説明するとき、「横 375px × 縦 667px」といった表現をよく目にします。ここでの `px`（ピクセル）とは何でしょうか？
「IT用語辞典」に分かりやすい説明がございますので、引用して紹介いたします。

> ピクセルとは、デジタル画像や画面などを構成する最小単位である、色のついた微細な点のこと。また、その数を表す単位。単位を表す場合は “px” と略記されることもある。

たとえば iPhone SE（第2世代／第3世代）などでは、ブラウザから見た画面サイズは横 375px・縦 667px 程度です。横に 375 個、縦に 667 個のピクセルを並べたキャンバスに、ページが描かれているイメージだと考えると分かりやすいでしょう。
実際の端末には、より木目細かく見えるように 2 倍〜3 倍の画素が搭載されており、このような高精細ディスプレイは Retina（レティナ）ディスプレイと呼ばれます。

:::

### ヘッダーにリンクを追加する

先ほどは、サイト名をただ表示するだけのヘッダーでしたが、実際のサイトでは「ロゴをクリックするとトップページに戻れる」ようにしておくのが定番です。`wave-studio` でも、画面左上の「WAVE」を押すといつでもトップに戻れるようにしておきましょう。

```html
<!-- ヘッダー(サイト名) -->
<header><a href="index.html">WAVE</a></header>
```

`<a href="index.html">...</a>` の部分が、「この文字をクリックすると index.html に飛びますよ」という意味になります。`index.html` はいま作っているトップページ自身ですから、どのページにいてもロゴを押せば安心して帰って来られる“港”のような存在になります。


### ナビゲーションメニュー

```html
<!-- ナビゲーションメニュー -->
<nav>
  <ul>
    <li><a href="#recent">最新記事</a></li>
    <li><a href="about.html">サイトについて</a></li>
    <li><a href="contact.html">お問い合わせ</a></li>
  </ul>
</nav>
```

ナビゲーションメニューの 3 つの項目には、それぞれ小さな「行き先」が仕込まれています。どれも `<a>...</a>` というタグで囲まれていて、`a` は *anchor（アンカー）* の略です。「ここをクリック（タップ）すると、どこか別の場所へ飛びますよ」という“碇（いかり）”のような役割を持つ要素だ、とイメージしてみてください。

リンクでもっとも大事なのが `href` 属性です。ここに「飛び先（URL）」を書き込むことで、リンクがはじめて意味を持ちます。

- `href="#recent"`  
  同じページの中にある `id="recent"` の場所へジャンプする「ページ内リンク」です。のちほど登場する `<section id="recent">` と組み合わせることで、ナビゲーションの「最新記事」を押した瞬間に、画面下の最新記事一覧まで一気に視線を連れて行くことができます。
- `href="about.html"`  
  同じフォルダにある `about.html` へ飛ぶリンクです。
- `href="contact.html"`  
  同じフォルダにある `contact.html` へ飛ぶリンクです。

`about.html` や `contact.html` の中身は、まだこれから作っていきます。この章では「将来ここに自己紹介やお問い合わせのページがぶら下がるんだな」という、ちょっとした予告編のような気持ちで眺めておいてください。

### ページの本編を包む `<main>`

```html
<!-- 主要なコンテンツ -->
<main>
  <!-- ヒーローイメージ -->
  <figure class="hero">
    <img src="images/blue_sea_1920x1280.webp"
         alt="爽やかな青い空と海">
  </figure>

  <!-- キャッチフレーズ -->
  <h1 class="catch_phrase">
    Best place to visit in the world
  </h1>

  <!-- サブタイトル -->
  <p class="sub_title">
    <img src="images/circle.svg" alt=""><br>
    <a href="#">爽やかな青い空と海</a>
  </p>

  <!-- 記事一覧 -->
  <section id="recent">
    <h2 class="title">最新記事</h2>
    <article>(略)</article>
  </section>
</main>
```

ここで新しく登場した `<main>` は、「このページでいちばん大事な中身が入っている場所」を示すためのタグです。ヘッダーやナビゲーション、フッターといった「まわりの飾り」ではなく、そのページならではの本文や記事、写真などを、ひとまとめに包むための役目を持っています。

`wave-studio` では、トップページの主役になるのは、海辺のヒーローイメージとキャッチフレーズ、そして最新記事の一覧です。そこで、これらをひとかたまりの「主要なコンテンツ」と見なし、`<main>...</main>` の中にそっと収めています。逆に言うと、サイト名のヘッダーやナビゲーションメニュー、ページ最下部のフッターなどは、たとえどのページにも現れる大事な存在ではあっても、`<main>` の外側に置くのがお作法です。

このように、ページ全体を大づかみに眺めたときに、「ここからここまでがこのページの本編だな」と思える部分を `<main>` で囲っておくと、人間にもブラウザにも分かりやすい構造になります。スクリーンリーダーなどの支援技術も、`<main>` を手がかりにして「本文の先頭」へ一気にジャンプできるようになるため、アクセシビリティの面でも小さくない助けになります。

この章では、`<main>` の中にどんな部品が入っているかをざっくり眺めるところまでにしておき、ヒーローイメージやキャッチフレーズ、最新記事のそれぞれについては、このあと少しずつ掘り下げていきます。

### ヒーローイメージ
「ヒーローイメージ」とは「サイトを印象づける大きな画像」のことです。

`wave-studio` では、爽やかな青い空と海の写真を、ページ最上部いっぱいに配置して、見る人に「ここは海辺のライフスタイルを紹介するサイトなんだ」と直感的に伝えることにしましょう。

写真そのものは、自分で撮りに行ってもよいですし、巻末の参考リンクにも掲載したように、多くのサイトで豊富な写真やイラストが提供されていますので、そちらを利用するのも便利です。

それでは、先ほどの`<main>`タグの中にコードを書いていきましょう。

**▼index.html**

```html
<main>
  <!-- ヒーローイメージ -->
  <figure class="hero">
    <img src="images/blue_sea_1920x1280.webp"
         alt="爽やかな青い空と海">
  </figure>
</main>
```

ここでの要点は、何といっても次の一行です。

**▼index.html**

```html
<img
  src="images/blue_sea_1920x1280.webp"
  alt="爽やかな青い空と海">
```

`<img>` は、画像を表示するために用意されているタグです。
先に登場した `<header>` タグと比べると、大きく二つの違いがあります。

:::{.notice}
**`<img>` と `<header>` の比較**

`<img src="images/blue_sea_1920x1280.webp" alt="爽やかな青い空と海">`<br>
`<header>WAVE</header>`

* **コンテンツ(内容物)**や **終了タグ`</img>**`がない。 <br>
　`<header>` タグの場合には、印付け(マークアップ)した「WAVE」の前後を `<header>` と `</header>` で囲んでいました。<br>
　`<img>`は、画像の表示を目的としているため、**開始タグ**のみで目的を達成できます。そのため`<header>`タグにはあった**コンテンツ(内容物、WAVE)**や**終了タグ**`</header>`がありません。<br>
このように、コンテンツを持たない要素のことを**空要素**と言います。

* `src`と`alt`の二つの**属性**を持っている。 <br>
  　**属性**は、**要素への追加情報を指定する**ためのもので、必須の属性と任意の属性があります。<br>
  　`src` 属性は、source(源泉)の略で表示したい画像の所在を与えます。これは必須の属性です。ここでは、`src="images/blue_sea_1920x1280.webp"` と書くことで、`images` ディレクトリ(フォルダ)の中にある海の写真を表示することを示しています。 <br>
  　もう一つの、`alt` 属性は、alternative({代替|だいたい})の略で何らかの理由で画像が見つからないときに表示する代替テキストを指定します。これも(一部の例外はありますが)ほぼ必須の属性です。目の見えない方が利用する音声読み上げソフトも、代替テキストを利用して画像の内容を伝えます。`alt`属性には、その画像に相応しいテキストを記述しましょう。

もう一つのポイントは、この`<img>`タグを囲むように書かれた `<figure>`タグです。

**▼index.html**

```html
<figure class="hero">
  (略)
</figure>
```
`<figure>`タグは、その内容物（コンテンツ）が、「図版」であることを示すために用いるタグです。写真の他にも、イラストや図、詩やプログラムコードなど、さまざまなものを含むことができます。

また`<figure>`タグを、単に `<figure>` と書くのではなく `<figure class="hero">` と、**属性(要素への追加情報)**を記述しています。`class`は**属性名**と呼ばれ、`hero`は**属性値**と呼ばれます。属性名`class`は、特に**クラス名**とも呼ばれます。

**クラス名**を付けると「特定の要素を他の要素から区別する、異なる要素を同じものとして取り扱う」ことが容易になります。

例えばこのサイトでは、いずれ複数の写真を掲載していきます。ヒーローイメージの写真に `class="hero"` と `hero` クラスを付与しておくと、多数の写真の中から「トップに大きく表示する 1 枚」を区別できるようになります。

どのように見えるのか、結果を確認したい方もいらっしゃるかもしれません。実際に確認してみると、写真が見切れていることが分かります。写真が大きく画面幅に全てを収めきれていないためですが、後ほど CSS を用いて見栄え良く出来ますので、今は先へ進むことと致しましょう。

:::{.note}
**画像形式について**

コンピュータで画像を扱うための主な形式には、**ラスター形式**と**ベクター形式**があります。大まかに言うと、写真のようなピクセルの集まりを扱うのがラスター画像、ロゴやアイコンのように線や図形で表現するのがベクター画像です。

### ラスター形式
* JPEG（ジェイペグ） - フルカラー・非可逆圧縮・写真用途
* PNG（ピング） - フルカラー・可逆圧縮・半透明化用途
* GIF（ジフ）  - 256色の図やイラスト・簡易アニメーション用途

とくに写真やイラストなどの「ピクセル画像（ラスター画像）」では、最近は WebP（ウェッピー）を使うと便利です。
WebP は JPEG / PNG / GIF を置き換えられる次世代の画像形式で、同じ画質ならファイルサイズを小さくできるため、表示も速くなります。

画像の形式を変換したいときは、`ImageMagick` [^113] などのツールを使うと簡単です。
WebP について詳しく知りたい方は、制作会社 ICS による解説記事 [^114] も参考になります。

ここまでは主にラスター画像について見てきましたが、ロゴやアイコンなど「どのサイズでもくっきり見せたい」ものには、次に紹介するベクター画像もよく使われます。

### ベクター形式
* SVG（エスブイジー） - ベクター形式の図やアイコン用途。HTML に似たタグのテキストで書かれており、拡大縮小しても画質が劣化しないという特徴があります。例えば `circle.svg` の中身は次のようになります。

```html
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 192 192">
  <style>
    /* 柔らかな RGB の三原色をイメージした半透明の円（白い枠線付き） */
    .decor-r {
      fill: rgb(255, 128, 128);
      fill-opacity: 0.25;
      stroke: #c001;	/* cool */
      stroke-width: 64;
    }
    .decor-g {
      fill: rgb(128, 255, 160);
      fill-opacity: 0.06125;
      stroke: #1dea;	/* idea */
      stroke-width: 2;
    }
    .decor-b {
      fill: rgb(128, 192, 255);
      fill-opacity: 0.25;
      stroke: #c0de;	/* code */
      stroke-width: 1;
    }
  </style>
  <circle  class="decor-r" cx="117.3" cy="133.7" r =" 31.50" />
  <ellipse class="decor-g" cx="084"   cy="46.49" rx="077.34" ry="14.106" />
  <ellipse class="decor-b" cx=" 75.8" cy="37.3"  rx=" 80.80" ry="25.25" transform="rotate(22.3 37 76)"/>
</svg>
```

ここでは、円の中心座標を表す `cx`（x 座標）、 `cy`（y 座標）と、半径 `r` を数値で指定して円を描いています。ピクセルを並べるのではなく「円」という図形として記録しているため、何倍に拡大してもきれいな円のままです。

:::

### キャッチフレーズ

![](h1.png)

それでは、続いてキャッチフレーズのコーディングに移りましょう。

`wave-studio` では、海辺のライフスタイルを紹介するサイトらしく、英語のキャッチコピーを大きく配置します。

それでは、ヒーローイメージに続けて、次のようにコードを書きましょう。

**▼index.html**

```html
<!-- ヒーローイメージ（サイトを印象づける大きな画像） -->
<figure class="hero">
  <img
    src="images/blue_sea_1920x1280.webp"
    alt="爽やかな青い空と海">
</figure>

<!-- キャッチフレーズ -->
<h1 class="catch_phrase">
  Best place to visit in the world
</h1>
```

![](h1-non-css.png)

`<h1>` は `Heading（ヘディング）(見出し)` に由来するタグで、「大見出し」であることを示しています。 `<h2>` は「中見出し」、`<h3>` は「小見出し」と順に小さくなり、`<h6>` までが用意されています。一般的には、そのページ全体の題名を表す要素として、`<h1>` は 1 ページに 1 つだけ使うのが基本です。

また、後から CSS での指定がしやすいよう、`<h1 class="catch_phrase">` とクラス名を付けています。このようにクラス名を付けておくと、スタイルシート側では `h1.catch_phrase { ... }` のように指定でき、「ページの中でもこのキャッチフレーズだけ」にピンポイントでデザインを適用できます。後でサイト全体のデザインを調整したくなったときも、HTML の構造はそのままに、`catch_phrase` 用のスタイルだけを書き換えれば見た目をまとめて変更できるので、保守もしやすくなります。また、`catch_phrase` のように「その要素がページの中でどんな役割を持っているか」がすぐに分かる名前にしておくと、HTML や CSS を後から読み返したときにも迷いにくくなります。

### サブタイトルと装飾画像

:::{.sideimage-right}
![](link_button.png)

続いては、`wave-studio` のサブタイトルと装飾画像を配置しましょう。完成形のイメージは右のようになります。

青色のグラデーションなど、見た目の装飾は CSS の章でじっくり扱いますので、ここでは HTML 側の「文書構造」を整えることに専念します。

それでは、キャッチフレーズのすぐ下に、次のようなコードを書き足しましょう。

**▼index.html**

```html
<!-- キャッチフレーズ -->
<h1 class="catch-phrase">

</h1>

  <img src="images/circle.svg" alt=""><br>

</p>
```

ここで登場した `<p>` タグは、`paragraph（パラグラフ）` (段落) を表すためのタグです。通常の文章などを記述する際に広く用いることができるタグです。

`<p>` タグの中には、装飾用の小さな画像 `<img>` と、リンク用の `<a>` タグが含まれています。`<br>` タグは改行で、「装飾用の画像」と「リンク文字列（爽やかな青い空と海）」を別の行に分けるために使っています。

また、`<a>` タグの `href` 属性は、リンク先の URL を指定します。ここでは `href="#"` と書いており、`#` は「とりあえずどこにも飛ばさない」という意味のプレースホルダー（仮のリンク先）として使っています。後の章で実際のリンク先が決まったら、`href="post01.html"` のように、ここを書き換えて本物の URL を指定していきます。

#### Git（ギット） / GitHub（ギットハブ）

学習用のコードでは、まだリンク先を決めていない段階のため、ひとまず `href="#"` と書いて「どこにも飛ばさない仮のリンク」としておきました。実際に公開している `wave-studio` のページでは、このボタンが GitHub へつながる本物のリンクになっており、クリック（タップ）するとソースコード一式を置いてあるページが開きます。完成形のサイトでは「このボタン = ソースコード置き場への入り口」と覚えておくとよいでしょう。

ここで登場する **Git（ギット）** は、ファイルの変更の履歴をこつこつと記録してくれる「分散型バージョン管理システム」と呼ばれる道具です。作業の区切りごとにコミットと呼ばれるしおりを挟んでおくことで、「昨日の状態に戻してみる」「どの変更で不具合が入り込んだのかをたどる」といったことが、あとから落ち着いて行えるようになります。

一方の **GitHub（ギットハブ）** は、その Git のリポジトリ（履歴つきの作業フォルダ）をインターネット上に預かってくれるサービス（Web サイト）です。自分ひとりのバックアップ場所として使うこともできますし、チームメンバーとリポジトリを共有して、Pull Request（プルリクエスト）や Issue（イシュー）といった仕組みを通じて、少しずつ相談しながら共同開発を進めることもできます。今回の `wave-studio` では、この GitHub 上に HTML や CSS のコード一式を公開しており、先ほどのリンクボタンを押すと、そのリポジトリのページがすっと開くようになっています。

それでは、実際にリンク先の [https://github.com/Atelier-Mirai/wave-studio](https://github.com/Atelier-Mirai/wave-studio) にも、そっと足を運んでみましょう。

![](github.png)

![](github_download.png)

`Git（ギット）` や `GitHub（ギットハブ）` は、ソースコードのバージョン管理や、コードの公開・共有を助けてくれる、開発に欠かせないサービスです。自分の書いたコードを GitHub に公開しておけば、今回の `wave-studio` のように、他の人にもソースコード一式を配布したり、複数人で共同開発したりしやすくなります。

### 最新記事セクション

![](haiku.png)

続いて、トップページの中央に並ぶ「最新記事」のカードを作っていきましょう。


「最新記事」という見出しの下に、

* 記事のサムネイル画像
* 記事タイトル

がセットになったカードを、いくつか横並び（または縦並び）で配置していきます。

写真の載せ方は、`<figure>` タグや `<img>` タグを用いれば良いと、すでに学習しています。タイトルには `<h3>` を使い、画像とタイトルを 1 セットの「記事」として扱いたいので、それらを包む箱として HTML に用意されている `<article>` タグを使うことにしましょう。こうしておくと、「この画像とタイトルは 1 つのまとまりである」ということが、ブラウザにも読む人にもはっきり伝わります。

また、複数の記事をひとまとめにした「節」として、`<section（セクション）>` タグも用意されています。書籍や文章の「節」という意味の英単語が由来で、ウェブサイトの区分・区間として用いることができますので、使っていましょう。

**▼index.html**

```html
<!-- サブタイトル -->
<p class="sub_title">
  <img src="images/circle.svg" alt=""><br>
```

#### class 属性と id 属性

HTML では、`class="..."` や `id="..."` のように、タグに名前と値のペアで「属性」を付け加えることで、意味や役割をより詳しく指定できます。

`class` 属性は「グループ名」のようなもので、同じクラス名を複数の要素に付けられます。例えば、このページでは `<h1 class="catch_phrase">` や `<p class="sub_title">` と書き、後から CSS で `.catch_phrase` や `.sub_title` にまとめて装飾を指定します。同じクラス名を付けておけば、別の場所でも同じスタイルを簡単に再利用できます。

一方、`id` 属性はページの中で 1 つだけに付ける「固有の名前」です。ここで作った `<section id="recent">` のように書いておくと、CSS や JavaScript から `#recent` という目印でそのブロックを指し示せますし、ナビゲーションメニューから `#recent` へジャンプするリンクも作れます。
:::

### フッター

長い HTML の旅もいよいよ最後です。フッターを作っていきましょう。フッターは、文書の最後に置かれる定型情報で、著作権情報や連絡先などが良く記されます。
ヘッダー(サイト名)を表示する際に `<header>` タグを用いましたので、`<footer>` タグは、対になる存在として覚えやすいと思います。

`wave-studio` では、会社名や住所、電話番号、メールアドレスといった連絡先と、サイト全体の著作権表記をフッターにまとめて記載します。ここでは、実際のウェブサイトでよく見かける形に近づけるために、これらの情報を `<address>` 要素とクラス付きの `<p>` 要素で丁寧に整理してみましょう。

**▼index.html**

```html
<!-- フッター -->
<footer>
  <!-- 連絡先 -->
  <address>
  <!-- 会社名 -->
    <p class="company_name">(有) WAVE商会</p>
    <!-- 住所 -->
    <p class="address">名古屋市美波区美波町三丁目七番三号</p>
    <!-- 電話番号 -->
    <p class="tel">
      <a href="tel:052-373-4649">
        052-373-4649
      </a>
    </p>
    <!-- メールアドレス -->
    <p class="email">
      <a href="mailto:contact@wave.example.jp">
        contact@wave.example.jp
      </a>
    </p>
  </address>
  <!-- 著作権表示 -->
  <p class="copyright">
    Copyright © 令和七年 WAVE All rights reserved.
  </p>
</footer>
```

`<address>` タグは、「このサイトやこのページの連絡先情報」をひとまとまりにするための要素です。会社名や所在地、電話番号、問い合わせ用のメールアドレスなどを、この中にまとめて記述しておくと、「ここに書いてあるのは連絡先ですよ」ということが、ブラウザや検索エンジンにも分かりやすく伝わります。

電話番号のリンクに書かれている `tel:` や、メールアドレスのリンクに書かれている `mailto:` は、それぞれ「電話アプリでこの番号を開く」「メールアプリでこの宛先に新規メールを作成する」といった動きを指定するための特別な書き方です。特にスマートフォンでは、電話番号リンクをタップするとそのまま電話アプリが起動したり、メールリンクをタップするとメール作成画面が開いたりするので、連絡先を素早く呼び出せるようになります。

また、開発者ツールの画面上部には、様々な端末に切り替えられるよう、多くの端末が登録されています。クリックしてみましょう。

iPad や iPhone など様々な端末を選ぶことができます。`iPhone SE 2nd gen iOS 14.6` を選ぶと、端末の画面サイズ、横 375px 縦 667 px が表示されています。その右側にある iPhone のマークをクリックすると、横向きにした際の表示も確認できます。

ウェブサイトの作成は、モバイルファーストで進めていきますので、このまま iPhone モードにしておきましょう。以降の章でも、まずはこのモードでレイアウトを整え、そのあとで iPad や Mac 向けの調整を重ねていきます。


## この章のまとめ

ここまでで、`wave-studio` トップページの HTML を一通り組み上げました。

- `<!DOCTYPE html>` から `<html>`, `<head>`, `<body>` まで、HTML 文書の基本的な枠組みを確認しました。
- `<header>`, `<footer>`, `<section>`, `<article>`, `<figure>`, `<h1>`〜`<h3>`, `<p>`, `<a>` などを使って、サイト全体の「骨組み」と「見出し・段落・リンク」の関係を表現しました。
- 画像の `alt` 属性やリンクの `href` 属性、`class` / `id` 属性といった「属性」の役割を押さえ、あとから CSS や JavaScript で扱いやすい印付けのしかたを体験しました。
- ヒーローイメージや最新記事カードのように、「意味のあるまとまり」を HTML のタグで素直に表現しておくと、後のデザイン調整が楽になることも確認しました。

100 行弱のコードでも、ひとつひとつのタグの意味と役割をたどっていけば、十分に読み解けることを実感していただけたと思います。

## コラム：次は CSS でレイアウトの衣装をまとわせる

この章で書いてきた HTML は、いわば「骨組み」と「テキストの内容」をきちんと並べた状態です。まだ余白や色づかいは素朴かもしれませんが、ヘッダー・ヒーローイメージ・最新記事・フッターといった区切りは、すでに HTML のうえでできあがっています。

次の章からは、この骨組みに CSS を重ねていきます。文字の大きさや色を変えたり、余白を整えたり、画像とテキストの配置をそろえたりしながら、「海辺のライフスタイル誌らしい」レイアウトに近づけていきましょう。

CSS では、フォントや色だけでなく、グリッドレイアウトといったレイアウトの仕組みも使えるようになります。ちょうど雑誌の紙面レイアウトを考えるときのように、縦横2次元のマス目の上に文章や写真を自由に配置しながら、「海辺のライフスタイル誌」らしいページ構成を試せるのが、とても楽しいところです。同じ HTML でも、スタイルを変えるだけで驚くほど雰囲気が変わる様子を、これから一緒に体験していきます。
「タグの意味を考えて素直にマークアップする」という、この章で身につけた癖は、そのまま CSS からも扱いやすい HTML づくりにつながります。次の章では、その HTML をキャンバスに見立てて、CSS という絵の具で少しずつ色とリズムを乗せていきましょう。
:::
