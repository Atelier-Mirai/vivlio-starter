# 開発環境の準備

:::{.chapter-lead}
プログラミングを始めるにあたり、必要となる開発環境を整えていきます。
エディタとして「Windsurf」を、ブラウザとして「Firefox」をご紹介します。  
:::


プログラミングを行うために整える道具は二つ、「エディタ」と「ブラウザ」です。
（もちろん「コンピュータ」も必要です。iPadやiPhoneでプログラミングできる環境として [cloud9](https://aws.amazon.com/jp/cloud9/) や [paiza](https://paiza.io/ja)、[CodePen](https://codepen.io) などもありますが、Mac や Linux、Windows が動くデスクトップやラップトップコンピュータがお勧めです。）

エディタとは、プログラミングが快適に行えるよう様々な支援機能を備えたコーディングのためのソフトウェアです。さまざまなエディタがありますが、本書では `Windsurf（ウィンドサーフ）` をおすすめのエディタとしてご紹介いたします。

「ブラウザ」は、ウェブサイトを閲覧するためのソフトウェアです。自分の書いたコードの詳細を確認する為の「開発者ツール」や、iPhone や iPad など異なる端末での表示結果を確認する機能も備わっています。さまざまなブラウザがありますが、ここでは `Firefox（ファイヤフォックス）` をご紹介いたします。


他にも、ソースコードのバージョン管理システム(Git, GitHub)や、画像編集ソフト(Affinity Photo, Affinity Designなど)など、サイト作成の為の様々なツールがございますので、適宜学習されて下さい。

## AI支援のモダンなエディタ `Windsurf（ウィンドサーフ）`

Windsurf は、AI アシスタントを組み込んだモダンなコードエディタです。VS Code に近い操作感を持ちながら、エディタ内で AI ペアプロ（この本で登場する Cascade）をそのまま使えるのが大きな特長です。

- 複数ファイルにまたがる検索やリファクタリング
- 内蔵ターミナルから `vs build` などのコマンド実行
- Git 連携や差分表示
- Markdown / HTML / CSS / JavaScript などのシンタックスハイライト
- AI によるコード補完や文章のリライト

といった機能が、最初からひとまとめに用意されています。

### ダウンロードとインストール

1. **公式サイトを開く**  
   `https://codeium.com/windsurf` などから、最新の Windsurf を入手できます。
2. **ご利用の OS を選ぶ**  
   macOS / Windows / Linux 用のインストーラが用意されています。  
   本書では主に macOS を前提に説明します。
3. **インストール**  
   ダウンロードしたインストーラを開き、画面の案内にしたがって進めてください。  
   macOS の場合は、アプリケーションフォルダにドラッグ＆ドロップすれば完了です。

インストール後は、Dock やスタートメニューから Windsurf を起動できます。

### 初期設定のポイント

細かな設定は後から少しずつ触っていけば十分ですが、はじめに次のあたりを整えておくと快適です。

- **テーマやフォントサイズの調整**  
  長時間眺めても疲れにくい配色と文字サイズにしておきましょう。最初のうちは、コントラストがはっきりして見やすい `Monokai` テーマなどがおすすめです。
- **日本語表示の確認**  
  メニューやファイル名など、日本語が正しく表示されているか確認します。
- **ターミナルのシェル**  
  macOS であれば `zsh` など、ふだん使っているシェルが選ばれているかを確認しておくと安心です。

### コラム：ちょっとだけ Unix シェル

本書の内容は、基本的にはマウスやキーボード操作だけで完走できますが、長い目で見ると「シェル」と呼ばれるコマンドライン環境にも少しずつ慣れておくと役に立ちます。

macOS や Linux では、標準で Unix 系のシェル（`zsh` や `bash` など）が用意されていて、`ls` や `cd` のようなコマンドでファイル操作ができます。クラウドサーバやレンタルサーバでも同じコマンドがそのまま通用することが多く、「一度覚えるといろいろな場面で使い回せる」知識です。

Windows の場合も、PowerShell やコマンドプロンプトだけでなく、WSL（Windows Subsystem for Linux）や Git Bash などを使うことで、Unix 系のシェルに近い環境を手元に用意できます。将来クラウド開発などに進んでいきたい方は、少しずつ Unix 系のコマンドにも触れておくとよいでしょう。

この本の本文では、シェルの細かい使い方までは立ち入りませんが、「ちょっと試してみたい」という方のために、巻末に簡単な入門編を付けてあります。興味が湧いたら、付録の「Unix シェル超ダイジェスト（97-unix.md）」を覗いてみてください。

### おすすめ拡張機能

Windsurf には、見た目や作業効率を高めてくれる拡張機能（Extensions）がいくつか用意されています。ここでは、最初に入れておくと便利な系統を三つだけご紹介いたします。

- **日本語化（Language Pack）**  
  メニューやエラーメッセージなどを日本語で表示できる言語パックがある場合は、インストールしておくと安心です。拡張機能パネルで`Japanese Language Pack for VS Code`と検索し、有効化しておきましょう。
- **ローカルプレビュー（Go Live）**  
  HTML ファイルを保存するたびにブラウザを自動でリロードしてくれる拡張機能です。Windsurf にも `Live Server` が用意されていますので、本書ではこれを導入してローカルプレビュー用として使っていくことにいたしましょう。
- **ファイルアイコンテーマ**  
  ファイルの種類ごとにアイコンを変えてくれる拡張機能です。HTML や CSS、JavaScript、画像ファイルなどが一目で判別しやすくなり、プロジェクトの全体像をつかみやすくなります。Windsurf で利用可能なアイコンテーマには`Material Icon Theme`などがあります。お好みのものをひとつ選んでおくとよいでしょう。

拡張機能は「便利そうだから全部入れる」というよりも、「いまの自分の作業スタイルに本当に役立ちそうかどうか」を基準に、少しずつ増やしていくのがおすすめです。

### プロジェクトフォルダを開く

Windsurf では、プロジェクトごとにフォルダをひとつ開き、その中のファイルを編集していきます。

1. メニューから「File > Open Folder...」を選びます。  
2. 作業用のフォルダ（例：`projects/study`）を選択します。  
3. 左側のエクスプローラに、そのフォルダ内のファイルやサブフォルダが一覧表示されます。

あとは、新規ファイルを作成したり、既存のファイルを開いたりしながら、HTML や CSS、JavaScript を編集していくことができます。

## 基本的な開発方法

Windsurf と Firefox を導入できたので、ウェブサイトの基本的な作り方をご案内します。

- 0. 画面の左側に Windsurf を、右側に Firefox を配置します。
- 1. Windsurf で、プログラムをいろいろ書いていきます。
- 2. Firefox で、作ったウェブサイトの表示や動作を確認します。
- 3. 出来上がりが良ければ、ウェブサイトを公開して完了です。 <br>
    もう少し改良したければ、1. に戻ります。

- 2. Windsurf と Firefox を <br>左右に配置します。

- 3. Windsurf では、左側のエクスプローラに「フォルダ」を追加して管理します。作業用フォルダ（例：`projects/study`）を開くと、その中のファイルやディレクトリが一覧表示され、ファイルの追加や編集、削除を簡単に行うことができます。メニューの「File > Open Folder...」から、先ほど作成したフォルダを選びましょう。

- 4. `Finder`が開くので、サイドバーから `projects`を選びます。 `projects`ディレクトリから `study` ディレクトリを開きます。<br>左側の「ツリービュー」に `study` ディレクトリが表示されますので、この `study` ディレクトリ内にいろいろなファイルを納めていき、最終的にウェブサイトを創り上げます。新しいファイルを作成しましょう。


![](html1.png)

- 5. ツリービューの`study`ディレクトリを右クリックして、「新規ファイル」をクリックします。

![](html2.png)

![](html3.png)

- 7. 左側のエクスプローラに、新規作成した `index.html` が表示されましたので、これをクリックします。すると右側の編集領域（ペイン）に、今作成した `index.html` が表示されます。今作成したばかりなので、中身は何もなく空っぽです。早速 HTML を書いていきましょう。Windsurf にも、コードの入力を手助けしてくれる補完機能が備わっています。`html` と入力して、エンターキーを押します。


![](template.png)

- 8. 次のように雛形が入力されます。


![](study_html.png)

- 9. 「色」がついていることにも着目してください。「シンタックスハイライト」と呼ばれる、HTMLの文法に分かりやすいよう、着色する機能です。それでは雛形で入力した結果を利用して、次のようにしましょう。 `h1` とタイプしエンターキーを押すと、 `<h1></h1>` と入力されます。Atom の入力支援機能を活用して編集しましょう。

![](af6.png)

- 10. HTML を入力して、`index.html` ファイルが完成しました。Windsurf の「File」メニューから「Save」をクリックするか、`Command + S` を押して保存します。保存できたら、どのように表示されるか、Firefox を使って確認しましょう。Firefox の「ファイル」メニューから「ファイルを開く」をクリックするか、`Command + O` を押して、`index.html` を開きます。


![](af4b.png)

- 11. `Finder`が開くので、サイドバーから `projects`を選びます。 `projects`ディレクトリ内には、 `taskleaf` と `study` の 二つのディレクトリがありますが、ここでは `study` ディレクトリを開きます。


![](af8.png)

- 12. Windsurf で編集した `index.html` を Firefox で閲覧できます。


![](af9.png)

- 13. 綺麗に出来上がっていたら完成です。もう少し改善したい点があるなら、Windsurf に戻って `index.html` を編集します。編集が終わったら `Command + S` を押して保存します。保存が終わったら、編集結果を Firefox で見てみましょう。Firefox の「再読み込みボタン」を押すか、または `Command + R` を押して、編集結果を再確認することができます。
