# 置換リストで記法を拡張する

:::{.chapter-lead}
Vivlio Starter には、Markdown から HTML に変換したあと、ビルドの最終段階で文字列を書き換える「置換リスト」の仕組みがあります。これを使うと、CSS を書き足すだけでは実現しづらい **独自記法の追加** や **HTML 構造の微調整** を、正規表現 1 行で定義できます。本章では `config/post_replace_list.yml` の文法と、すでに組み込まれているビルトイン記法の使い方を解説します。
:::

## 置換リストとは

`config/post_replace_list.yml` は、`vs build` の後処理（`post_process`）で読み込まれる **HTML 文字列置換ルール** の定義ファイルです。VFM が生成した HTML に対して、上から順に `gsub` を適用していきます。

```yaml
---
- f: "検索パターン（正規表現）"
  r: "置換文字列（$1〜$9 でキャプチャ参照）"
```

- **`f`** — Ruby の正規表現パターン（前後のスラッシュ不要）
- **`r`** — 置換後の文字列。`$1`〜`$9` で後方参照が使える

ルールは **上から順に** 適用されるため、前段の置換結果に後段のルールがさらにマッチします。一時的に無効化したいときは、その行を `#` でコメントアウトするだけで済みます。

:::{.note}
**実装の出典**: この置換リストの発想は、[atom-markdown-book-preview](https://github.com/lwohtsu/atom-markdown-book-preview) に由来します。Vivlio Starter ではそれを Vivliostyle 向けに拡張し、会話文やガイド線、改ページなどの実用記法をビルトインで同梱しています。
:::

## ファイルの構成

`config/post_replace_list.yml` は、大きく次のグループに分かれています。

| グループ | 概要 |
| :--- | :--- |
| `:::` コンテナ展開 | Pandoc 風の `:::{.class}` を `<div class="...">` に展開 |
| 簡易マクロ | `@nega` `@posi` `@clear` `@comment` などの本文ショートカット |
| ガイド線・絶対配置 | リスト項目を自由な位置に配置する `@lu` `@ld` `@ru` `@rd` 系 |
| 会話・装飾 | 【先生】【生徒】、`〘 〙` キーキャップ、`▶`、`❶` など |
| HTML の整形 | 空段落除去や `<p>` と `<div>` のねじれ修正 |
| 段落クラス | `{.aki}` `{.aki2}` による段落後の余白調整 |

以下、実際に原稿で使える記法を見ていきます。

## 簡易マクロ

### 余白の調整 `@nega` / `@posi`

段落の上下に余白を差し込みます。図版とキャプションの間隔を詰めたり、節の直後に少し空けたりするのに便利です。

```markdown
本文の流れ。

@posi:10

次の段落を 10mm 下げて始めたい。
```

| 記法 | 展開結果 | 用途 |
| :--- | :--- | :--- |
| `@nega:N` | `<div style="margin-top:-Nmm"></div>` | 上方向に `N` mm 詰める |
| `@posi:N` | `<div style="margin-top:Nmm"></div>` | 下方向に `N` mm 空ける |

### 回り込みの解除 `@clear`

サイドイメージの直後で本文の回り込みを止めたいときに使います。

```markdown
:::{.sideimage-right}
![](images/fig.webp)
回り込むテキスト。
:::

@clear

次の節はここから新しい行で始まる。
```

### 編集者コメント `@comment ... @commend`

推敲中のメモを本文に残したいときに使います。`<span class="hen-comment">` に変換されるので、CSS で色を付けたりビルド時にまとめて消したりできます。

```markdown
本機能は @comment:要確認：仕様書 §3 @commend 次回リリース予定です。
```

## `:::` コンテナ記法

`:::{.class}` で始めて `:::` で閉じると、任意のクラスを持つ `<div>` にまとめられます。詳細は本書の拡張記法の章を参照してください。

```markdown
:::{.column}
補足コラムをここに書きます。
:::
```

置換リスト側では次の 2 本のルールがこれを担当しています。

```yaml
- f: ":{3,}\\{\\.?([a-z|0-9|\\-|_|\\s]+)\\}"
  r: <div class="$1">
- f: ":{3,}"
  r: "</div>"
```

同じ要領で、`@div:CLASS` … `@divend` の短縮形も用意されています。

```markdown
@div:memo
1 行だけクラス付き div で囲みたいとき。
@divend
```

## 会話文の記法

対話形式のページを書くときは、行頭に **【先生ラベル】** または **【生徒ラベル】** を置きます。`ラベル` 部分はクラス名としてそのまま CSS に渡るので、キャラクターごとにスタイルを切り替えられます。

```markdown
【先生A】Vivlio Starter の置換リストは正規表現で動いているんだ。

【生徒B】じゃあ、自分で好きな記法を足せるんですね！
```

展開後の HTML は次のようになります。

```html
<p class="kaiwa sense A">Vivlio Starter の置換リストは……</p>
<p class="kaiwa deshi B">じゃあ、自分で……</p>
```

## キーボード入力の `〘 〙`

半角の `<kbd>` を毎回書くのは手間なので、全角括弧で囲むだけで `<kbd>` タグに変換されます。

```markdown
保存は 〘Ctrl〙 + 〘S〙 で行います。
```

## リスト装飾

### ▶ で始まる「青コメ」

箇条書きの行頭を `▶` にすると、その `<li>` に `aokome` クラスが付きます。補足説明のリストを視覚的に区別したいときに便利です。

```markdown
- ▶ ここはポイント解説
- ▶ ここは補足
```

### ❶❷❸ で始まる「赤コメ」

囲み数字（`❶`〜`❿`、`⓫`〜`⓴`）で始まる項目は `akakome` クラスになり、番号部分が `<span>` で囲まれるのでフォントや色を個別に装飾できます。

```markdown
- ❶ 最初に設定ファイルを開く
- ❷ `post_replace_list.yml` を編集する
- ❸ `vs build` で反映を確認する
```

## リスト項目の絶対配置とガイド線

図解ページなどで、リスト項目を紙面上の好きな位置に引き出したい場合に使う上級記法です。項目末尾にガイド線の向きと寸法を付記すると、`<li>` が絶対配置に変換され、SVG の L 字ガイド線が自動で添えられます。

```markdown
- 項目テキスト@lu30,20@15,40
```

| 記法 | 向き | 意味 |
| :--- | :--- | :--- |
| `@luW,H@X,Y` | 左上 (left-up) | ガイドを左上方向に伸ばす |
| `@ldW,H@X,Y` | 左下 (left-down) | ガイドを左下方向に伸ばす |
| `@ruW,H@X,Y` | 右上 (right-up) | ガイドを右上方向に伸ばす |
| `@rdW,H@X,Y` | 右下 (right-down) | ガイドを右下方向に伸ばす |
| `@urW,H@X,Y` | 上右 (up-right) | ガイドを上→右へ曲げる |
| `@lsW@X,Y` | 左水平 | 左方向の水平線のみ |
| `@rsW@X,Y` | 右水平 | 右方向の水平線のみ |
| `@usH@X,Y` | 上垂直 | 上方向の垂直線のみ |
| `@dsH@X,Y` | 下垂直 | 下方向の垂直線のみ |

- `W`, `H` — ガイド線の幅・高さ（mm）
- `X`, `Y` — 項目を配置する左上座標（mm。親要素基準）

:::{.tip}
この記法は、図版の上に番号付きの注釈を並べる「図解ページ」で真価を発揮します。図と同じ座標系で注釈を置けるので、あとから図を差し替えても Markdown 側を修正するだけで済みます。
:::

## コード内コメントの強調

コードリスト（Prism でハイライトされた `<span class="token comment">`）の中で、矢印を含むコメントを目立たせる置換も組み込まれています。

| 書き方 | 効果 |
| :--- | :--- |
| `# ← コメント` | コメント行を赤色で強調 |
| `// ← コメント` | 同上（C系言語） |
| `/* ← コメント */` | ブロックコメントを赤色で強調 |
| `/* ←← コメント */` | 赤色＋右寄せ |
| `<!-- ← コメント -->` | HTML コメントを赤色で強調 |

```ruby
def greet(name)
  puts "Hello, #{name}!"   # ← この行が強調される
end
```

## 段落末のクラス指定 `{.aki}` / `{.aki2}`

段落の末尾に `{.aki}` または `{.aki2}` と書くと、その段落に該当クラスが付きます。図版の直前に余白を確保したいときに使います。

```markdown
ここで 1 行分の余白が入ります。{.aki}

次の段落はこの下から始まります。
```

## 改ページ

`---`（水平線）は、置換リストによって改ページ用の `<hr class="pagebreak">` に書き換えられます。章の途中で明示的に改ページしたい場合に使えます。

```markdown
ここまでが見開きの左ページ。

---

ここから右ページに進みたい。
```

## 独自ルールを追加する

新しい記法を追加したいときは、`config/post_replace_list.yml` に **1 行のルール** を書き足すだけです。たとえば「`@mark:文字`」で黄色マーカーを引きたい場合は次のようになります。

```yaml
- f: "@mark:([^@]+)@markend"
  r: <span class="marker-yellow">$1</span>
```

あとは `stylesheets/` 側で `.marker-yellow { background: #ffeb3b; }` のように定義すれば、原稿中で `@mark:重要@markend` と書くだけで蛍光ペン風の装飾が反映されます。

:::{.notice}
**注意**: 置換ルールは上から順に適用されます。追加したルールが既存ルールの結果に干渉しないか、必ず `vs build` で確認してください。正規表現が誤っているとビルド時に警告ログが出ますが、HTML は壊れないようフォールバックされます。
:::

## 歴史的な記法 — `?svgimg=…`

`post_replace_list.yml` のルーツである [atom-markdown-book-preview](https://github.com/lwohtsu/atom-markdown-book-preview) には、次のような **画像トリミング記法** がありました。

```markdown
![](sample.png?svgimg=30,180,200,-10,-10)
```

パラメータは `?svgimg=倍率,横幅mm,高さmm,横シフト量mm,縦シフト量mm` の順で、倍率以外は省略可能。IT 書のスクリーンショットをいちいちグラフィックスソフトで切り抜かずに済むようにする工夫でした。

Vivlio Starter では同等の用途を **CSS の `object-fit` と `transform`**、あるいは **`{width=…}` 属性** で実現しています。どうしても画像 URL 側だけで完結させたい場合は、本章の「独自ルールを追加する」を参考に、自前で `?svgimg=…` 用の置換ルールを書き起こすこともできます。

```yaml
# 例: ?svgimg=倍率 を transform:scale() に展開する自作ルール
- f: '<img src="([^"?]+)\\?svgimg=([0-9.]+)"'
  r: '<img src="$1" style="transform:scale($2);transform-origin:top left"'
```

## まとめ

- `config/post_replace_list.yml` は **HTML 後処理の置換ルール集**
- `f`（正規表現）と `r`（置換文字列）のペアを **上から順に** 適用
- `@nega` `@posi` `@clear` `@comment` `〘 〙` `▶` `❶` などのビルトイン記法が用意されている
- 独自ルールを 1 行追加するだけで、原稿中に自分だけの記法を生やせる

原稿が長くなってくると、同じ表現を何度も書くのが億劫になります。そんなときこそ置換リストの出番です。まずは既存ルールを眺めて、**よく書くパターン** を 1 つだけ短縮形にしてみてください。書き味がぐっと軽くなります。
