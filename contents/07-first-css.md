# CSSの考え方

:::{.chapter-lead}
HTMLで作った骨組みに「見た目」を与えるCSSを学びます。この章では、CSSの詳細なプロパティを暗記するのではなく、「CSSがどう働くのか」「どういう考え方で書くのか」という本質を押さえます。

プロパティの細かな指定方法は、AIに聞けばいつでも教えてもらえます。
:::



## CSSとは何か

### 構造と見た目の分離

CSSは「Cascading Style Sheets」の略で、HTMLで作った文書構造に「見た目」を指定するための言語です。

```
HTML → 文書の「意味・構造」（これは見出し、これは段落...）
CSS  → 文書の「見た目」（色、大きさ、配置...）
```

この分離には大きなメリットがあります：

- HTMLを変えずに、CSSだけでデザインを変更できる
- 同じCSSを複数のページで共有できる
- 役割が明確なので、コードが読みやすくなる

### CSSの基本構文

CSSは「どの要素に」「どんな装飾を」という形式で書きます：

```css
セレクタ {
  プロパティ: 値;
}
```

例えば：

```css
h1 {
  color: blue;      /* 文字色を青に */
  font-size: 24px;  /* 文字サイズを24pxに */
}
```

この例では、「`h1`要素を選んで、文字色を青に、サイズを24pxにする」という意味になります。

### CSSのコメント

CSSでは、`/*` と `*/` で囲んだ部分がコメントになります：

```css
/* ヘッダーのスタイル */
header {
  background: #fff;  /* 背景を白に */
}
```

コードの意図をメモしたり、一時的に無効化したりするのに使えます。

## セレクタの考え方

セレクタは「どの要素を装飾するか」を指定します。主な種類：

| セレクタ | 書き方 | 選ばれる要素 |
|---------|--------|-------------|
| 要素セレクタ | `p` | すべての`<p>`要素 |
| クラスセレクタ | `.hero` | `class="hero"`を持つ要素 |
| IDセレクタ | `#recent` | `id="recent"`を持つ要素 |
| 子孫セレクタ | `nav a` | `<nav>`の中にある`<a>` |

クラスセレクタ（`.`で始まる）は最もよく使います。HTMLの`class`属性と組み合わせることで、特定の要素だけを狙って装飾できます。

## カスケードと詳細度

CSSは「Cascading Style Sheets」の略で、「Cascading」は「滞のように流れ落ちる」という意味です。複数のスタイルが同じ要素に適用されるとき、あるルールに従ってどれが優先されるかが決まります。

### 詳細度（Specificity）

セレクタには「強さ」があります。強いセレクタのスタイルが優先されます：

| セレクタ | 強さ |
|---------|-------|
| 要素セレクタ（`p`） | 弱い |
| クラスセレクタ（`.hero`） | 中 |
| IDセレクタ（`#recent`） | 強い |

同じ強さのセレクタなら、後に書いたものが勝ちます。

```css
p { color: blue; }    /* 弱い */
.intro { color: red; } /* 中（こちらが適用される） */
```

「なぜこのCSSが効かないの？」と困ったときは、詳細度の問題かもしれません。開発者ツールで「どのスタイルが適用されているか」を確認できます。

## ボックスモデル

CSSを理解する上で最も大切な概念が「ボックスモデル」です。すべての要素は、4つの層でできた「箱」として扱われます：

![](box-model.svg)

- **content**: テキストや画像などの内容
- **padding**: 内容と枠線の間の余白
- **border**: 枠線
- **margin**: 枠線の外側の余白（他の要素との距離）

```css
.card {
  padding: 20px;             /* 内側の余白 */
  border:  1px solid #ccc; /* 枠線 */
  margin:  10px;             /* 外側の余白 */
}
```

開発者ツールで要素を選択すると、ボックスモデルが視覚的に表示されます。余白が思った通りにならないときは、ここを確認してみましょう。

## よく使うプロパティ

CSSには数百のプロパティがありますが、よく使うものは限られています：

### 色と文字

| プロパティ | 役割 | 例 |
|-----------|------|-----|
| `color` | 文字色 | `color: #333;` |
| `background` | 背景 | `background: white;` |
| `font-size` | 文字サイズ | `font-size: 16px;` |
| `font-weight` | 太さ | `font-weight: bold;` |
| `text-align` | 配置 | `text-align: center;` |

### 余白とサイズ

| プロパティ | 役割 | 例 |
|-----------|------|-----|
| `margin` | 外側の余白 | `margin: 20px;` |
| `padding` | 内側の余白 | `padding: 10px;` |
| `width` | 幅 | `width: 100%;` |
| `height` | 高さ | `height: auto;` |

### 論理プロパティ

| 軸 | プロパティ | 横書き日本語での意味 | 説明 | 例 |
|----|------------|----------------------|------|----|
| block | `margin-block` | 上下の外側の余白 | 上下まとめて指定 | `margin-block: 20px;` |
| block | `margin-block-start` | 上の外側の余白 | ブロックの「先頭側」の余白 | `margin-block-start: 16px;` |
| block | `margin-block-end` | 下の外側の余白 | ブロックの「終わり側」の余白 | `margin-block-end: 24px;` |
| inline | `margin-inline` | 左右の外側の余白 | 左右まとめて指定 | `margin-inline: 10px;` |
| inline | `margin-inline-start` | 左の外側の余白（横書き） | 行の「書き始め側」の余白 | `margin-inline-start: 8px;` |
| inline | `margin-inline-end` | 右の外側の余白（横書き） | 行の「書き終わり側」の余白 | `margin-inline-end: 12px;` |

論理プロパティは、`margin-top` や `margin-left` のように「上・下・左・右」を直接指定する代わりに、**文章の流れに沿った軸（`block` / `inline`）と、開始側・終了側** で余白を指定する書き方です。

- 横書きの日本語の場合、`block` は上下方向、`inline` は左右方向を表します。
- 縦書きの日本語や右から左に書く言語に変えても、CSSを書き直さずにレイアウトを保ちやすい、という利点があります。

`padding-block` / `padding-inline` や、`border-block` / `border-inline` も同じ考え方で使えます。

### 枠線と影

| プロパティ | 役割 | 例 |
|-----------|------|-----|
| `border` | 枠線 | `border: 1px solid #ccc;` |
| `border-radius` | 角丸 | `border-radius: 8px;` |
| `box-shadow` | 影 | `box-shadow: 0 2px 4px rgba(0 0 0 / 10%);` |

### 色の指定方法

CSSでは、色を指定する方法がいくつかあります：

| 形式 | 書き方 | 説明 |
|------|--------|------|
| カラー名 | `red`, `blue` | 英語の色名をそのまま使う |
| 16進数 | `#ff0000`, `#f00` | 赤・緑・青をそれぞれ00～ffで指定 |
| 16進数（アルファ付き） | `#ff000088`, `#f008` | 赤・緑・青に加えて末尾2桁で透明度を指定。半透明の色を表現できる |
| RGB | `rgb(255 0 0)` | 赤・緑・青を0～255で指定 |
| RGB（アルファ付き） | `rgb(255 0 0 / 50%)` | RGB + 透明度（0～100%）。半透明の色を直感的に指定できる |
| HSL | `hsl(0 100% 50%)` | 色相・彩度・明度（Hue / Saturation / Lightness）で指定 |

```css
/* 同じ赤色を異なる形式で */
.red1 { color: red; }
.red2 { color: #ff0000; }
.red3 { color: rgb(255 0 0); }
```

半透明の背景には、`rgb(0 0 0 / 50%)` のように透明度付きの `rgb()` が便利です。グラデーションで透明度を使うときにもよく使います。

:::{.note}
**プロパティの詳細はAIに聞こう**

「この色をもう少し薄くしたい」「余白の指定方法がわからない」といったときは、AIに聞いてみましょう。

> 「CSSで要素を中央寄せにする方法を教えて」
> 「marginとpaddingの違いは？」
> 「グラデーション背景の書き方は？」

のように質問すれば、具体的なコードとともに説明してくれます。
:::

## CSS Gridによるレイアウト

現代のCSSでは、**CSS Grid**を使ったレイアウトが主流です。ページを「行と列のマス目」として考え、要素を配置していきます。

```css
body {
  display: grid;
  grid-template:
    "header header header" 80px
    "main   main   main  " auto
    "footer footer footer" 100px /
    20px    1fr    20px;
}
```

この例では：
- 3行（header、main、footer）
- 3列（左余白20px、中央は残り全部、右余白20px）

というレイアウトを定義しています。

### グリッドの考え方

CSS Gridの基本的な考え方：

1. **コンテナ**（親要素）に`display: grid;`を指定
2. **行と列**を`grid-template`で定義
3. **子要素**を`grid-area`でマス目に配置

```css
/* 親要素でグリッドを定義 */
body {
  display: grid;
  grid-template-areas:
    "head"
    "main"
    "foot";
}

/* 子要素をマス目に配置 */
header { grid-area: head; }
main   { grid-area: main; }
footer { grid-area: foot; }
```

雑誌の紙面レイアウトを考えるように、「ここにヘッダー、ここに本文、ここにフッター」と配置していくイメージです。

## display プロパティ

`display` プロパティは、要素の表示方法を制御します。HTMLの「ブロック要素・インライン要素」と深く関わります：

| 値 | 動作 |
|-----|------|
| `block` | 横幅いっぱいに広がり、前後で改行される |
| `inline` | 内容の幅だけ。`width`/`height`が効かない |
| `inline-block` | 横並びしつつ、`width`/`height`も指定可 |
| `none` | 要素を非表示にする |
| `flex` | Flexboxレイアウトを使う（並べるのに便利） |
| `grid` | Gridレイアウトを使う（大きなレイアウトに） |

```css
/* リンクをボタンのように表示 */
a.button {
  display: inline-block;
  padding: 10px 20px;
  background: #007bff;
}
```

`display: none;` はレスポンシブデザインで「スマホでは非表示」にするときにも使います。

## CSSファイルの読み込み

HTMLからCSSを読み込むには、`<head>`の中に次のように書きます：

```html
<link rel="stylesheet" href="style.css">
```

CSSファイルは通常、HTMLと同じフォルダか、`stylesheets/`のような専用フォルダに置きます。

## リセットCSSという考え方

ブラウザには、見出しや段落に対してあらかじめ「標準のスタイル」が設定されています。ただ、ブラウザごとに微妙に異なるため、自分でデザインを整えたいときには少し扱いづらくなります。

そこで、まず「リセットCSS」で標準スタイルを打ち消してから、自分のスタイルを重ねていくのが一般的です：

```css
* {
  margin: 0;
  padding: 0;
  box-sizing: border-box;
}
```

`*`（ユニバーサルセレクタ）で全要素を選び、余白をゼロにリセットしています。`box-sizing: border-box;`は、幅の計算をしやすくするための指定です。

## CSSの単位

CSSでは、サイズを指定するときに様々な「単位」を使います。

### 絶対単位と相対単位

| 単位 | 種類 | 説明 | 例 |
|------|------|------|-----|
| `px` | 絶対 | ピクセル。画面の1点 | `font-size: 16px;` |
| `%` | 相対 | 親要素に対する割合 | `width: 50%;` |
| `em` | 相対 | 親要素の文字サイズ基準 | `margin: 1.5em;` |
| `rem` | 相対 | ルート要素の文字サイズ基準 | `font-size: 1.2rem;` |
| `vw` | 相対 | ビューポート幅の1% | `width: 100vw;` |
| `vh` | 相対 | ビューポート高さの1% | `height: 100vh;` |
| `fr` | 相対 | グリッドの残りスペースの割合 | `grid-template-columns: 1fr 2fr;` |

### よく使う単位の使い分け

- **`px`**: ボーダーや細かな調整に。固定サイズが必要な場面で
- **`%`**: 親要素に対する相対サイズ。レイアウトの幅指定に
- **`rem`**: 文字サイズの指定に最適。アクセシビリティにも配慮
- **`vw`/`vh`**: 画面全体を基準にしたい場面で
- **`fr`**: CSS Gridで列幅・行高さを柔軟に分配

```css
/* 例：レスポンシブな文字サイズ */
h1 {
  font-size: clamp(24px, 5vw, 48px);  /* 最小24px、最大48px */
}
```

`clamp()` 関数を使うと、最小値と最大値の間で自動調整されるサイズを指定できます。

## レスポンシブデザイン

### モバイルファースト

現代のウェブサイトは、スマートフォン・タブレット・PCなど、様々な画面サイズで閲覧されます。どの端末でも快適に見られるデザインを**レスポンシブデザイン**と呼びます。

本書では**モバイルファースト**のアプローチを取ります：

1. まずスマートフォン向けのCSSを書く
2. 画面が広くなったら、追加のスタイルを上書き

この順番で書くと、シンプルで保守しやすいCSSになります。

### メディアクエリ

画面サイズに応じてスタイルを切り替えるには、**メディアクエリ**を使います：

```css
/* 基本スタイル（スマートフォン向け） */
section#recent {
  grid-template-columns: 1fr;  /* 1列 */
}

/* 768px以上の画面（タブレット・PC向け） */
@media (width >= 768px) {
  section#recent {
    grid-template-columns: repeat(3, 1fr);  /* 3列 */
  }
}
```

`@media (width >= 768px)` は「画面幅が768px以上のとき」という条件を表します。この書き方は**Range Query構文**と呼ばれ、従来の `@media (min-width: 768px)` より直感的です。

### 代表的なブレークポイント

画面サイズの境目を**ブレークポイント**と呼びます。

**一般的な目安**

| 端末 | 画面幅 |
|------|--------|
| スマートフォン | 〜767px |
| タブレット | 768px〜1023px |
| PC | 1024px〜 |

ただし、これは目安に過ぎません。実際には「コンテンツが窮屈に見え始めたら切り替える」という考え方が大切です。

### 開発者ツールでプレビュー

ブラウザの開発者ツール（`F12`または右クリック→「調査」）には、様々な端末での表示をシミュレートする機能があります。iPhone や iPad を選んで、レスポンシブデザインの確認ができます。

## AIへの質問例

CSSで困ったときの質問例：

> **基本的な質問**
> - 「要素を横並びにするには？」
> - 「ボタンにホバーエフェクトを付けたい」
> - 「画像を丸く切り抜くには？」

> **レイアウトの質問**
> - 「CSS Gridで2カラムレイアウトを作りたい」
> - 「フッターを常に画面下部に固定したい」
> - 「スマホとPCで配置を変えるには？」

> **デバッグの質問**
> - 「なぜこの余白が効かないの？」
> - 「要素が重なってしまう原因は？」
> - 「このCSSが効いているか確認する方法は？」
