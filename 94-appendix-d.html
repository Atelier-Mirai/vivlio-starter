Appendix-D よくあるエラーと対処法

学習・制作の現場で遭遇しがちなエラーを「症状→原因→確認→解決→再発防止」の順で短く整理しました。まずは共通チェックリストを確認し、該当セクションに進んでください。

最初にやること（共通手順）

  • 環境情報の把握: OS/アーキテクチャ、主要ツール/言語のバージョン
  • エラーメッセージ: 先頭50行と末尾50行を確認（ノイズは無視）
  • 直前の変更点: 設定/依存/PATH/権限/ネットワーク

共通チェック（概要）

  • PATH と権限（実行権、所有者、アクセシビリティ）
  • ネットワーク（プロキシ/VPN/FW、トークン期限）
  • キャッシュ/ロック破損（削除→再生成で改善するか）

インストール/環境のエラー

セットアップで多いトラブルをまとめました。

症状: コマンドが見つからない/動かない

  • 原因: PATH 未設定、実行権限なし、別バージョンが先に解決
  • 確認: which <cmd>、echo $PATH、ls -l <file>
  • 解決: PATH 追加、chmod +x、シェルの優先順位/シンボリックリンク調整
  • 再発防止: シェル初期化ファイルへ恒久設定、バージョンマネージャ導入

症状: 依存解決に失敗する

  • 原因: ロック不整合、キャッシュ破損、ネットワーク遮断
  • 確認: パッケージマネージャの詳細ログ（--verbose/--debug）
  • 解決: ロック/キャッシュ再生成、ミラー変更、リトライ
  • 再発防止: バージョン固定、CI での再現テスト

実行時エラー（スクリプト/コード）

症状: スタックトレースが出る

  • 原因: 入力の前提違い、null/未定義、境界ケース未処理
  • 確認: 例外の最下段付近（原因箇所）、最小再現の作成
  • 解決: 入力バリデーション、ガード節/デフォルト値の追加
  • 再発防止: 単体テスト、境界値テスト

症状: 文字化け

  • 原因: エンコーディングの混在（UTF-8/Shift_JIS 等）
  • 確認: ファイル実体のエンコーディング、CLI ロケール
  • 解決: 明示的 UTF-8 指定、変換フィルタ適用
  • 再発防止: エディタ/ツールのデフォルト統一

ビルド/生成のトラブル

症状: 生成が止まる/落ちる（PDF/HTML 等）

  • 原因: 画像サイズ過大、欠損パス、テンプレの構文エラー
  • 確認: エラーログ直前の1–3件、該当ファイルの存在/拡張子
  • 解決: 画像圧縮/形式変換、相対パス見直し、テンプレ修正
  • 再発防止: 自動検査（lint/リンクチェッカ）を CI へ

症状: 変更が反映されない

  • 原因: キャッシュヒット、生成物の残骸、監視対象外
  • 確認: タイムスタンプ、クリーン後の再ビルド
  • 解決: rake clean 相当を実行、ウォッチ設定の見直し
  • 再発防止: ビルド手順の一貫化、クリーン運用の徹底

ネットワーク/認証

症状: リモートに接続できない

  • 原因: プロキシ/VPN、FW、トークン期限切れ
  • 確認: curl -I <url>、DNS 解決、認証ヘッダの有無
  • 解決: プロキシ設定、証明書更新、トークン再発行
  • 再発防止: 有効期限のリマインド、自動更新/秘密管理の分離

付録: 共通チェックリスト（保存版）

  • 権限: 実行権限、所有者、アクセシビリティ設定（macOS）
  • PATH: 期待するコマンドが先に解決されるか
  • 環境変数: LANG, LC_ALL, HTTP_PROXY など
  • キャッシュ: ロック/キャッシュ削除で改善するか
  • ネットワーク: オフライン/プロキシ/証明書の影響
  • 再現性: 別ディレクトリ/新規プロファイル/CI で再現するか
