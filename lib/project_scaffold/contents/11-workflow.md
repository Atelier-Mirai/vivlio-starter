# 執筆ワークフロー概観

:::{.chapter-lead}
Vivlio Starter を使った書籍制作は、「プロジェクト作成 → 執筆 → 整える → ビルド → 入稿・配布」という5つのステップで完結します。本章では、それぞれのステップで何をするのかを俯瞰します。各ステップの詳細は、対応する章を参照してください。
:::

## 各ステップの概要

### ① プロジェクト作成

`vs new` コマンドで書籍プロジェクトの雛形を生成します。書籍名・著者名などを対話形式で入力すると、必要なファイル一式が自動的に用意されます。さらに内部で `vs doctor --fix` が自動的に呼び出され、Vivliostyle や ImageMagick など、書籍制作に必要な外部ツールが一括でセットアップされます。

```bash
vs new mybook   # 雛形生成 + 必要ツールの自動セットアップ
cd mybook
```

**参照:** 「インストール」「新規プロジェクトの作成（vs new）」「環境診断（vs doctor）」

### ② 執筆

`contents/` ディレクトリ内の Markdown ファイルに原稿を書きます。章ファイルの追加・削除・番号の振り直しには専用のコマンドが用意されており、ファイルの手動操作は不要です。

```bash
vs create 10-intro   # 新しい章を追加
vs build  10-intro   # その章だけ素早くビルドして確認
```

章単位でビルドすると、全体を組版するより速く結果を確認できます。書きながら `vs build <章名>` でこまめに確認するのが、Vivlio Starter での執筆スタイルです。

**参照:** 「Markdown 執筆チュートリアル」「章の管理」

### ③ 整える

原稿が一段落したら、文章の品質を確認します。

`vs lint` は textlint を使って表記揺れや冗長表現を検出します。`vs metrics` は文字数・漢字比率・語彙多様度などを数値化し、読みやすさの目安を示します。

```bash
vs lint       # 文章を校正する
vs metrics    # 原稿の統計を確認する
```

**②と③は何度でも繰り返せます。** 書いては整え、また書く——この往復が、読みやすい技術書を仕上げる近道です。

**参照:** 「Textlint」「Metrics」

### ④ ビルド

執筆が完了したら、`vs build` で書籍全体を組版します。閲覧用 PDF・印刷入稿用 PDF・EPUB・Kindle の四形式を、一度のコマンドで生成できます。

```bash
vs build   # 書籍全体をビルドする
```

カバー画像がまだ生成されていない場合、`vs build` が内部で自動的に `vs cover` を呼び出すため、別途実行する必要はありません。カバーのデザインを細かく調整したいときや、カバーだけを単独で生成・確認したい場合には `vs cover` を直接実行します。

```bash
vs cover   # カバーを個別に生成・調整したい場合
```

出力形式は `config/book.yml` の `output.targets` で指定します。技術書典向けには `pdf`、印刷所入稿には `print_pdf`（トンボ・塗り足し付き）、電子書籍配信には `epub` または `kindle` を選びます。

**参照:** 「ビルド（vs build）」「カバー画像の生成」「config/book.yml リファレンス」

### ⑤ 入稿・配布

生成された PDFやEPUBなどを、頒布チャネルに合わせて提出・アップロードします。

| 頒布先 | 使用するファイル |
| :--- | :--- |
| 技術書典・ダウンロード販売・PDF 配布 | `pdf`（閲覧用） |
| コミケ（印刷所入稿） | `print_pdf`（トンボ付き） |
| Apple Books, Kobo | `epub`|
| Kindle | `kindle`（専用） |

## コマンド早見表

| ステップ | 主なコマンド | 内容 |
| :--- | :--- | :--- |
| ① プロジェクト作成 | `vs new` | 雛形生成＋必要ツール自動セットアップ |
| ② 執筆 | `vs create` | 章ファイルを追加 |
| | `vs build <章名>` | 章単位で素早く確認 |
| ③ 整える | `vs lint` | 文章を校正 |
| | `vs metrics` | 原稿の統計を確認 |
| ④ ビルド | `vs build` | 書籍全体をビルド（カバー未生成時は自動生成） |
| | `vs cover` | カバーを個別に生成・調整したい場合 |
| ⑤ 入稿・配布 | — | 生成済みファイルを提出 |

これ以外にも便利なコマンドが揃っています。全体は `vs --help` で確認できます。

```bash
vs --help
```
